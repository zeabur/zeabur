# 构建

Zeabur 使用 [zbpack](https://github.com/zeabur/zbpack) 作为内部工具来构建用户服务，使用户无需了解复杂的细节，即可一键部署基于任何语言和框架的服务。

目前，[zbpack](https://github.com/zeabur/zbpack) 已经支持了所有主流的 Web 开发编程语言，并针对每个流行的开发框架进行了进一步的识别和优化。同时，它也在不断更新以支持新的编程语言和框架：

- Node.js
- Python
- PHP
- Ruby
- Go
- Java
- .NET
- Rust
- Elixir

如果您服务所使用的语言或框架不在上述列表中，或者您发现 zbpack 在构建过程中存在问题，请向 [GitHub](https://github.com/zeabur/zbpack/pulls) 提交 Pull Request (拉取请求) 以帮助我们改进 zbpack 的功能。

当您在项目中使用 zbpack 构建服务时，它会根据项目中的代码、配置文件和其他信息自动识别服务所使用的语言和框架，并根据这些信息选择合适的方式来构建服务。

## 生成构建计划

例如，假设您开发了一个基于 [Next.js](https://nextjs.org/) 的 Web 应用程序：

```bash
git clone https://github.com/zeabur/nextjs-template
cd nextjs-template
```

当您在该项目的文件夹中运行 `zbpack` 命令时，您可以看到为此定制的构建计划：

```bash
zbpack .

╔══════════════════════════════ Build Plan ═════════════════════════════╗
║ provider         │ nodejs                                             ║
║───────────────────────────────────────────────────────────────────────║
║ nodeVersion      │ 18                                                 ║
║───────────────────────────────────────────────────────────────────────║
║ installCmd       │ pnpm install                                       ║
║───────────────────────────────────────────────────────────────────────║
║ buildCmd         │ pnpm run build                                     ║
║───────────────────────────────────────────────────────────────────────║
║ startCmd         │ pnpm start                                         ║
║───────────────────────────────────────────────────────────────────────║
║ packageManager   │ pnpm                                               ║
║───────────────────────────────────────────────────────────────────────║
║ framework        │ next.js                                            ║
╚═══════════════════════════════════════════════════════════════════════╝
```

从图中我们可以看到，`zbpack` 选择使用 `nodejs` Provider (提供者) 作为构建的解决方案提供者，这意味着它将在后续计划中执行以下操作：

1. 确定服务使用的 Node.js 版本
2. 确定服务使用的包管理器，是 `npm`、`yarn` 还是 `pnpm`
3. 确定服务用于安装依赖的命令
4. 确定服务用于构建的命令
5. 确定服务用于启动的命令
6. 确定服务使用的 Node.js 框架
