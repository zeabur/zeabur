# コミュニティフォーラム

[Zeabur コミュニティフォーラム](https://zeabur.com/forum)は、ユーザーが Zeabur に関する議論、質問、サポートを受けるための公式ハブです。サービスの不具合、電話番号認証、請求に関する質問など、どのような内容でもこのフォーラムをご利用ください。

## サポートの優先順位

私たちはすべての方にサポートを提供できるよう努めています。最適な体験を提供するため、プランに基づいてサポートの優先順位を決定しています。

| プラン | 予想返信時間 |
| --- | --- |
| Team プラン | 12時間以内 |
| Dev プラン | 3営業日以内 |
| Free プラン | 5営業日以内 |

問題をできるだけ早く解決するために、投稿を作成する際は可能な限り多くの情報を提供してください。詳細な情報があるほど、追加の質問を減らすことができます。

<Callout type="info">
  **Team プラン**のユーザー様には、普段お使いのメッセージングアプリケーション（Discord、WhatsApp など）でプライベートグループチャットを作成します。フォーラムに投稿する代わりに、そのプライベートチャンネルで直接質問することができます。これにより、プライバシーが守られ、より直感的なサポート体験が可能になり、スタッフがお客様の状況をより迅速に把握できるようになります。
</Callout>

## 新規投稿の作成

新規投稿を作成するには、[zeabur.com/forum/new](https://zeabur.com/forum/new) にアクセスしてください。以下の項目を入力する必要があります。

![new post](/community/forum/new-post.png)

### 問題の種類 (Issue Type)

発生している問題の種類を選択してください。選択した問題の種類に応じて、フォームの入力項目が変化します。

- **サービスの問題 (Service Issue)** — デプロイ済みのサービスに関する問題（例：デプロイの失敗、ランタイムエラー、ネットワークの問題）。
- **電話番号認証 (Phone Verification)** — アカウントの電話番号認証をリクエストする場合。
- **返金リクエスト (Refund Request)** — サーバー、サブスクリプション、または残高の返金をリクエストする場合。
- **その他 (Others)** — 上記のカテゴリに該当しないその他の質問や問題。

### タイトル (Title)

問題の短い要約（最大150文字）です。チームが問題を迅速に理解できるよう、明確かつ具体的に記載してください。

### 基本情報 (Basic Info)

<Callout>
  問題の種類に応じて、追加情報の提供が必要になる場合があります。この情報は公開されず、トラブルシューティングを迅速化するためにのみ使用されます。これにより、スタッフが問題を調査する際の透明性が向上します。
</Callout>

#### サービスの問題 (Service Issue)

**Project** または **Server** タブのいずれかを選択してください。

- **Project タブ**: ドロップダウンから **Project** と **Service** を選択してください。これは、お客様のデプロイを特定するために必要です。
- **Server タブ**: 問題が特定のサーバーに関連している場合は、ドロップダウンから **Server** を選択してください。

また、**エラーメッセージ (Error Message)** を貼り付けることもできます（任意）。Zeabur ダッシュボードの右下にエラー通知（トースト）が表示された場合は、通知のコピーボタンをクリックして、そのエラーメッセージをこのフィールドに貼り付けてください。

#### 電話番号認証 (Phone Verification)

- **国 / 地域 (Country / Region)**: 国または地域を選択してください。
- **電話番号 (Phone Number)**: 認証したい電話番号を入力してください。

<Callout>
  電話番号認証の投稿はプライベート設定となり、お客様と Zeabur スタッフのみが閲覧できます。これにより、個人情報を保護しつつ、チームがサポートに必要な詳細情報を確認できるようになります。
</Callout>

#### 返金リクエスト (Refund Request)

返金をリクエストする対象に応じて、**Server**、**Subscription**、または **Balance** タブを選択してください。

- **Server タブ**: **Server** と返金の**理由 (Reason)** を選択してください。
- **Subscription タブ**: 返金の**理由 (Reason)** を選択してください。
- **Balance タブ**: 返金の**理由 (Reason)** を選択してください。

<Callout>
  返金リクエストの投稿はプライベート設定となり、お客様と Zeabur スタッフのみが閲覧できます。これにより、請求詳細の安全性を保ちながら、チームがリクエストを効率的に処理できるようになります。
</Callout>

#### その他 (Others)

追加の基本情報は必要ありません。内容（Content）フィールドにできるだけ詳細を記載してください。

### 内容 (Content)

リッチテキストエディタを使用して、問題を詳細に説明してください。見出し、リスト、コードブロック、リンクなどを使用してテキストをフォーマットできます。

**良い投稿を書くためのヒント:**

- 何をしようとして、代わりに何が起こったのかを説明してください。
- 問題を再現するための手順を含めてください。
- 問題が発生する前に行った変更があれば記載してください。
- 関連するログやエラーメッセージを貼り付けてください。
- エディタにドロップまたは貼り付けることで、スクリーンショットやファイルを添付してください。

説明が具体的であるほど、重複した質問をすることなく、より迅速にサポートを提供できます。
