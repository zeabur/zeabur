# ビルド

Zeabur は、ユーザーのサービスをビルドするための内部ツールとして [zbpack](https://github.com/zeabur/zbpack) を使用しています。これにより、ユーザーは複雑な詳細を理解することなく、あらゆる言語やフレームワークに基づいたサービスをワンクリックでデプロイできます。

現在、[zbpack](https://github.com/zeabur/zbpack) は Web 開発におけるすべての主要なプログラミング言語をサポートしており、さらに各人気開発フレームワークの識別と最適化を行っています。同時に、新しいプログラミング言語やフレームワークへの対応も継続的に更新されています。

- Node.js
- Python
- PHP
- Ruby
- Go
- Java
- .NET
- Rust
- Elixir

もし、お使いのサービスで使用されている言語やフレームワークが上記のリストにない場合、またはビルドプロセス中に zbpack の問題を発見した場合は、[GitHub](https://github.com/zeabur/zbpack/pulls) にプルリクエスト（Pull Request）を送って、zbpack の機能改善にご協力ください。

プロジェクトで zbpack を使用してサービスをビルドすると、プロジェクト内のコードや設定ファイル、その他の情報に基づいて、サービスで使用されている言語やフレームワークを自動的に識別します。そして、その情報に従って、サービスをビルドするための適切な方法を選択します。

## ビルドプランの生成

例えば、[Next.js](https://nextjs.org/) に基づいた Web アプリケーションを開発したとします。

```bash
git clone https://github.com/zeabur/nextjs-template
cd nextjs-template
```

このプロジェクトのフォルダーで `zbpack` コマンドを実行すると、以下のようにカスタマイズされたビルドプランを確認できます。

```bash
zbpack .

║ framework        │ next.js                                            ║
╚═══════════════════════════════════════════════════════════════════════╝
```

この図から、`zbpack` がビルドのソリューションプロバイダーとして `nodejs` プロバイダー（Provider）を使用することを選択したことがわかります。これは、今後のプランで以下のことを行うことを意味します。

1. サービスがどのバージョンの Node.js を使用しているかを特定する
2. サービスが `npm`、`yarn`、`pnpm` のどのパッケージマネージャーを使用しているかを特定する
3. サービスが依存関係をインストールするためにどのコマンドを使用するかを特定する
4. サービスがビルドのためにどのコマンドを使用するかを特定する
5. サービスが起動のためにどのコマンドを使用するかを特定する
6. サービスがどの Node.js フレームワークを使用しているかを特定する
