# 社群論壇

[Zeabur 社群論壇](https://zeabur.com/forum) 是使用者討論、提問以及獲取 Zeabur 相關協助的官方中心。無論您是遇到服務問題、需要手機驗證，還是有帳務問題，論壇都是您的首選之地。

## 支援優先級

我們致力於為每個人提供協助。為了確保最佳體驗，我們會根據您的方案優先提供支援：

| 方案 | 預計回應時間 |
| --- | --- |
| 團隊方案 (Team Plan) | 12 小時內 |
| 開發者方案 (Dev Plan) | 3 個工作天內 |
| 免費方案 (Free Plan) | 5 個工作天內 |

為了幫助我們盡快解決您的問題，請在建立貼文時提供盡可能詳細的資訊。您提供的細節越多，我們需要追問的問題就越少。

<Callout type="info">
  如果您是 **團隊方案 (Team plan)** 使用者，我們會在您最常用的通訊軟體（例如 Discord、WhatsApp）中建立一個私訊群組。您可以直接在該私訊頻道中提出所有問題，而無需在論壇上發文 —— 這能提供更好的隱私保護、更直覺的支援體驗，並幫助我們的團隊更快了解您的情況。
</Callout>

## 建立新貼文

若要建立新貼文，請前往 [zeabur.com/forum/new](https://zeabur.com/forum/new)。您需要填寫以下欄位。

![new post](/community/forum/new-post.png)

### 問題類型 (Issue Type)

選擇您遇到的問題類型。表單欄位會根據您選擇的問題類型而改變。

- **服務問題 (Service Issue)** — 關於您部署的服務問題（例如：部署失敗、執行階段錯誤、網路問題）。
- **手機驗證 (Phone Verification)** — 申請對您的帳號進行手機號碼驗證。
- **退款申請 (Refund Request)** — 申請伺服器、訂閱或餘額的退款。
- **其他 (Others)** — 任何上述類別未涵蓋的其他問題。

### 標題 (Title)

問題的簡短摘要（最多 150 個字元）。請保持清晰且具體，以便我們的團隊快速了解您的問題。

### 基本資訊 (Basic Info)

<Callout>
  根據問題類型的不同，您可能需要提供額外資訊。這些資訊不會公開，僅用於加速疑難排解 (troubleshooting) 程序 —— 這能讓我們的團隊在調查您的問題時更加清晰。
</Callout>

#### 服務問題 (Service Issue)

選擇 **專案 (Project)** 或 **伺服器 (Server)** 分頁：

- **專案分頁**：從下拉選單中選擇您的 **專案** 和 **服務**。這是必填項，以便我們定位您的部署。
- **伺服器分頁**：如果問題與特定伺服器有關，請從下拉選單中選擇您的 **伺服器**。

您也可以貼上 **錯誤訊息 (Error Message)**（選填）。如果您在 Zeabur 控制台的右下角看到錯誤提示 (toast)，請點擊提示上的複製按鈕並將錯誤訊息貼到此欄位中。

#### 手機驗證 (Phone Verification)

- **國家 / 地區 (Country / Region)**：選擇您的國家或地區。
- **電話號碼 (Phone Number)**：輸入您想要驗證的電話號碼。

<Callout>
  手機驗證貼文是私密的，僅對您和 Zeabur 工作人員可見。這能確保您的個人資訊受到保護，同時提供我們團隊所需的細節來協助您。
</Callout>

#### 退款申請 (Refund Request)

根據您申請退款的對象，選擇 **伺服器 (Server)**、**訂閱 (Subscription)** 或 **餘額 (Balance)** 分頁：

- **伺服器分頁**：選擇您的 **伺服器** 並填寫退款 **原因**。
- **訂閱分頁**：選擇退款 **原因**。
- **餘額分頁**：選擇退款 **原因**。

<Callout>
  退款申請貼文是私密的，僅對您和 Zeabur 工作人員可見。這能確保您的帳務細節安全，同時讓我們的團隊高效處理您的請求。
</Callout>

#### 其他 (Others)

不需要額外的基本資訊。只需在內容欄位中提供盡可能詳細的資訊即可。

### 內容 (Content)

使用富文本編輯器詳細描述您的問題。您可以使用標題、清單、程式碼區塊、連結等格式化您的文字。

**撰寫優質貼文的建議：**

- 描述您嘗試執行的操作以及實際發生的情況。
- 包含重現問題的步驟。
- 提及在問題發生前您所做的任何變更。
- 貼上相關的日誌或錯誤訊息。
- 透過拖放或貼上到編輯器中來附加螢幕截圖或檔案。

您的描述越詳盡，我們就能越快提供協助，而無需詢問重複的問題。
