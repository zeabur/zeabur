# 建置 (Builds)

Zeabur 使用 [zbpack](https://github.com/zeabur/zbpack) 作為內部工具來建置使用者服務，讓使用者無需了解複雜的細節，即可一鍵部署基於任何語言和框架的服務。

目前，[zbpack]((https://github.com/zeabur/zbpack)) 已經支援所有主流的網頁開發程式語言，並進一步針對各個熱門的開發框架進行了識別與優化。同時，它也在不斷更新以支援新的程式語言和框架：

- Node.js
- Python
- PHP
- Ruby
- Go
- Java
- .NET
- Rust
- Elixir

如果您服務所使用的語言或框架不在上述列表中，或者您發現 zbpack 在建置過程中出現問題，請向 [GitHub](https://github.com/zeabur/zbpack/pulls) 提交 Pull Request，幫助我們改進 zbpack 的功能。

當您在專案中使用 zbpack 建置服務時，它會根據專案中的程式碼、設定檔和其他資訊，自動識別服務所使用的語言和框架，並根據這些資訊選擇合適的方式來建置服務。

## 產生建置計畫 (Generate Build Plan)

例如，假設您開發了一個基於 [Next.js](https://nextjs.org/) 的網頁應用程式：

```bash
git clone https://github.com/zeabur/nextjs-template
cd nextjs-template
```

當您在該專案的資料夾中執行 `zbpack` 指令時，您可以看到為此產生的自定義建置計畫：

```bash
zbpack .

╔══════════════════════════════ Build Plan ═════════════════════════════╗
║ provider         │ nodejs                                             ║
║───────────────────────────────────────────────────────────────────────║
║ nodeVersion      │ 18                                                 ║
║───────────────────────────────────────────────────────────────────────║
║ installCmd       │ pnpm install                                       ║
║───────────────────────────────────────────────────────────────────────║
║ buildCmd         │ pnpm run build                                     ║
║───────────────────────────────────────────────────────────────────────║
║ startCmd         │ pnpm start                                         ║
║───────────────────────────────────────────────────────────────────────║
║ packageManager   │ pnpm                                               ║
║───────────────────────────────────────────────────────────────────────║
║ framework        │ next.js                                            ║
╚═══════════════════════════════════════════════════════════════════════╝
```

從圖中我們可以看到，`zbpack` 選擇使用 `nodejs` Provider 作為建置的解決方案提供者，這意味著它在後續計畫中將執行以下操作：

1. 找出服務使用的 Node.js 版本
2. 找出服務使用的套件管理器（Package Manager），是 `npm`、`yarn` 還是 `pnpm`
3. 找出服務用於安裝依賴項（Dependencies）的指令
4. 找出服務用於建置的指令
5. 找出服務用於啟動的指令
6. 找出服務使用的 Node.js 框架
