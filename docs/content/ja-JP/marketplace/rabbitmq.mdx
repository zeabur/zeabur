# ラビットMQのサービスデプロイ

Zeaburでは、1クリックでラビットMQメッセージキューをデプロイできるように、ラビットMQサービスを提供しています。

<Callout type="info" emoji="ℹ️">
    現在、RabbitMQサービスのバージョンは3-managementです。
</Callout>

## ラビットMQのサービスデプロイ

<CreateProject />

プロジェクトで、**新しいサービス**をクリックし、その後**プリビルト**ボタンをクリックします。ポップアップ検索バーで`RabbitMQ`を検索します。

![deploy](/deploy/common/select-service.en-US.png)

デプロイをクリックすると、Zeaburが自動的にラビットMQサービスをデプロイします。

## 環境変数

ラビットMQサービスをデプロイした後、Zeaburは自動的に関連する環境変数を他のサービスに注入します。

- `RABBITMQ_HOST`
- `RABBITMQ_PORT`
- `RABBITMQ_DEFAULT_USER`
- `RABBITMQ_DEFAULT_PASS`

上記の環境変数の代わりに、URIの宣言を簡素化するために、独自の`DATABASE_URL`変数を追加することもあります。例：

```bash
amqp://<RABBITMQ_DEFAULT_USER>:<RABBITMQ_DEFAULT_PASS>@<RABBITMQ_HOST>:<RABBITMQ_PORT>/<DATABASE_NAME>
```

ここで、`<DATABASE_NAME>`は追加したVhostの名前です。

## オンライン管理インターフェイス

ラビットMQサービスにはオンライン管理インターフェイスがあり、ブラウザでラビットMQサービスを管理できます。

ラビットMQサービスのオンライン管理インターフェイスにアクセスするには、`http://<RABBITMQ_HOST>:15672`を訪問します。

## DSN設定

ラビットMQサービスのDSN（つまり、ラビットMQサービスの接続文字列）を変数`${RABBITMQ_URI}`として使用できます。
