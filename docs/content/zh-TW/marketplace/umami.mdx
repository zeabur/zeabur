# 在 Zeabur 部署 Umami

[Umami](https://umami.is/) 是一個簡單的分析工具，可以幫助你了解你的網站訪客，同時也是最著名的 [Google Analytics](https://analytics.google.com/) 開源替代品。

這篇文章將教你如何在 Zeabur 上透過服務市集輕鬆部署一個自己的 Umami 服務，並且綁定自己的網域，讓你的其他網站能夠用自己部署的 Umami 服務來分析訪客流量。
## 從模板部署 Umami

由於 Umami 需要 PostgreSQL 作為資料庫，因此無法直接從鏡像部署。

我們已經為您製作了一個模板，可以一鍵部署帶有 PostgreSQL 的 Umami。

[![在 Zeabur 上部署](https://zeabur.com/button.svg)](https://zeabur.com/templates/01NQCC)

### 步驟一：打開 Umami 模板頁面

首先，訪問 [Umami 的模板頁面](https://zeabur.com/templates/01NQCC)。

你可以看到模板已經配置了 PostgreSQL 服務，因此你無需做任何額外的配置。

![模板頁面](/marketplace/umami/template.png)

### 步驟二：部署模板

點擊部署按鈕，然後選擇一個區域來部署你的服務。

![選擇區域](/marketplace/umami/region.png)

選擇後，你將自動重定向到項目頁面，在那裡你可以看到部署進度。

![部署進度](/marketplace/umami/deployment.png)

再等待幾秒鐘，你的 Umami 服務將成功部署並開始執行。

### 步驟三：綁定域名

部署完成後，你需要綁定一個域名以訪問你的 Umami 服務。

你可以按照[綁定域名的指南](/deploy/domain-binding)獲取更多詳細資訊。

### 步骤四：登入並設定 Umami

完成綁定後，你可以點選網域進入 Umami 的登入頁面，並使用預設的帳號 `admin` 及密碼 `umami` 來登入。

![Umami 登入頁面](/marketplace/umami/screenshot.png)

關於其他 Umami 的設定，請參閱 [Umami 官方文件](https://umami.is/docs)。
