# Desplegar el Servicio Halo

[Halo](https://www.halo.run/) es una poderosa y fácil de usar herramienta de creación de sitios web de código abierto. Con una rica variedad de plantillas y complementos, te ayuda a construir el sitio ideal que tienes en mente.

Este artículo te guiará sobre cómo desplegar fácilmente tu propio servicio Halo en Zeabur a través del mercado de servicios y vincular tu propio nombre de dominio.

## Paso 1: Abrir la Página de Plantillas de Halo

Dado que Halo requiere un servicio de base de datos independiente, no puedes completar la implementación solo con una imagen de Halo independiente.

Hemos creado una plantilla para ti que puede desplegar Halo con PostgreSQL con un solo clic.

[![Desplegar en Zeabur](https://zeabur.com/button.svg)](https://zeabur.com/templates/Q6H2MA)

## Paso 2: Desplegar la Plantilla

![Plantilla](/marketplace/halo/template-overview.en-US.png)

Haz clic en el botón de desplegar y elige una región para desplegar tu servicio.

![Seleccionar Región](/marketplace/halo/create-by-template.en-US.png)

Después de seleccionar, serás redirigido automáticamente a la página del proyecto donde podrás ver el progreso del despliegue.

Espera unos segundos y tu servicio Halo se desplegará con éxito y comenzará a funcionar.

![Estado del Servicio](/marketplace/halo/service-overview.en-US.png)

## Paso 3: Vincular el Nombre de Dominio

Una vez completado el despliegue, necesitas vincular un nombre de dominio para acceder a tu servicio Halo.

Puedes seguir la [guía para vincular dominios](/deploy/domain-binding) para obtener más detalles.

<Callout>
Después de vincular el nombre de dominio, la próxima vez que reinicies el servicio Halo, este dominio se configurará automáticamente como la dirección de acceso externo de Halo, sin necesidad de configurarlo manualmente de nuevo.
</Callout>

## Paso 4: Iniciar Sesión y Configurar Halo

Después de completar la vinculación del dominio, puedes hacer clic en el dominio para entrar en la interfaz de inicialización de Halo.

![Inicialización](/marketplace/halo/setup.en-US.png)

Una vez inicializado e iniciado sesión, podrás acceder a la consola de Halo.

![Consola](/marketplace/halo/console.en-US.png)

La documentación de uso de Halo se puede consultar en: [Documentación de Halo - Guía del Usuario](https://docs.halo.run/category/%E7%94%A8%E6%88%B7%E6%8C%87%E5%8D%97)

## Mantenimiento Posterior

### Actualizar la Versión

Para actualizar la versión de Halo en el futuro, solo necesitas cambiar el número de versión de la imagen en la configuración del servicio, como se muestra a continuación:

![Cambiar nombre de etiqueta](/marketplace/halo/change-tag-name.en-US.png)

La lista de versiones de Halo se puede ver en [halohub/halo](https://hub.docker.com/r/halohub/halo/tags), y las notas de la versión se pueden consultar en: [Sitio Oficial de Halo - Notas de la Versión](https://www.halo.run/categories/release-notes)

<Callout>
Después de cambiar la versión de la imagen, el servicio se reiniciará automáticamente, y la actualización se completará después de una breve espera.
</Callout>

### Copia de Seguridad y Migración

Aunque Zeabur también proporciona una función de copia de seguridad para los servicios, la función de copia de seguridad integrada de Halo es independiente de la plataforma, por lo que se recomienda utilizar la función de copia de seguridad integrada de Halo.

Además, si necesitas migrar tu servicio Halo existente a Zeabur, usar la función de copia de seguridad y restauración de Halo también es la mejor opción.

Para obtener instrucciones sobre cómo usar la función de copia de seguridad y restauración de Halo, consulta: [Documentación de Halo - Copia de Seguridad y Restauración](https://docs.halo.run/user-guide/backup)
