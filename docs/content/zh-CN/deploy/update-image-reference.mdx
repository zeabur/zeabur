# 更新 Docker 服务的映像引用

Zeabur 允许你在创建服务后更新映像的引用，适合用于更新 Docker 服务的映像版本，或切换服务到其他映像。

<Callout type="info">
    只有 **Docker 服务** 可以更新映像引用。Git 和 Once 服务由于没有映像引用，因此无法更改。
</Callout>

## 更新 tag

“Tag”（标签）是映像引用中指定映像版本的部分。例如，在 `nginx:latest` 映像引用中，`latest` 是 tag。你可以通过更新 tag 来切换到映像的不同版本。

首先展开服务的“设置”区域，你会看到“服务映像”部分。这里可以让你更新服务的映像引用。

![服务映像](/deploy/update-image-reference/service-image.webp)

点击并修改映像引用中的 tag 部分（即第二个输入框），你会看到允许你应用新 tag 的 **保存** 按钮。

![更新 tag](/deploy/update-image-reference/update-tag.webp)

保存更改后，Zeabur 会自动重新启动你的服务并应用新的映像引用。如果无法拉取映像，Zeabur 会提示你再次修改映像引用。

![拉取失败](/deploy/update-image-reference/pull-failed.webp)

## 更新 repository

“Repository”（映像库）是映像引用中指定映像名称的部分。例如，在映像引用 `nginx:latest` 中，`nginx` 是 repository 名称。你可以通过更新 repository 来切换到不同的映像。

打开服务的“设置”区域，你会看到“服务映像”部分。在这里，你可以更新服务的映像 repository。

![服务映像](/deploy/update-image-reference/service-image.webp)

点击并修改映像引用中的 repository 部分（即第一个输入框），你会看到允许你应用新 repository 的 **保存** 按钮。

![更新 repository](/deploy/update-image-reference/update-repository.webp)

保存更改后，Zeabur 会自动重新启动你的服务并应用新的映像引用。如果无法拉取映像，Zeabur 会提示你再次修改映像引用。

![拉取失败](/deploy/update-image-reference/pull-failed.webp)
