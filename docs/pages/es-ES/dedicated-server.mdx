# Servidor Dedicado

Puedes usar Zeabur para desplegar fácilmente varios servicios en dos tipos diferentes de servidores: Clúster Compartido (Shared Cluster) y Servidor Dedicado (Dedicated Server). Estos dos tipos de recursos de computación son adecuados para diferentes escenarios de uso.

## Clúster Compartido

El Clúster Compartido es la forma más sencilla de desplegar servicios usando Zeabur porque no necesitas entender ningún conocimiento o configuración relacionada con servidores, y ni siquiera percibirás el concepto de un servidor. Solo necesitas elegir la ubicación del centro de datos adecuada.

Cada Clúster Compartido es un clúster de computación compuesto por múltiples servidores gestionados por Zeabur. Un clúster contiene múltiples servidores, y los servicios que despliegues compartirán estos servidores con otros usuarios que elijan el mismo clúster.

El Clúster Compartido es adecuado para desplegar servicios con demandas de hardware fluctuantes porque utiliza un modelo de pago por uso. Puede ahorrar costos cuando la demanda de hardware es baja y escalar automáticamente cuando la demanda es alta, evitando así la escasez de recursos de hardware.

Actualmente, Zeabur tiene los siguientes Clústeres Compartidos:

- California, EE.UU. (proporcionado por Amazon Web Services)
- Frankfurt, Alemania (proporcionado por Amazon Web Services)
- Tokio, Japón (proporcionado por Amazon Web Services)
- Hong Kong (proporcionado por Amazon Web Services)
- Taipéi (proporcionado por Google Cloud) — disponible solo para clientes empresariales

## Servidor Dedicado

Un Servidor Dedicado es un servidor que te pertenece solo a ti, por lo que tus servicios no se verán afectados por los servicios de otros usuarios, y los recursos de hardware son completamente tuyos, sin ser ocupados por otros usuarios.

Los Servidores Dedicados son adecuados para los siguientes escenarios:

- Usuarios que han alquilado o comprado servidores de otros proveedores de la nube pueden registrar sus servidores en Zeabur con un solo clic, evitando así la necesidad de pagar por el uso de recursos de hardware.
- Usuarios con altos requisitos de estabilidad y seguridad del servicio, como entornos de producción para sistemas grandes.
- Usuarios cuya principal área de acceso al servicio no tiene una región de Clúster Compartido pueden alquilar o construir un Servidor Dedicado que se ajuste a su ubicación geográfica.

Puedes crear múltiples proyectos en un Servidor Dedicado, pero debes prestar atención al uso de los recursos de hardware de estos servicios para evitar que el impacto en el servicio debido a la insuficiencia de recursos de hardware en el Servidor Dedicado.

## Registrar un Servidor Dedicado

Puedes elegir comprar un servidor dedicado en Zeabur o registrar un servidor que ya hayas alquilado o comprado de otro proveedor de servicios en la nube.

Si deseas aprender cómo registrar tu propio servidor, puedes consultar [Registrar un Servidor Dedicado](/dedicated-server/register).

Si deseas comprar un servidor dedicado de Zeabur, puedes ir a "Mis Servidores" → "Crear" → "Comprar Servidor de Zeabur" para hacer tu selección. Después de elegir comprar, deduciremos el monto de tu saldo de Zeabur o tarjeta de crédito. El proceso de compra es el siguiente:

## Conexión vía SSH

Para los servidores dedicados comprados en Zeabur, puedes conectarte vía SSH siguiendo estos pasos:

### Paso 1: Revelar la Contraseña SSH

Navega a la página de información de tu servidor y haz clic en el campo de Contraseña SSH para revelarla.

![Revelar Contraseña SSH](/dedicated-server/ssh-step-1.png)

### Paso 2: Conectar Usando SSH

Una vez que hayas revelado la contraseña, usa la Dirección IP, el Nombre de Usuario SSH y la Contraseña SSH que se muestran en la página de información del servidor para conectarte vía SSH.

![Detalles de Conexión SSH](/dedicated-server/ssh-step-2.png)

Puedes conectarte usando una terminal con el siguiente comando:

```bash
ssh <nombre-de-usuario>@<dirección-ip>
```

Luego ingresa la contraseña cuando se te solicite.

<Callout type="warning">
  Una vez que cambies la contraseña SSH, Zeabur ya no podrá proporcionar soporte técnico para este servidor. Por favor, procede con precaución.
</Callout>
