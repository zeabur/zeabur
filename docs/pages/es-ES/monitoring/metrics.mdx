# Métricas

Zeabur proporciona métricas de uso de recursos en tiempo real para cada servicio, ayudándote a monitorear el rendimiento y tomar decisiones informadas de escalado.

## Ver Métricas

Para ver las métricas de un servicio:

1. Abre tu proyecto en el panel de Zeabur.
2. Selecciona el servicio que deseas monitorear.
3. Haz clic en la pestaña **Métricas**.

El panel de métricas muestra gráficos en tiempo real para:

- **Uso de CPU** — Consumo actual e histórico de CPU.
- **Uso de memoria** — Consumo actual e histórico de RAM, incluyendo el límite configurado.
- **Tráfico de red** — Transferencia de datos entrante y saliente.

## Entender el Uso de Recursos

### CPU

El uso de CPU se muestra como un porcentaje de los recursos de CPU asignados. Los picos en el uso de CPU pueden indicar:

- Alto volumen de solicitudes
- Operaciones computacionalmente costosas
- Rutas de código ineficientes que podrían beneficiarse de optimización

### Memoria

El uso de memoria muestra cuánta RAM está consumiendo tu servicio en relación con su límite configurado. Si tu servicio se acerca constantemente a su límite de memoria, considera aumentar la asignación en la [configuración del servicio](/operations/scaling) para evitar reinicios por falta de memoria (OOM).

### Red

Las métricas de red muestran el volumen de datos transferidos dentro y fuera de tu servicio. Esto es útil para entender patrones de tráfico y estimar costos de ancho de banda.

## Uso del Servicio y Facturación

Para un desglose detallado del consumo de recursos y los costos asociados, visita la página **Uso del Servicio** en la configuración de tu proyecto. Esta página muestra:

- Uso acumulado de CPU y memoria durante el período de facturación
- Cargos por tráfico de red saliente
- Desglose de costos por servicio

Consulta [Uso del Servicio](/operations/service-usage) para más detalles sobre cómo el uso se traduce en facturación.

<Callout>
  Zeabur utiliza un modelo de precios basado en uso. Monitorear tus métricas regularmente te ayuda a mantenerte al tanto del consumo de recursos y evitar cargos inesperados. Consulta [Precios](/pricing) para todos los detalles de precios.
</Callout>

## Usar Métricas para Guiar el Escalado

Las métricas son esenciales para decidir cuándo y cómo escalar:

- **CPU consistentemente por encima del 80%** — Considera aumentar la asignación de CPU u optimizar tu aplicación.
- **Memoria cerca del límite** — Aumenta la asignación de memoria para prevenir fallos por OOM.
- **Crecimiento constante del tráfico** — Planifica con anticipación ajustando los recursos antes de que los picos de uso causen un rendimiento degradado.

Para información sobre cómo ajustar los recursos del servicio, consulta [Escalado](/operations/scaling).
