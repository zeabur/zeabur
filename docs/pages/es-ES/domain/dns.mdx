# Registros DNS

Cada dominio registrado tiene un editor DNS en su página de detalles. Haz clic en cualquier dominio de la lista de gestión para acceder.

## Tipos de Registro Soportados

| Tipo | Descripción | Soporte de Proxy |
|------|-------------|-----------------|
| **A** | Mapea un dominio a una dirección IPv4 | Sí |
| **AAAA** | Mapea un dominio a una dirección IPv6 | Sí |
| **CNAME** | Alias a otro nombre de dominio | Sí |
| **MX** | Servidor de intercambio de correo | No |
| **TXT** | Texto arbitrario (SPF, DKIM, verificación, etc.) | No |
| **SRV** | Localizador de servicios | No |
| **CAA** | Autorización de autoridad de certificación | No |
| **NS** | Delegación de servidor de nombres | No |

## Agregar un Registro

<Steps>

### Abre el Editor DNS

Navega a la página de detalles del dominio y haz clic en **Agregar registro** sobre la tabla de registros DNS.

### Completa el Registro

- **Tipo**: Selecciona el tipo de registro del menú desplegable.
- **Nombre**: El subdominio o `@` para el dominio raíz.
- **Contenido**: El valor del registro (dirección IP, nombre de host, texto, etc.).
- **TTL**: Tiempo de vida. Selecciona entre valores preestablecidos (auto, 60s, 300s, 3600s, 86400s, etc.). Cuando el proxy está habilitado, el TTL se establece en auto.
- **Prioridad**: Requerido para registros MX y SRV.
- **Proxy**: Disponible solo para registros A, AAAA y CNAME. Ver [Proxy de Cloudflare](#proxy-de-cloudflare).

### Guardar

Haz clic en **Guardar** para crear el registro. Los cambios generalmente se propagan en pocos minutos.

</Steps>

## Editar y Eliminar Registros

- Haz clic en el **icono de edición** en la fila del registro para modificar su contenido, TTL, prioridad o configuración de proxy. El tipo y nombre del registro no se pueden cambiar después de la creación — elimina y recrea el registro si es necesario.
- Haz clic en el **icono de eliminación** para borrar un registro.

## Proxy de Cloudflare

Los registros A, AAAA y CNAME tienen un interruptor de proxy. Cuando está habilitado:

- El tráfico se enruta a través de Cloudflare, proporcionando protección DDoS, caché y terminación SSL.
- El TTL se establece automáticamente en **Auto** y no se puede cambiar manualmente.
- La dirección IP real del origen se oculta en las consultas DNS.

<Callout type="info">
  El proxy está deshabilitado por defecto. Habilítalo cuando necesites las funciones de protección y rendimiento de Cloudflare para ese registro.
</Callout>
