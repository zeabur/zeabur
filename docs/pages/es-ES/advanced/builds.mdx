# Builds

Zeabur utiliza [zbpack](https://github.com/zeabur/zbpack) como herramienta interna para construir los servicios de los usuarios, permitiendo desplegar servicios basados en cualquier lenguaje y framework con un solo clic, sin necesidad de entender los detalles complejos.

Actualmente, [zbpack](https://github.com/zeabur/zbpack) soporta todos los lenguajes de programación principales para desarrollo web y ha optimizado aún más la detección y el rendimiento para los frameworks más populares. Al mismo tiempo, se actualiza constantemente para nuevos lenguajes y frameworks:

- Node.js
- Python
- PHP
- Ruby
- Go
- Java
- .NET
- Rust
- Elixir

Si el lenguaje o framework que utiliza tu servicio no está en la lista anterior, o si encuentras algún problema con zbpack durante el proceso de construcción, por favor contribuye con un Pull Request en [GitHub](https://github.com/zeabur/zbpack/pulls) para ayudarnos a mejorar la funcionalidad de zbpack.

Cuando utilizas zbpack para construir un servicio en un proyecto, identifica automáticamente el lenguaje y el framework basándose en el código, archivos de configuración y otra información del proyecto, y selecciona la forma adecuada de construir el servicio.

## Generar el Plan de Construcción

Por ejemplo, supongamos que has desarrollado una aplicación web basada en [Next.js](https://nextjs.org/):

```bash
git clone https://github.com/zeabur/nextjs-template
cd nextjs-template
```

Al ejecutar el comando `zbpack` en la carpeta del proyecto, puedes ver el plan de construcción personalizado para este caso:

```bash
zbpack .

╔══════════════════════════════ Build Plan ═════════════════════════════╗
║ provider         │ nodejs                                             ║
║───────────────────────────────────────────────────────────────────────║
║ nodeVersion      │ 18                                                 ║
║───────────────────────────────────────────────────────────────────────║
║ installCmd       │ pnpm install                                       ║
║───────────────────────────────────────────────────────────────────────║
║ buildCmd         │ pnpm run build                                     ║
║───────────────────────────────────────────────────────────────────────║
║ startCmd         │ pnpm start                                         ║
║───────────────────────────────────────────────────────────────────────║
║ packageManager   │ pnpm                                               ║
║───────────────────────────────────────────────────────────────────────║
║ framework        │ next.js                                            ║
╚═══════════════════════════════════════════════════════════════════════╝
```

En la imagen podemos ver que `zbpack` elige usar el **Provider** `nodejs` como solución de construcción, lo que implica que realizará los siguientes pasos en el plan:

1. Detectar qué versión de Node.js utiliza el servicio
2. Identificar qué gestor de paquetes usa el servicio (`npm`, `yarn` o `pnpm`)
3. Determinar qué comando se usa para instalar dependencias
4. Determinar qué comando se usa para construir
5. Determinar qué comando se usa para iniciar
6. Identificar qué framework de Node.js está utilizando el servicio
