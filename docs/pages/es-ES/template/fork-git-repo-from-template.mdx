# Bifurcar Código desde Servicios de Plantilla

En Zeabur puedes usar servicios de plantilla para desplegar aplicaciones rápidamente. Una plantilla incluye múltiples servicios, cada uno siendo un servicio Prebuilt (desplegado desde una imagen Docker) o un servicio Git (desplegado desde GitHub).

Por defecto, al desplegar una plantilla con un servicio Git, Zeabur despliega directamente el servicio desde el repositorio GitHub de origen de la plantilla en tu proyecto y vincula automáticamente CI/CD al repositorio establecido por el autor de la plantilla. Esto significa que cuando el autor de la plantilla sube nuevos commits, Zeabur activa automáticamente CI/CD y despliega la nueva versión en tu proyecto.

Sin embargo, puede haber ocasiones en las que quieras modificar el código del autor de la plantilla, agregar tus propias funciones o cambiar configuraciones, etc. Para este escenario, Zeabur ofrece una conveniente función de Fork con un solo clic.

## Cómo Bifurcar Código desde un Servicio de Plantilla

Tomando como ejemplo la [plantilla oficial de Express.js de Zeabur](https://zeabur.com/templates/W96I3Z), después de desplegar esta plantilla, haz clic en la página de configuración del servicio express. Verás que el campo Git Repository tiene actualmente el valor `https://github.com/zeabur/expressjs-template`, que es el repositorio GitHub especificado por esta plantilla.

Si queremos modificar el código basado en esta plantilla, puedes hacer clic en el botón Fork Repository que aparece debajo:

![Express.js github fork](/template/fork-git-repo-from-template/Expressjs_github_fork.png)

Después de hacer clic, aparecerá una ventana preguntando el nombre del repositorio. Elige un nombre de repositorio que no haya sido usado antes, de lo contrario, el Fork podría fallar. También hay un cuadro de entrada debajo del nombre que te permite elegir si configurar el repositorio como privado, lo que evita que otros vean tus cambios de código:

![git fork](/template/fork-git-repo-from-template/git_fork.png)

Después de elegir un nombre, haz clic en Fork. Si aún no has autorizado GitHub, es posible que te redirijan a la página de autorización OAuth de GitHub. Una vez completada la autorización, verás la pantalla de carga del Fork:

![Configure Github](/template/fork-git-repo-from-template/Configure_Github.png)

![github connection](/template/fork-git-repo-from-template/github_connection.png)

Al finalizar, verás que se ha añadido un nuevo repositorio a tu cuenta de GitHub, con el mismo código que el original.

Zeabur automáticamente bifurcará el código de esta plantilla a tu cuenta de GitHub y establecerá este repositorio bifurcado como el Git Repository de tu servicio. Esto significa que cuando modifiques este repositorio bifurcado en GitHub y subas commits, Zeabur activará automáticamente CI/CD y desplegará la nueva versión en tu proyecto.
