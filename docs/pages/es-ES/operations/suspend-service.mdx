# Suspender Servicio

Zeabur te permite suspender (pausar) servicios individuales directamente desde el panel. Los servicios suspendidos dejan de ejecutarse, dejan de consumir recursos y dejan de generar cargos — mientras conservan toda su configuración para cuando estés listo para reanudar.

## Suspender un Servicio

Para suspender un servicio:

1. Abre tu servicio en el panel de Zeabur.
2. Ve a **Configuración**.
3. Haz clic en **Suspender Servicio**.

El servicio se detendrá inmediatamente. Mientras esté suspendido:

- El contenedor del servicio se apaga y ya no se ejecuta.
- **No se generan cargos de cómputo** por el servicio.
- Toda la configuración se conserva, incluyendo variables de entorno, enlaces de dominio y montajes de volumen.
- Los datos en [volúmenes](/operations/volumes) se conservan.

## Reanudar un Servicio

Para reanudar un servicio suspendido:

1. Abre el servicio suspendido en el panel.
2. Ve a **Resumen**.
3. Haz clic en **Reiniciar Servicio**.

Zeabur iniciará un nuevo despliegue usando la configuración más reciente. El servicio pasa por el proceso normal de [verificación de salud](/operations/health-checks) y comienza a aceptar tráfico una vez listo.

## Casos de Uso

### Mantenimiento Temporal

Suspende un servicio cuando necesites realizar mantenimiento en la infraestructura dependiente (como una migración de base de datos) y quieras asegurarte de que ningún tráfico llegue a la aplicación durante ese período.

### Ahorro de Costos

Si tienes servicios que solo se necesitan durante ciertas horas o para tareas específicas (como un entorno de staging o un servicio de procesamiento por lotes), suspenderlos cuando no estén en uso puede reducir costos.

### Depuración

Suspende un servicio con mal comportamiento para detener el procesamiento de solicitudes mientras investigas. Una vez que hayas identificado y corregido el problema, vuelve a desplegar y reanuda.

## Suspensión vs. Eliminación

| | Suspender | Eliminar |
| --- | --- | --- |
| El servicio deja de ejecutarse | Sí | Sí |
| Configuración conservada | Sí | No (después del período de gracia) |
| Volúmenes conservados | Sí | No (después del período de gracia) |
| Se puede revertir | Sí, instantáneamente | Solo dentro del período de gracia de 2 horas |
| Facturación | Se detiene | Se detiene |

<Callout type="warning">
  Eliminar un servicio es una operación destructiva. Zeabur proporciona un **período de gracia de 2 horas** después de la eliminación durante el cual puedes contactar al soporte para recuperar el servicio. Después del período de gracia, todos los datos — incluyendo volúmenes — se eliminan permanentemente. Si solo necesitas detener temporalmente un servicio, usa **Suspender** en su lugar.
</Callout>

## Servicios Suspendidos y Dominios

Mientras un servicio está suspendido, cualquier dominio vinculado mostrará una página de error. Si necesitas mostrar una página de mantenimiento personalizada, considera usar un servicio ligero separado (como un sitio estático) para servir un aviso de mantenimiento, y actualiza el DNS de tu dominio para apuntar a ese servicio durante el período de mantenimiento.
