# Escalado

Zeabur te permite ajustar los recursos asignados a tus servicios para que puedan manejar cargas de trabajo variables. Puedes escalar verticalmente aumentando la CPU y la memoria, y elegir el nivel de infraestructura adecuado para tus necesidades.

## Escalado Vertical

El escalado vertical significa aumentar (o disminuir) la CPU y la memoria asignada a una única instancia de servicio.

Para ajustar recursos:

1. Abre tu servicio en el panel de Zeabur.
2. Ve a **Configuración**.
3. En la sección **Recursos**, configura los límites deseados de CPU y memoria.
4. Guarda los cambios. La nueva asignación de recursos entrará en efecto en el próximo despliegue.

### Planes de Recursos

Zeabur ofrece varios niveles de recursos. La asignación predeterminada es adecuada para la mayoría de los servicios ligeros, pero puedes aumentar los recursos para cargas de trabajo más exigentes como bases de datos, servidores de compilación o inferencia de IA.

<Callout>
  La asignación de recursos afecta directamente la facturación. Límites más altos de CPU y memoria resultan en costos más altos. Consulta [Precios](/pricing) para obtener detalles sobre los precios por unidad.
</Callout>

## Servidores

Para cargas de trabajo que requieren recursos garantizados, rendimiento predecible o cumplimiento con requisitos de residencia de datos, Zeabur ofrece **Servidores**. Un servidor te proporciona un entorno de cómputo aislado que no se comparte con otros usuarios.

Los beneficios de los servidores incluyen:

- **Recursos garantizados** — Sin problemas de vecinos ruidosos.
- **Regiones personalizadas** — Despliega en ubicaciones geográficas específicas.
- **Límites de recursos más altos** — Ejecuta servicios que exceden los límites del clúster compartido.

Consulta [Servidores](/dedicated-server) para instrucciones de configuración y configuraciones disponibles.

## Estrategias de Escalado

| Escenario | Recomendación |
| --- | --- |
| Picos de tráfico ocasionales | Aumentar CPU/memoria para manejar la carga máxima |
| Tráfico alto constante | Usar un servidor para rendimiento predecible |
| Optimización de costos | Dimensionar recursos correctamente basándose en [métricas](/operations/metrics) |
| Inactividad temporal | [Suspender](/operations/suspend-service) el servicio para detener la facturación |

## Monitoreo del Uso de Recursos

Usa el panel de [Métricas](/operations/metrics) para monitorear el uso de CPU, memoria y red en tiempo real. Estos datos te ayudan a determinar cuándo escalar hacia arriba o hacia abajo. Para el análisis de costos, consulta la página de [Uso del Servicio](/operations/service-usage).
