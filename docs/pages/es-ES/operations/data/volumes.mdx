# Volúmenes

En Zeabur, puedes montar Volúmenes para almacenar de forma persistente tus datos dentro de un servicio. Normalmente, los servicios en Zeabur restablecen los datos a los valores predeterminados al reiniciar (esto se conoce como <abbr title="Sin estado">stateless</abbr>), pero si necesitas persistir datos (haciendo un servicio <abbr title="Con estado">stateful</abbr>), puedes montar el directorio que necesita almacenamiento persistente en Volúmenes. De esta manera, incluso si el servicio se reinicia, los datos no se perderán.

Para información de precios sobre Volúmenes, consulta [nuestra página de precios](/pricing/pricing-plans).

<Callout type="info">
    Si solo deseas subir archivos de configuración, usar el [Editor de Configuración](/data-management/config-edit) es más conveniente y no genera costos adicionales.
</Callout>

## Aviso

- Por defecto, los servicios de Zeabur admiten reinicios sin tiempo de inactividad, lo que significa que cuando reinicias un servicio, el tráfico se cambia sin problemas a la nueva instancia y los usuarios no experimentarán ninguna interrupción.
- Sin embargo, una vez que los Volúmenes están habilitados, el servicio **no puede admitir reinicios sin tiempo de inactividad**. Cada reinicio primero apagará completamente el servicio antes de iniciarlo de nuevo, resultando en un breve período de inactividad.

## Montar Volúmenes

Abre la página del servicio donde deseas montar Volúmenes, haz clic en la pestaña "Volúmenes", y verás un botón "Montar Volúmenes". Después de hacer clic, verás dos campos:

![montar volúmenes](/data-management/volumes/mount-volumes.webp)

- **ID del Volumen** es un identificador único para este Volumen. Puedes usar un nombre simple (como `data`, `cache`, `photos`) para representar fácilmente este Volumen. Este identificador no está relacionado con el resultado real del montaje.
- **Directorio de Montaje** es el directorio donde se montará el Volumen, esencialmente la "carpeta que necesita almacenamiento persistente." Si deseas almacenar de forma persistente el directorio `/data` en el servicio, ingresa `/data` aquí.

**Ten en cuenta que después del montaje, los datos en este directorio se borrarán completamente**. Si ya tienes datos importantes dentro, descárgalos antes de montar y reiniciar el servicio para evitar pérdidas, y luego vuelve a importar los datos después del montaje.

## Desmontar Volúmenes

Si deseas dejar de pagar por un Volumen en particular, haz clic en el botón "Eliminar" junto al Volumen. Eliminarlo detendrá la facturación de este espacio. Ten en cuenta que después de la eliminación, los datos en este directorio serán **completamente borrados** y **no se pueden recuperar**.

![desmontar volúmenes](/data-management/volumes/unmount-volumes.webp)

## Usar Volúmenes en YAML de Plantilla

Puedes usar `volumes` en la "Especificación del Servicio" para montar Volúmenes. Para más información, consulta el artículo [Crear Plantillas desde YAML](/template/template-in-code).
