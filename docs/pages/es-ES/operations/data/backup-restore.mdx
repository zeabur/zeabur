# Respaldo y Restauración

Zeabur proporciona funcionalidad de <abbr title="Respaldo almacenado separadamente de los datos de tu host">respaldo externo</abbr> para el Plan Dev y superiores. Tus respaldos se almacenarán de forma segura y temporal en nuestro espacio de almacenamiento Amazon S3, que no ocupará espacio en tu host existente y no estará sujeto a cargos adicionales.

Para servicios de base de datos, Zeabur admite respaldos en línea y automáticos, permitiendo que tu base de datos se respalde sin tiempo de inactividad. Para otros servicios <abbr title="Con estado, servicios con discos duros adjuntos">con estado</abbr>, también proporcionamos funcionalidad de respaldo externo fuera de línea después de pausar.

## Respaldar tu Servicio

### Respaldo Externo Fuera de Línea

<Callout>
    Si no has montado [volúmenes](/operations/volumes), la función de respaldo no se mostrará. "Respaldo" solo respaldará los datos dentro de los volúmenes; otros archivos del servicio (como el código fuente) no se respaldarán.
</Callout>

Primero, necesitas suspender el servicio. Haz clic en "Configuración" → "Suspender Servicio" para suspenderlo. Ten en cuenta que la suspensión no causará pérdida de datos.

Luego, haz clic en la pestaña "Respaldo" de tu servicio, y verás el botón "Respaldo".

![pestaña de respaldo](/data-management/backup/backup-tab.webp)

Después de hacer clic, Zeabur respaldará automáticamente tu servicio en segundo plano, y puedes esperar unos minutos. Una vez que el respaldo esté completo, verás un botón de descarga, que puedes hacer clic para descargar el archivo de respaldo. El contenido del archivo de respaldo es el contenido completo de la carpeta donde están montados los volúmenes de tu servicio.

![respaldo exitoso](/data-management/backup/backup-success.webp)

Después de que el respaldo esté completo, puedes ir a "Resumen" y hacer clic en "Reiniciar Servicio" para reanudar tu servicio.

### Respaldo Externo en Línea

Zeabur admite respaldo externo en línea para bases de datos como PostgreSQL, MySQL, MariaDB, MongoDB, etc., permitiendo respaldos seguros sin pausar tu servicio. Además, el servicio de "Respaldo Externo en Línea" también admite respaldo automático.

![respaldo en línea](/data-management/backup/online-backup.webp)

Similar al "Respaldo Externo Fuera de Línea", después de hacer clic en el botón "Respaldo", Zeabur respaldará automáticamente en segundo plano, y puedes esperar unos minutos. Una vez que el respaldo esté completo, también verás un botón de descarga, que puedes hacer clic para descargar el archivo de respaldo.

El contenido del archivo de respaldo será un archivo de volcado estándar de la base de datos (para PostgreSQL, MySQL, es un archivo `.sql`; para MongoDB, son varios archivos `.bson`). Estos archivos de respaldo pueden restaurarse posteriormente usando herramientas integradas de restauración de base de datos (como `psql` y `mongoimport`).

### Respaldo Automático

Zeabur también admite respaldos automáticos, actualmente ofreciendo respaldos diarios a una hora establecida.

![respaldo automático](/data-management/backup/auto-backup.webp)

Después de habilitar "Respaldo Automático", puedes establecer la hora del respaldo en UTC. Es mejor elegir una hora cuando tu servicio experimenta menos tráfico. Puedes consultar la indicación "A tu hora local" junto a ella para ver a qué corresponde ese período UTC en tu hora local.

## Período de Retención de Respaldos y Método de Recuperación por API

Zeabur **solo retiene tus respaldos por 7 días**. Los respaldos que excedan este período se eliminarán automáticamente. Si necesitas almacenar respaldos por un período más largo, descarga los archivos de respaldo tú mismo y almacénalos en otro lugar.

Actualmente, Zeabur aún no admite respaldos a tu propio espacio de almacenamiento Amazon S3, pero puedes escribir un programa para recuperar respaldos llamando a la [API Pública de Zeabur](/operations/developer-tools/public-api) y almacenarlos en tu propio espacio de almacenamiento de tu elección:

```graphql
query Backups($environmentId: ObjectID!, $serviceId: ObjectID!) {
  backups(environmentID: $environmentId, serviceID: $serviceId) {
    createdAt
    downloadURL
    status
  }
}
```

Respuesta de ejemplo:

```json
{
  "data": {
    "backups": [
      {
        "createdAt": "2025-06-23T16:00:10.864Z",
        "downloadURL": "https://backup.zeabur.com/XXXXXX",
        "status": "SUCCESS"
      }
    ]
  }
}
```

Puedes usar el enlace de descarga del archivo de respaldo (`downloadURL`) junto con la hora del respaldo (`createdAt`) para almacenar los archivos de respaldo.

## Restaurar Respaldo

Esta sección describe brevemente varios métodos para restaurar el respaldo de tu servicio.

### Base de Datos

#### PostgreSQL

Consulta la sección "Restaurar Respaldo" en la [Página de Plantilla del Servicio PostgreSQL](https://zeabur.com/templates/B20CX0).

#### MySQL y MariaDB

Consulta la sección "Restaurar Respaldo" en la [Página de Plantilla del Servicio MySQL](https://zeabur.com/templates/DGLGRG) y la [Página de Plantilla del Servicio MariaDB](https://zeabur.com/zh-TW/templates/46YKAR).

#### Redis

Puedes usar el [comando `RESTORE` de Redis](https://redis.io/commands/restore) para restaurar un respaldo.

Expande la cadena "Comando para conectar a tu Redis" del servicio Redis, y verás algo como esto:

```bash
redis-cli -h <host> -p <port> -a <password>
```

Inserta el `<host>`, `<port>` y `<password>` en el siguiente comando:

```bash
cat /path/to/backup.rdb | redis-cli -h <host> -p <port> -a <password> -x RESTORE <key> 0
```

#### MongoDB

Expande la cadena "Comando para conectar a tu MongoDB" del servicio MongoDB, y verás algo como esto:

```bash
mongosh "<connection string>"
```

Copia la `<connection string>` en la terminal y ejecuta el comando `mongorestore` para restaurar el respaldo:

```bash
mongorestore "<connection string>" "<backup directory>"
```

### Volúmenes

Si deseas restaurar datos dentro de [volúmenes](/operations/volumes), puedes consultar el método de [Gestión de Archivos](/operations/file-management) para subir los archivos de respaldo al directorio de destino y descomprimirlos.
