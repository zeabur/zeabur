# Configuración de Variables de Entorno

Las variables de entorno se utilizan para configurar tu aplicación. Zeabur ofrece una forma de establecer variables de entorno para tu servicio.

## Añadir, Editar, Eliminar

Puedes configurar variables de entorno en la pestaña **Variables** de tu servicio, y puedes **Añadir Variables** a tu servicio.

![environment variables](/environment/add-variables.png)

Zeabur inyectará automáticamente estas variables de entorno en tu servicio.

- `<YOUR_SERVICE_NAME>_HOST`: El host del servicio.
- `PORT`: El puerto del servicio.
- Otras variables como credenciales de base de datos ([PostgreSQL](https://zeabur.com/templates/B20CX0)) o cadena de conexión de [Redis](https://zeabur.com/templates/KQZHXT), etc.

No necesitas configurar estas variables manualmente.

![db env](/environment/env_var.png)

Al pasar el cursor sobre una variable, puedes elegir editarla o eliminarla.

![hovering](/environment/env_var_hover.png)

## Editar como Raw

Si deseas añadir múltiples variables a la vez, usa la función de edición por lotes.

Haz clic en el botón **Edit as Raw** para entrar en el modo de edición por lotes, que configura las variables de entorno del servicio como un archivo `.env`.

![edit-raw](/environment/edit_raw_env_var.png)

## Referencia de Variables

Zeabur permite referenciar otras variables usando una sintaxis de plantilla simple:

1. Usa la sintaxis de plantilla para referenciar directamente otras variables.
2. Al establecer una nueva variable, rodea el nombre de la variable existente con `${}`.

**Ejemplo:**

Si tienes dos variables, `USERNAME` y `PASSWORD`, puedes combinar sus valores en una nueva variable `AUTH_INFOR`:

```bash
AUTH_INFOR=${USERNAME}:${PASSWORD}
```

![reference](/environment/var-reference.png)

De esta forma, los valores de `USERNAME` y `PASSWORD` se combinarán automáticamente e inyectarán en `AUTH_INFOR`.

## Variables Especiales

Las "Variables Especiales" son variables de entorno predefinidas por Zeabur que puedes usar directamente.

### Referenciación

- En el cuadro de entrada de variables de entorno en Zeabur: Usa el formato `${VARIABLE}` para insertar variables especiales.
- En tu aplicación: Usa el mismo método que para variables de entorno normales (`process.env.VARIABLE`).

### Orden de Prioridad de Variables

El orden de expansión de variables dentro de un servicio es el siguiente:

- Variables de entorno definidas por el servicio actual se obtienen primero.
- Si no están definidas, se obtienen variables expuestas por otros servicios.
- Si no están definidas, se obtienen variables especiales.

### Variables

#### Hostname/Puerto

- `${ZEABUR_[PORTNAME]_URL}`: La URL correspondiente a un puerto específico. Ejemplo: `https://test.zeabur.app`.  
  Esta es la URL que configuras en la sección "Domain". Para servicios de Git, `PORTNAME` siempre es `web`.

- `${ZEABUR_[PORTNAME]_DOMAIN}`: El dominio correspondiente a un puerto específico. Ejemplo: `test.zeabur.app`.  
  Similar al anterior.

- `${CONTAINER_HOSTNAME}`: El hostname del servicio actual en el proyecto Zeabur.  
  Útil para exponer como `${MYSQL_HOST}` = `${CONTAINER_HOSTNAME}`.

- `${[PORTNAME]_PORT}`: El puerto definido en el servicio Prebuilt. Ejemplo: `DATABASE_PORT` = `3306`. Solo visible en el servicio actual. Para Git, es `8080` (usa `${PORT}`).

#### Información de Zeabur

- `${ZEABUR_SERVICE_ID}`: ID del servicio
- `${ZEABUR_PROJECT_ID}`: ID del proyecto
- `${ZEABUR_ENVIRONMENT_ID}`: ID del entorno del servicio
- `${ZEABUR_USER_ID}`: ID del usuario que creó el proyecto

#### Información de Git

Solo durante la fase de build de servicios Git:

- `${ZEABUR_GIT_COMMIT_SHA}`: SHA del commit
- `${ZEABUR_GIT_AUTHOR}`: Autor del commit
- `${ZEABUR_GIT_BRANCH}`: Nombre de la rama
- `${ZEABUR_GIT_REPO_NAME}`: Nombre del repositorio
- `${ZEABUR_GIT_REPO_OWNER}`: Propietario del repositorio
- `${ZEABUR_GIT_COMMIT_MESSAGE}`: Mensaje del commit

#### Utilidades

- `${PASSWORD}`: Contraseña aleatoria generada por Zeabur para cada servicio.
