# Ejecutar Comandos

Puedes ejecutar comandos dentro de los servicios de Zeabur para instalar paquetes, ejecutar programas, ver información o incluso hacer copias de seguridad de datos específicos.

## Abrir la interfaz de ejecución de comandos

Abre la pestaña **Overview** del servicio y haz clic en **Command** para mostrar la interfaz “Execute Command”.

![Abrir la interfaz de ejecución de comandos](/deploy/command-execution/Command_part1.png)

La interfaz de ejecución de comandos incluye:

- Área de salida de comandos: Muestra los resultados de la ejecución. Si el comando se ejecuta correctamente, se muestra la salida; de lo contrario, aparece un código de error.

![Interfaz de ejecución de comandos](/deploy/command-execution/terminal.png)

## Ejecutar comandos

Como ejemplo básico, ingresa `echo Hello, World!` y deberías ver la salida `Hello, World!`.

![Ejecutar comando hello world](/deploy/command-execution/terminal_2.png)

Si ingresas un comando inexistente, se mostrará un código de error.

![Fallo en la ejecución de comando](/deploy/command-execution/terminal_3.png)

También puedes ejecutar sentencias SQL en contenedores de bases de datos como MySQL o PostgreSQL, usar cURL para subir logs o respaldar archivos, o incluso ejecutar programas de migración (por ejemplo, `prisma migrate`, `node seeder.js`).

Para MySQL, puedes copiar el comando “Command to connect to your MySQL”, eliminar los parámetros `host` y `port` (no se necesitan para conexiones remotas dentro del contenedor), cambiar `mysqlsh` por `mysql` (ya que `mysqlsh` no está disponible en el contenedor), y finalmente añadir `-e` para ejecutar la sentencia SQL.

```bash
mysql --user=root --password=<password> --database=zeabur -e "SHOW DATABASES;"
