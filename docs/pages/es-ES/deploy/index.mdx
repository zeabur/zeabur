# Despliegue y Servicios

Esta sección cubre todos los métodos de despliegue de Zeabur, la configuración de servicios y las funciones de gestión de proyectos. Cada encabezado responde a una pregunta específica para referencia rápida.

## ¿Dónde Comienza el Despliegue?

Todos los servicios en Zeabur viven dentro de un "Proyecto". Un solo proyecto puede contener tu frontend, backend, bases de datos y más.

- [Crear Proyecto](/deploy/create-project) — Crear y gestionar proyectos
- [Crear Servicio](/deploy/create-service) — Agregar servicios a un proyecto
- [Cómo Funcionan los Despliegues](/deploy/how-deploys-work) — Aprende cómo Zeabur auto-detecta lenguajes y compila

## ¿Qué Activa un Redespliegue?

- **Push a GitHub**: Por defecto, cada push a la rama vinculada activa un nuevo despliegue.
- **Activación Manual**: Haz clic en "Redesplegar" en el Panel.
- **Rutas de Vigilancia**: Configura despliegues para activarse solo cuando cambian rutas específicas. Ver [Rutas de Vigilancia](/deploy/watch-paths).

## ¿Cómo Configuro Variables de Entorno?

En la pestaña "Configuración" del servicio, puedes agregar, editar y eliminar variables de entorno. Zeabur también soporta la referencia de variables entre servicios.

> [Variables de Entorno](/deploy/environment-variables)
