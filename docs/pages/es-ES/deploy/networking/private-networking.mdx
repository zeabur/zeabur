# Red Privada

La Red Privada permite a los servicios usar un nombre de host fijo para conectarse a otro servicio en el mismo proyecto.

## Encontrar el Nombre de Host

Abre la sección "Red" del servicio para encontrar el nombre de host bajo "Privado".

![encontrar nombre de host](/networking/private/find-hostname.webp)

## Conectarse al Servicio

Usa el nombre de host anterior para conectarte a este servicio (en este ejemplo, PostgreSQL). Además, según [la documentación de PostgreSQL](/marketplace/postgresql), el puerto se inyecta en la variable de entorno `POSTGRES_PORT`. Entonces, puedes completar tu información de conexión de la siguiente manera:

- Nombre de host: El nombre de host que encontraste en el paso anterior, en este ejemplo es `postgresql.zeabur.internal`.
- Puerto: Obtén el valor de la variable de entorno `POSTGRES_PORT`, que [por defecto es 5432](https://github.com/zeabur/zeabur/blob/bd565389cf1868a917333d0f74003b4aad6ef0ce/prebuilts/postgresql.toml#L14).

## Notas Importantes

- Cambiar el nombre del servicio **no** cambia el nombre de host.
