# **Watch Paths**

Después de desplegar tu servicio, es común actualizar el código con frecuencia, pero algunas actualizaciones no requieren redeplegar el servicio.

Por ejemplo, en un proyecto donde el frontend y el backend están en el mismo repositorio Git, podrías haber actualizado solo el código del backend, pero al hacer commit, descubres que el servicio del frontend también se redepliega.

Para evitar esto, ve a la configuración y modifica las watch paths:

### **Paso 1: Abre la página de configuración del servicio**

Haz clic en el servicio cuya configuración deseas modificar, expande la página **Settings** y desplázate hacia abajo para encontrar el ítem **Watch Paths**, donde puedes modificar las watch paths del servicio.

![Watch path settings](/deploy/watch-path/watch_path_settings.png)

### **Paso 2: Modifica las watch paths**

Verás que la watch path por defecto es `*`, lo que significa que el servicio monitorea todo el repositorio y cualquier actualización de archivo desencadena un redepliegue.

Puedes modificar las watch paths usando el mismo formato que un archivo `.gitignore`.

La única diferencia es que `.gitignore` indica qué ignorar, mientras que aquí indica qué activar (trigger).

Por ejemplo, has desplegado una aplicación full-stack con la siguiente estructura de código:

![Repository structure](/deploy/watch-path/repo.png)

Y has desplegado dos servicios para este repositorio: uno con el root directory en `client` y otro en `server`. Esto solo especifica la ruta de build. Para aislar mejor, configura las **Watch paths**.

Por ejemplo, al establecer las **Watch paths** del servicio client en `/client`, las actualizaciones en la carpeta `server` ya no activarán automáticamente el redepliegue del servicio client.

![Watch paths saved](/deploy/watch-path/watch_paths_save.png)
