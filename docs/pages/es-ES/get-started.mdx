# Primeros Pasos

## Paso Uno: Crea una Cuenta

Necesitas crear una cuenta para empezar a implementar tus servicios en Zeabur.

Si eres nuevo en Zeabur, deberás registrarte usando tu correo electrónico (la primera opción). Introduce tu dirección de correo electrónico aquí y luego haz clic en "Continuar con el correo electrónico".

![Login](/start/login.webp)

Luego, recibirás un mensaje que dice "Enlace de inicio de sesión enviado".

![Verification mail sent](/start/verification-mail-sent.webp)

Abre tu bandeja de entrada de correo electrónico (por ejemplo, Gmail, Outlook, Apple Mail) y verás un correo electrónico de Zeabur. Haz clic en el botón "Iniciar sesión" dentro del correo electrónico.

![Verification mail](/start/verification-mail.webp)

<Callout>
  Si el navegador desde el que inicias sesión no es el navegador predeterminado de tu sistema (es decir, si al abrir el enlace se inicia Safari en lugar de Chrome),
  puedes hacer clic derecho en el botón "Iniciar sesión" para copiar el enlace y luego pegarlo en el navegador que desees usar.
</Callout>

Una vez que aparezca esta interfaz de inicio de sesión, ¡significa que has iniciado sesión con éxito!

![Onboarding UI](/start/onboarding-ui.webp)

¡Recuerda que puedes cambiar el idioma de la interfaz en la esquina superior derecha! Actualmente admitimos chino tradicional, chino simplificado, inglés, japonés, español e indonesio.

![Switch language](/start/switch-language.webp)

## Paso Dos: Crea un Proyecto

En Zeabur, un Proyecto sirve como un único contenedor para todos los servicios relacionados que componen una aplicación real. Por ejemplo, si tu aplicación consta de un sitio web frontend, una API backend y una base de datos, todos deben mantenerse dentro del mismo Proyecto.

![Project structure](/start/project-structure.webp)

Para comenzar a crear un proyecto, haz clic en el botón **Crear proyecto** en el centro (o en la esquina superior derecha).

![Create new project](/start/create-new-project.webp)

Si es la primera vez que creas un proyecto, es probable que veas una ventana de "Verificación de cuenta".

![Account verification](/start/account-verification.webp)

Puedes elegir:

1. Verificar usando un número de teléfono. La mayoría de los países y regiones admiten este método. Si tu país o región no lo admite, o si necesitas vincular varias cuentas de Zeabur, además de [contactarnos](https://zeabur.com/dc), también puedes consultar las opciones segunda y tercera.
2. Realizar un prepago de créditos de uso. Siempre y cuando no te hayas suscrito a un Plan Dev o a un servidor, y no estés involucrado en un abuso grave (por ejemplo, violar los Términos de Servicio iniciando ataques DoS de gran volumen), no deduciremos de tu saldo.
3. Vincular una tarjeta de crédito. Similar al segundo punto, no iniciaremos ningún cargo a menos que realices operaciones de suscripción; es puramente para fines de verificación.

Elige tu método preferido y luego haz clic en el botón "Verificar" en la sección correspondiente.

Una vez completada la verificación, puedes volver a crear el proyecto y ver el proyecto creado con éxito.

![Project page with add service](/start/project-page-with-add-service.webp)

## Paso Tres: Implementar Servicios

Zeabur puede implementar una amplia variedad de servicios, incluyendo bases de datos (PostgreSQL, MySQL, Redis), proyectos de código abierto (n8n, WordPress) o tus propias aplicaciones personalizadas (importadas desde GitHub o subidas desde una carpeta).

![Project page with add service](/start/project-page-with-add-service.webp)

- Si quieres implementar una base de datos, haz clic en "Bases de datos" para implementar directamente varias bases de datos comunes.
  ![Database services](/start/database-services.webp)
- Si quieres implementar un proyecto de código abierto, las "Plantillas" de Zeabur proporcionan muchos proyectos de código abierto comunes preconfigurados, que puedes implementar con un solo clic seleccionando "Plantillas".
  ![Template services](/start/template-services.webp)
- Si quieres implementar tu propia aplicación personalizada, puedes subir tu código haciendo clic en "GitHub" o "Proyecto local", o puedes hacer clic en "Cursor" para implementar directamente tu proyecto actual.
- Si el proyecto de código abierto que quieres implementar no aparece en "Plantillas", también puedes configurarlo tú mismo usando una "Imagen Docker", lo cual es una operación avanzada.

Tomando como ejemplo la implementación de GitHub, primero debes vincular tu cuenta a GitHub y esperar aproximadamente 1 minuto, luego buscar el repositorio que deseas implementar:

![Search repository](/start/search-repository.webp)

Después de hacer clic en el repositorio, Zeabur analizará automáticamente tu código y seleccionará el método de implementación más adecuado para ti.

![Build plan](/start/build-plan.webp)

Haz clic en "Implementar" para implementar el proyecto con un solo clic. Si es necesario, puedes ir a "Configuración" para ajustar las variables de entorno.

![Build configuration](/start/build-configuration.webp)

## Implementar Servicios usando IA

El Asistente de IA de Zeabur te permite implementar rápidamente los servicios que necesitas usando lenguaje natural.

![Zeabur Agent](/start/zeabur-agent.webp)

Haz clic en "Agente" en la barra de navegación para acceder a la interfaz del Asistente de IA. Si encuentras algún problema de implementación, ¡también puedes pedirle ayuda!

## Paso Cuatro: Lanza tu Sitio Web

Para que tu sitio web sea accesible para otros, puedes vincular tu propio dominio en "Dominios":

![Bind domain](/start/bind-domain.webp)

Espera unos 30 segundos, luego podrás hacer clic en el dominio. Zeabur gestionará HTTPS y otras configuraciones por ti. Una vez que todo esté confirmado, podrás compartir tu sitio web.

Si necesitas vincular tu propio dominio, puedes encontrar más información en la documentación de [Vinculación de Dominio](/deploy/domain-binding).

## ¿Qué sigue?

Si deseas iniciar sesión usando tu cuenta de Google o GitHub, puedes vincular tus cuentas de GitHub y Google en "Configuración" > "Integraciones".

![Account integration](/start/account-integration.webp)

A continuación, puedes seguir explorando la rica biblioteca de plantillas de Zeabur y varias características para diferentes propósitos (por ejemplo, Gestión de archivos, Registros). Si necesitas soporte técnico, el [servidor de Discord](https://zeabur.com/dc) de Zeabur tiene canales dedicados para ayudarte a resolver cualquier problema.
