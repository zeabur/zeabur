# Implementar el Servicio Logto

[Logto](https://logto.io/) es un servicio de verificación de identidad de código abierto que te ayuda a implementar rápidamente la verificación de identidad en tus aplicaciones, y también es la alternativa de código abierto más famosa a [Auth0](https://auth0.com/).

Este artículo te enseñará cómo implementar el servicio Logto en Zeabur a través del mercado de servicios y vincular tu propio nombre de dominio para que tus otros sitios web puedan usar tu servicio Logto implementado para verificar la identidad.

## Paso 1: Implementar desde la plantilla

### Método 1: Implementar con un nuevo proyecto

Primero, visita [la página de la plantilla](https://zeabur.com/templates/AV1NZ4) para Logto.

Puedes ver que la plantilla ya está configurada con el servicio PostgreSQL, por lo que no necesitas hacer nada.

![Página de Plantilla](/marketplace/logto/template.png)

Haz clic en el botón Implementar y luego selecciona una región para implementar tus servicios.

![Seleccionar Región](/marketplace/logto/region.png)

Después de la selección, serás redirigido automáticamente a la página del proyecto donde puedes ver el progreso de la implementación.

![Implementación](/marketplace/logto/deployment.png)

Espera unos segundos más y tu servicio Logto se implementará con éxito y comenzará a funcionar.

### Método 2: Implementar en un proyecto existente

Si ya tienes un proyecto y deseas agregar un sistema de autenticación para el proyecto, también puedes implementar el servicio Logto en el proyecto existente con un solo clic.

Navega a la página del proyecto, haz clic en el botón **Implementar Nuevo Servicio** y luego selecciona Logto de la categoría de plantillas.

![Plantilla en Proyecto](/marketplace/logto/template-in-project.png)

Después de seleccionar Logto, los servicios en la plantilla (Logto y PostgreSQL) se implementarán automáticamente en tu proyecto.

## Paso 2: Vincular nombre de dominio para cada puerto

En el tablero de Zeabur, puedes vincular el nombre de dominio para cada puerto del servicio Logto en la pestaña **Redes**.

Hay dos puertos para el servicio Logto, uno para la interfaz web y otro para la interfaz de la consola de administración. Puedes vincular un nombre de dominio para cada puerto en la consola.

Haz clic en **Generar Dominio** o **Dominio Personalizado**, verás un cuadro de entrada y un selector de puerto, puedes vincular un nombre de dominio para cada puerto aquí.

![Dominio Logto para web](/marketplace/logto/web-domain.png)

![Dominio Logto para consola](/marketplace/logto/console-domain.png)

Puedes consultar [Vinculación de Dominios](/deploy/domain-binding) para obtener más información sobre cómo vincular un nombre de dominio para tu servicio.

## Paso 3: Configurar variables de entorno

Después de vincular el nombre de dominio, necesitamos informar al servicio Logto sobre el nombre de dominio que vinculamos.

El servicio Logto requiere 2 variables de entorno:

- `ENDPOINT`: El nombre de dominio que vinculaste para la interfaz web, por ejemplo `https://logto-web.zeabur.app`.
- `ADMIN_ENDPOINT`: El nombre de dominio que vinculaste para la interfaz de la consola de administración, por ejemplo `https://logto-console.zeabur.app`.

Abre la pestaña **Variables** en la página del servicio, haz clic en el botón **Editar Variables en Bruto** y puedes completar las siguientes variables en el cuadro de entrada.

```json
{
  "ENDPOINT": "https://logto-web.zeabur.app", // Tu nombre de dominio para la interfaz web
  "ADMIN_ENDPOINT": "https://logto-console.zeabur.app" // Tu nombre de dominio para la interfaz de la consola de administración
}
```

Después de agregar la variable, presiona el botón **Reiniciar** y espera unos 30 segundos, luego el servicio se reiniciará y tomará efecto.

## Paso 4: Iniciar sesión y configurar Logto

Ahora todo está listo para tu servicio Logto, es posible que desees configurarlo. Puedes abrir el nombre de dominio que vinculaste para la interfaz de la consola de administración, por ejemplo `https://logto-console.zeabur.app`, y verás la página de inicio de sesión del servicio Logto.

![Logto admin](/marketplace/logto/admin.png)

Y luego puedes crear una cuenta para comenzar con el servicio Logto, solo sigue las instrucciones proporcionadas por el servicio Logto.
