# 建立模板

Zeabur 上的模板可以由任何人建立，它們可以幫助其他使用者更輕鬆地建立自己的項目或服務。本指南將向您展示如何在 Zeabur 上建立模板。

## 如何建立模板

前往 [建立模板](https://dash.zeabur.com/create-template) 頁面。您可以在這裡建立一個模板，以便為其他使用者提供一個單擊部署您的項目或服務的方式。

![Create Template](/template/create/section-1.png)

### 步驟一：填寫基本模板資訊

在你開始加入服務和配置到模板之前，你需要填寫你的模板的基本資訊。

- **模板名稱**：您的模板的名稱。
- **模板描述**：您的模板的簡要描述，推薦 10-30 個字。
- **圖示 URL**：您的模板的圖示圖片的 URL。推薦 128x128 像素。
- **封面 URL**：您的模板的封面圖片的 URL。
- **預覽示範 URL**：您的模板的示範的 URL。

### 步驟二：變數和服務

你的模板由服務和變陣列成。您可以向您的模板加入服務和變數。

![Section 2](/template/create/section-2.png)

#### 模板變數

你可以在這個部分加入一些 *項目級別* 的變數。

> 項目級別變數是所有服務共享的變數。

點選 **建立模板變數** 按鈕，你將看到一個簡單的表單填寫。

![Variables](/template/create/variables.png)

- **變數問題**：當使用者部署模板時，問題將顯示給使用者，引導他們填寫值。
- **變數鍵**：變數的鍵，可以在服務中使用 `${{ key }}` 引用。
- **變數描述**：變數的描述，當使用者部署模板時將顯示給使用者。

#### 模板服務

一個模板需要它的服務才能工作。你可以在這裡加入一些來自 GiHub 儲存庫或預建置鏡像的服務。

你也可以加入一些配置，比如服務變數，根目錄等。

![Services](/template/create/services.png)

#### 模板標籤

你也可以加入一些標籤到你的模板，以幫助使用者更容易地找到你的模板。

### 步驟三：README

為了讓其他使用者知道如何部署和使用模板，您可能需要一個更詳細的指南。您可以在這裡以 markdown 格式編寫 README.md 文件。

![Readme](/template/create/section-3.png)

### 步驟四：提交您的模板

當所有 3 部分都填寫完畢後，您可以點選 **建立模板** 按鈕提交您的模板。

關於如何使用部署按鈕分享此模板，您可以查看這個[指南](../deploy/deploy-button)。
