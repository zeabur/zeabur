# サーバーレスを有効にする

現在、Zeaburはこれらの言語/フレームワークのサーバーレスデプロイをサポートしています：

- [Node.js](/guides/nodejs)
- [Golang](/guides/go)
- [Python](/guides/python)

## 自分のZeaburサービスで試す

「環境変数」欄に `ZBPACK_SERVERLESS=1` を追加し、「再デプロイ」をクリックすると、サービスがサーバーレスサービスとしてデプロイされます。

## 直接サーバーレスサービスになる

サービスのソースコードのルートディレクトリに `zbpack.json` ファイルを作成します（すでに存在する場合は変更します）そして以下の内容を追加します：

```json
{
  "serverless": true
}
```

これにより、サービスはサーバーレス形式でデプロイできると宣言され、Zeaburは自動的にサーバーレス形式でデプロイします。

## コンテナモードに戻す

サービスがサーバーレスモードをサポートしていないことが確実な場合（例えば、重要なバックグラウンドタスクがある場合）、`zbpack.json` の `serverless` フィールドを `false` に変更することをお勧めします。将来的に `ZBPACK_SERVERLESS` を手動で設定しない限り、Zeaburはサーバーレスモードでサービスをデプロイしません。

従来のコンテナモードが正常に動作するかどうかをテストしたい場合は、環境変数に `FORCE_CONTAINERIZED=1` を追加し、再デプロイして従来のコンテナモードに切り替えることができます。この変数は古いサービスに対して事前に有効になっています。
