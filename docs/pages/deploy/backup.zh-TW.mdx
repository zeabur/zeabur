# 備份服務

在生產環境和開發過程中，你可能需要為你的服務建立備份，增強應用程式的可靠性。
我們為所有部署在 Zeabur 上的 [Prebuilt Service](/prebuilt/intro) 提供了一個簡單的方法來建立和復原備份。

<Callout>
    上方提到的 [Prebuilt Service](/prebuilt/intro)，是屬於服務下的一個子分類，[Prebuilt Service](/prebuilt/intro) 是有狀態服務，不同於 `Git Service` 是無狀態的， 所以 `Git Service` 不支援備份功能。
</Callout>

## 步驟一：暫停你的服務

在備份之前請先暫停你的服務，以免在備份過程遺失資料。如果你的服務處於執行狀態，系統會提示你暫停服務。

![backup-not-suspend](/deploy/backup/backup-not-suspend.png)

此時你可以打開 **設定**，找到 **危險區域**，點擊 **暫停服務** 按鈕暫停你的服務。

![suspend](/deploy/backup/backup-suspend-service.png)

## 步驟二：打開備份設定

選擇你想要進行備份的服務，展開 **Backup** 區塊，你會看到一個按鈕，用於為你的服務建立備份。

![button](/deploy/backup/backup-not-suspend.png)

## 步驟三：建立備份

點擊 **Backup** 按鈕，系統會為你的服務建立一個備份，這個過程可能需要一些時間。

![created](/deploy/backup/backup-created.png)

## 步驟四：下載備份

等待一段時間後，你可以看到備份已經建立完成，**Backup** 區塊上會顯示 `SUCCESS`，點擊此處即可下載備份檔案。

![success](/deploy/backup/backup-success.png)

得到下載的檔案後，你可以將其解壓縮並檢閱其中的內容。

![files](/deploy/backup/backup-files.png)
