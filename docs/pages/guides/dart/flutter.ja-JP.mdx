# ZeaburでFlutterをデプロイする

[Flutter](https://flutter.dev)は、Googleが開発したオープンソースのUIソフトウェア開発キットで、クロスプラットフォームのモバイルアプリケーションを構築するために使用されます。

この記事では、Zeabur上でFlutterウェブサイトをデプロイする方法をご案内します。

## ステップ1：Flutterプロジェクトを作成する

### 方法1：`flutter create`コマンドを使用してプロジェクトを作成する

まず、[Flutter公式ドキュメント](https://docs.flutter.dev/get-started/install)に従ってFlutterをインストールしてください。

以下のコマンドを入力して、Flutterが正しくインストールされているかを確認できます。

```bash
flutter doctor
```

すべてのチェック項目が緑色であれば、おめでとうございます。Flutterが正しくインストールされています。

次に、`flutter create`コマンドを使用して新しいFlutterプロジェクトを作成します。

```bash
flutter create myapp
cd myapp
flutter run -d chrome
```

### 方法2：テンプレートを使用してプロジェクトを作成する

あなたがすばやくFlutterプロジェクトを始めるために、私たちはテンプレートを用意しました：[zeabur/flutter-template](https://github.com/zeabur/flutter-template)。

GitHubページで`Use this template`ボタンをクリックして、テンプレートを使用できます。

## ステップ2：Zeaburにデプロイする

### 方法1：Zeabur CLIを使用する

まず、[Zeabur CLIデプロイガイド](/deploy/deploy-in-cli)に従ってZeabur CLIをインストールしていることを確認してください。

次に、新しく初期化したFlutterプロジェクトで、バージョン管理にまだコミットされていないファイルをGitに追加します。

```bash
git init
git add -A
git commit -m "Initial commit"
```

その後、以下のコマンドを使用してサービスをデプロイします。

```bash
zeabur
```

### 方法2：Zeaburダッシュボードを使用する

[Zeaburダッシュボード](https://dash.zeabur.com)にアクセスして新しいプロジェクトを作成します。

次に、`Deploy New Service`ボタンをクリックし、GitHubからデプロイするを選択します。

<Callout>
    Zeaburで初めてGitHubからプロジェクトをデプロイする場合は、先にGitHub Appを設定してインストールする必要があります。
    詳細なガイドは[こちら](/deploy/github)で確認できます。
</Callout>

リポジトリとブランチを選択した後、Zeaburは自動的にサービスのビルドを開始します。

ZeaburはあなたのサービスがFlutterで構築されたことを自動的に検出するため、追加の設定は必要ありません。デプロイは数分以内に完了します。

## 次のステップ

デプロイが完了したら、Flutterウェブサイトのドメインを設定する必要があります。

サービスページの「Domain」タブを開き、「Generate Domain」または「Custom Domain」をクリックします。

サービスにドメインをバインドする方法の詳細については、[ドメインバインディング](/deploy/domain-binding)を参照してください。
