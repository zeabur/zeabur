# Bun

Bun 是一個全能的 JavaScript 執行環境和工具包，旨在加快執行速度，包括打包工具，測試運行器和與 Node.js 兼容的套件管理器。

你可以輕鬆地在 Zeabur 上部署你的 Bun 應用程式。

## 快速開始

### 創建 Bun 應用程式

你可以使用以下命令初始化一個 Bun 應用程式：

```bash
mkdir bun-app

cd bun-app

bun init
```

### 創建 HTTP 伺服器

打開 `index.ts` 並貼上以下程式碼片段，該片段使用 `Bun.serve` 實作簡單的 HTTP 伺服器。

請確保在 `serve` 函數中將 `port` 設為 `Bun.env.PORT`，以確保應用程式能夠使用 Zeabur 提供的 `PORT` 環境變數監聽正確的埠。

```typescript filename="index.ts" {2}
const server = Bun.serve({
  port: Bun.env.PORT ?? 3000, // 將埠設為 `Bun.env.PORT`。
  fetch() {
    return new Response("Bun on Zeabur!");
  },
});

console.log(`Listening on http://localhost:${server.port} ...`);
```

此外，請確保在 `package.json` 文件中指定一個入口點。

<Callout type="info">
目前 Zeabur 支援以下副檔名的檔案作為入口點：`.js`、`.ts`、`.tsx`、`.jsx`、`.mjs`、`.mts`、`.cjs`、`.cts`。
 </Callout>

```json filename="package.json" {3}
{
  "name": "bun-app",
  "module": "index.ts", // let zeabur know where to start
  "type": "module",
  // ...
}
```

## 辨識應用程式

Zeabur 會自動將你的應用程式辨識為 Bun 應用程式，請確保你的專案根目錄下有一個 `bun.lockb` 檔案。

```plainc {2}
root
├── bun.lockb
├── index.ts
└── package.json
```

如果你不希望將 `bun.lockb` 文件提交到儲存庫，你可以將 `bun-types` 安裝到你的專案中。

```json filename="package.json" {3}
{
  "devDependencies": {
    "bun-types": "latest"
  }
}
```

## 部署

點擊 **Git** 按鈕。

搜尋你的 **Bun app** 儲存庫，點擊導入，你的 **Bun app** 將開始部署。

![deploy](/deploy/common/select-service.en-US.png)

## 生態系統

Bun 擁有強大的生態系統，你可以使用各種框架（如 Astro、Nuxt、SvelteKit、ElysiaJS 等）來建立應用程式。

你可以部署這些應用程式並使用 Bun 作為執行環境或套件管理器。

了解更多關於 [Bun - 生態系統](https://bun.sh/guides/ecosystem)的資訊。
