# メトリクス

Zeabur は、すべてのサービスにリアルタイムのリソース使用量メトリクスを提供し、パフォーマンスの監視と情報に基づいたスケーリング判断を支援します。

## メトリクスの表示

サービスのメトリクスを表示するには：

1. Zeabur ダッシュボードでプロジェクトを開きます。
2. 監視したいサービスを選択します。
3. **Metrics** タブをクリックします。

メトリクスダッシュボードには、以下のリアルタイムチャートが表示されます：

- **CPU 使用量** — 現在および過去の CPU 消費量。
- **メモリ使用量** — 現在および過去の RAM 消費量（設定された上限を含む）。
- **ネットワークトラフィック** — 受信および送信データ転送量。

## リソース使用量の理解

### CPU

CPU 使用量は、割り当てられた CPU リソースに対するパーセンテージとして表示されます。CPU 使用量のスパイクは以下を示している可能性があります：

- リクエスト量の増加
- 計算負荷の高い処理
- 最適化の余地がある非効率なコードパス

### メモリ

メモリ使用量は、設定された上限に対してサービスが消費している RAM の量を表示します。サービスが常にメモリ上限に近づいている場合は、アウトオブメモリ（OOM）による再起動を避けるため、[サービス設定](/operations/scaling)で割り当てを増やすことを検討してください。

### ネットワーク

ネットワークメトリクスは、サービスへの入出力データ転送量を表示します。これはトラフィックパターンの理解と帯域幅コストの見積もりに役立ちます。

## サービス使用量と課金

リソース消費量と関連コストの詳細な内訳については、プロジェクト設定の**サービス使用量**ページをご覧ください。このページには以下が表示されます：

- 課金期間中の累積 CPU およびメモリ使用量
- ネットワークエグレス料金
- サービスごとのコスト内訳

使用量が課金にどのように反映されるかの詳細は、[使用量分析](/operations/service-usage)をご覧ください。

<Callout>
  Zeabur は従量課金制を採用しています。メトリクスを定期的に監視することで、リソース消費量を把握し、予期しない課金を避けることができます。料金の詳細は[料金ページ](/pricing)をご覧ください。
</Callout>

## メトリクスを活用したスケーリング判断

メトリクスは、いつ、どのようにスケーリングするかを判断するために不可欠です：

- **CPU が常に80%以上** — CPU 割り当てを増やすか、アプリケーションを最適化することを検討してください。
- **メモリが上限に近い** — OOM クラッシュを防ぐためにメモリ割り当てを増やしてください。
- **安定したトラフィック増加** — 使用量のスパイクがパフォーマンス低下を引き起こす前に、リソースを事前に調整してください。

サービスリソースの調整については、[スケーリング](/operations/scaling)をご覧ください。
