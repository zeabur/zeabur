# ドメイン管理

Zeabur Email でメールを送信するには、まず送信者ドメインを検証する必要があります。ドメイン検証により、ドメインの所有権が確認され、メールの配信性が向上します。

## ドメイン検証が必要な理由

import { Callout, Steps } from 'nextra/components'

<Callout type="info">
  検証されていないドメインはメール送信に使用できません。ドメイン検証は、DKIM、SPF、DMARC レコードを通じてメールの信頼性を向上させます。
</Callout>

- **なりすまし防止**：ドメイン所有者のみがそのドメインからメールを送信できるようにする
- **配信性の向上**：検証済みドメインはスパムフィルターを通過しやすい
- **ブランド保護**：DKIM 署名はフィッシング攻撃からブランドを保護

## ドメインの追加

<Steps>

### コンソールにログイン

Zeabur コンソールの Zeabur Email 管理ページにアクセスします。

### 新しいドメインを追加

1. 「ドメイン管理」に移動
2. 「ドメインを追加」をクリック
3. ドメインを入力（例：`yourdomain.com`）
4. 送信リージョンを選択（ユーザーに近いリージョンを選択することを推奨）

### DNS レコードの設定

システムは追加する必要のある DNS レコードを自動的に生成し、コンソールページに表示します。ページの指示に従って、DNS プロバイダー（Cloudflare、Aliyun、Tencent Cloud など）でこれらのレコードを追加してください。

<Callout type="info">
  **追加する必要のあるレコードタイプ**：
  - **DKIM レコード**（通常3つ）：メールのデジタル署名に使用され、メールの真正性を証明
  - **SPF レコード**：ドメインからメールを送信する権限を持つサーバーを指定
  - **DMARC レコード**：メール認証が失敗した場合の処理ポリシーを定義
  - **MX レコード**（オプション）：バウンス通知の受信に使用
</Callout>

### 検証の完了を待つ

DNS レコードを追加した後、コンソールの「検証」ボタンをクリックします。Zeabur Email は DNS 設定を自動的に確認します。

**検証には通常 5〜30 分かかり、DNS プロバイダーの伝播時間に依存します。**

<Callout type="success">
  コンソールでドメインのステータスが **`verified`** と表示されたら、検証は成功です。このドメインを使用してメールの送信を開始できます！
</Callout>

</Steps>

## DNS レコードの説明

メールが正常に配信されるよう、Zeabur Email は以下の標準メール認証技術を使用します：

### DKIM（メールデジタル署名）

DKIM は各メールにデジタル署名を追加し、メールが実際にあなたのドメインから送信されたことを証明し、メールの改ざんや偽造を防ぎます。通常、3つの DKIM CNAME レコードを追加する必要があります。

**目的**：
- メール偽造の防止
- メール配信性の向上
- ブランドの評判の保護

### SPF（送信サーバー認証）

SPF レコードは、あなたのドメインに代わってメールを送信する権限を持つサーバーを指定し、悪意のある第三者があなたのドメインをなりすますことを防ぎます。

**目的**：
- ドメインのなりすまし防止
- スパムの誤分類の削減

<Callout type="warning">
  ドメインが既に他の場所（Google Workspace、Microsoft 365 など）で SPF レコードを設定している場合は、競合を避けるためにサポートチームに連絡してください。
</Callout>

### DMARC（メール認証ポリシー）

DMARC は、メール認証（SPF または DKIM）が失敗した場合の処理ポリシーを定義し、ドメインの悪用を監視および保護するのに役立ちます。

**目的**：
- 認証失敗レポートの受信
- フィッシング攻撃の防止
- ドメインの信頼性の向上

### MX レコード（オプション）

MX レコードはバウンス通知の受信に使用され、メール送信が失敗した理由を理解するのに役立ちます。

**目的**：
- バウンス通知の受信
- 送信問題の迅速な検出

## 検証ステータス

ドメインは以下のステータスになる可能性があります：

| ステータス | 説明 |
|---------|------|
| `pending` | 検証待ち、DNS レコードがまだ設定されていない |
| `verifying` | DNS レコードを検証中 |
| `verified` | 検証済み、使用可能 |
| `failed` | 検証失敗、DNS 設定を確認してください |

## よくある質問

### DNS レコードが有効になるまでにどのくらいかかりますか？

通常、5分から48時間の間で、DNS プロバイダーによって異なります。検証を試みる前に15〜30分待つことをお勧めします。

### 検証が失敗した場合はどうすればよいですか？

1. **より長く待つ**：DNS の伝播には通常5〜30分かかり、一部の DNS プロバイダーはより長い時間（最大48時間）を必要とする場合があります
2. **DNS 設定を確認**：コンソールに表示されているとおりにレコードを正確に追加したことを確認し、余分なスペースや引用符がないようにします
3. **エラーメッセージを表示**：Zeabur Email コンソールは具体的な検証エラーを表示します。プロンプトに従って変更してください
4. **サポートに連絡**：検証が引き続き失敗する場合は、Zeabur サポートチームに連絡してヘルプを求めてください

### サブドメインを使用できますか？

はい！サブドメイン（`mail.yourdomain.com` など）を検証できます。これにより：
- 異なるタイプのメール（マーケティング vs トランザクション）を分離
- ドメインの評判をより適切に制御

### 検証成功の目印は何ですか？

Zeabur Email コンソールでドメインのステータスが **`verified`** と表示された場合、すべての DNS レコードが正しく設定され、検証に合格したことを意味します。すぐにこのドメインを使用してメールの送信を開始できます。

## ドメインの削除

ドメインが不要になった場合：

1. 「ドメイン管理」に移動
2. 削除するドメインを見つける
3. 「削除」ボタンをクリック
4. 削除を確認

<Callout type="warning">
  ドメインを削除すると、そのドメインからメールを送信する API 呼び出しは失敗します。このドメインを使用しているアプリケーションがないことを確認してください。
</Callout>

## ベストプラクティス

### 専用サブドメインを使用

メール送信には専用サブドメイン（`mail.yourdomain.com` など）を使用することをお勧めします。これにより：
- メインドメインの評判を保護
- メールの追跡と管理が容易
- 異なるタイプのメール（マーケティングメール、トランザクションメールなど）を分離

### DNS の頻繁な変更を避ける

ドメイン検証が成功した後は、DNS レコードをランダムに変更または削除しないでください。そうしないと、メール送信の失敗やメールがスパムとしてマークされる可能性があります。
