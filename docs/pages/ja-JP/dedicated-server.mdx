# 専用サーバー

共有クラスターに加えて、お持ちのサーバーを **Zeabur に登録**し、サービスのデプロイ、ファイアウォールルール、リソース割り当てを Zeabur に任せることができます——SSH での手動操作は不要です。

## 専用サーバーを使う理由

- **リソース独占**：他のユーザーとハードウェアを共有しないため、パフォーマンスが安定し予測可能です。
- **自前のハードウェア（BYOH）**：AWS、GCP、Hetzner、または自社データセンターでサーバーを運用中？ワンクリックで Zeabur にマウントし、デプロイ管理機能をフル活用できます。
- **地理的な柔軟性**：共有クラスターがカバーしていない地域を、自前のインフラで補えます。
- **高信頼性の本番環境**：SLA やセキュリティ要件が厳しい大規模システムに最適です。

## Zeabur が管理すること

サーバーを登録すると、ホストに直接触れることなく、Zeabur Dashboard から以下の操作が可能です：

- サービスのデプロイと更新
- リアルタイムのログとパフォーマンス指標の閲覧
- 環境変数の設定
- 永続ストレージとネットワーク設定の管理

<Callout type="info">
  専用サーバーはリソース使用量ではなく、月額固定料金で課金されます。ハードウェア要件が安定した本番ワークロードに最適です。
</Callout>

## よくある質問

### 専用サーバーを使用するにはサブスクリプションプランが必要ですか？

いいえ、専用サーバーの使用にサブスクリプションプランは必須ではありません。ただし、Zeabur の完全な機能を活用するために開発者プランをお勧めします。バックアップなどの機能は開発者プランの加入者のみが利用可能です。開発者プランユーザーはスタッフからの優先サポートを受けられますが、無料ユーザーは [Zeabur フォーラム](https://zeabur.com/forum)でサポートを受けられます（回答時間の保証なし）。詳細は[プラン](/ja-JP/billing/plans)をご覧ください。

### 小規模なウェブサイトをホストしたいだけですが、プランとサーバーの両方が必要ですか？

サブスクリプションプランなしでも専用サーバーにサービスをデプロイできます。ただし、バックアップなどの特定の機能は開発者プラン限定です。Zeabur のプラットフォーム機能と優先サポートをフル活用したい場合は、サーバーに加えて開発者プランへの加入をお勧めします。詳細は[コミュニティとサポート](/ja-JP/community/forum)をご覧ください。
