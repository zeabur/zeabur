# プライベートネットワーク

プライベートネットワークを使用すると、同じプロジェクト内のサービスが固定のホスト名で別のサービスに接続できます。

## ホスト名の確認

サービスの「ネットワーク」セクションを開き、「プライベート」の下にあるホスト名を確認します。

![ホスト名の確認](/networking/private/find-hostname.webp)

## サービスへの接続

上記のホスト名を使用してこのサービス（この例では PostgreSQL）に接続します。また、[PostgreSQL のドキュメント](/marketplace/postgresql)に記載されている通り、ポートは `POSTGRES_PORT` 環境変数に注入されます。接続情報は以下のように入力できます：

- ホスト名：前のステップで確認したホスト名（この例では `postgresql.zeabur.internal`）
- ポート：`POSTGRES_PORT` 環境変数から値を取得（[デフォルトは 5432](https://github.com/zeabur/zeabur/blob/bd565389cf1868a917333d0f74003b4aad6ef0ce/prebuilts/postgresql.toml#L14)）

## 重要な注意事項

- サービス名を変更しても、ホスト名は**変更されません**。
