# プロジェクトをコピー

Zeaburは**プロジェクトコピー機能（Project Copy）**を提供しており、プロジェクトを任意のリージョンに簡単にコピーできます。

プロジェクト設定ページに行くだけで、**ワンクリックプロジェクトコピー機能**を見つけることができます。この機能は、プロジェクトアーキテクチャ、環境変数、さらにはPVデータも含めて、選択した任意のリージョンにプロジェクト全体を自動的にコピーします。プロセス全体はわずか数分で完了し、時間と労力を大幅に節約できます。

- **PVなしプロジェクト：** プロジェクトにマウントされたPVがない場合、コピープロセスは**ゼロダウンタイム**を実現でき、サービスを再起動するような滑らかな体験ができます。
- **PVありプロジェクト：** プロジェクトにマウントされたPVがある場合、コピー時間はデータサイズに依存します。バックエンドは自動的にPVデータをS3ストレージにバックアップし、新しいプロジェクトに復元して、データが完全かつ安全であることを保証します。

## コピー内容の説明

<Callout>
    プロジェクトコピー機能はプロジェクトを完全にコピーし、**すべてのデータを含みます**。コピー後はドメインを再バインドする必要があり、元のプロジェクトのドメイン設定を自動的に継承しません。
</Callout>

プロジェクトをコピーする際、以下の内容が完全にコピーされます：
- **プロジェクトアーキテクチャ**：すべてのサービスの設定と関係
- **サービスタイプ**：Dockerイメージ、Gitリポジトリなど
- **環境変数**：設定されたすべての環境変数
- **PVデータ**：永続ストレージスペース内のすべてのデータ
- **ネットワーク設定**：ポート設定とネットワークルール
- **リソース制限**：CPUとメモリ制限設定

## 注意事項

1. **ゼロダウンタイムコピー**：PVのないプロジェクトはゼロダウンタイムコピーを実現でき、PVのあるプロジェクトはデータサイズに依存してコピー時間が決まります。

2. **データ整合性**：コピープロセス中、すべてのデータが完全にバックアップされ復元され、データセキュリティが保証されます。

3. **ドメイン再バインド**：コピーされたプロジェクトはドメインを再バインドする必要があり、元のプロジェクトのドメイン設定を自動的に継承しません。

4. **リソース消費**：コピープロセス中、データバックアップ用に一時的に追加のストレージスペースを消費します。
