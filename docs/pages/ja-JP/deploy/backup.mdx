# バックアップサービス
本番環境や開発中に、アプリケーションの堅牢性と信頼性を高めるためにサービスのバックアップを作成する必要があるかもしれません。
Zeabur でデプロイされたすべてのプリビルトサービスのバックアップを作成および復元するための簡単な方法を提供します。

<Callout>
    前述のプリビルトサービスは、サービスカテゴリに属するサブカテゴリです。プリビルトサービスはステートフルサービスであり、`Git Service`のようなステートレスサービスではありません。そのため、`Git Service`はバックアップ機能をサポートしていません。
</Callout>

## ステップ1: バックアップを作成する

[Zeaburダッシュボード](https://dash.zeabur.com)を開き、バックアップしたいサービスページに移動して**バックアップ**タブを開くと、サービスのバックアップを作成するためのボタンが表示されます。

バックアップボタンをクリックすると、自動的にサービスのバックアップが作成されます。このプロセスには時間がかかる場合があります。

![created](/deploy/backup/backup-created.png)

<Callout>
    私たちは、MongoDB、MySQL、PostgreSQL、およびRedisを含むデータベースのサービスを停止せずにバックアップを作成することのみをサポートしています。
    その他のサービスについては、バックアップを作成する前にサービス設定でサービスを一時停止する必要があります。
    バックアップが作成された後、サービスを再起動できます。
</Callout>

## ステップ2: バックアップをダウンロードする

バックアップが作成されると、**SUCCESS**メッセージが表示されます。ダウンロードボタンをクリックしてバックアップファイルをダウンロードします。

![success](/deploy/backup/backup-success.png)

ダウンロードしたファイルを解凍して、その内容を確認できます。

![files](/deploy/backup/backup-files.png)
