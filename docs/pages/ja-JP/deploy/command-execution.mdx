# コマンドの実行

Zeaburのサービス内でコマンドを実行して、パッケージのインストール、プログラムの実行、情報の確認、特定のデータのバックアップなどを行うことができます。

## コマンド実行インターフェースを開く

サービスの **Overview** タブを開き、**Command** をクリックすると「Execute Command」インターフェースが表示されます。

![コマンド実行インターフェースを開く](/deploy/command-execution/Command_part1.png)

コマンド実行インターフェース：

- コマンド出力エリア：コマンドの実行結果を表示。成功時は出力が表示され、エラー時はエラーコードが表示されます。

![コマンド実行インターフェース](/deploy/command-execution/terminal.png)

## コマンドを実行

基本例として `echo Hello, World!` を入力すると、出力に `Hello, World!` が表示されます。

![hello worldコマンドの実行](/deploy/command-execution/terminal_2.png)

存在しないコマンドを入力するとエラーコードが表示されます。

![コマンド実行失敗](/deploy/command-execution/terminal_3.png)

MySQLやPostgreSQLなどのデータベースコンテナ内でSQL文を実行したり、cURLでログをアップロードしたりファイルをバックアップしたり、マイグレーションプログラム（例：`prisma migrate`、`node seeder.js`）を実行することも可能です。

MySQLの場合、「Command to connect to your MySQL」のコマンドをコピーし、`host` と `port` パラメータを削除（コンテナ内接続なので不要）、`mysqlsh` を `mysql` に書き換え（コンテナ内に `mysqlsh` はない）、最後に `-e` を追加してSQL文を実行します。

```bash
mysql --user=root --password=<password> --database=zeabur -e "SHOW DATABASES;"
```

![MySQLコマンドの実行](/deploy/command-execution/MySQL_terminal.png)
