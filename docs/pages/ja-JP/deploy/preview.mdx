# プレビュー環境

たまに、プルリクエストのためにプレビュー環境を展開したい場合があります。これは、メインブランチにマージする前に変更をテストするのに便利です。
Zeabur では、Zeabur で展開されているすべてのサービスに対して簡単な方法を提供しています。

このガイドでは、1クリックでプルリクエストのプレビューを有効化する方法を示します。

<Callout>
  この機能は Developer プランと Team プランのユーザーにのみ利用可能です
</Callout>

## Git サービスを選択

[はじめ方](/get-started)で述べたように、Zeabur には **Git サービス** と **マーケットプレイスサービス** の2種類のサービスがあります。
プルリクエストのプレビューは、Git サービスのみで利用できます。なぜなら、それは Git リポジトリに基づいているからです。

まだ Git サービスを作成していない場合は、[はじめ方ページ](/get-started)のガイダンスに従って作成することができます。

## プルリクエストのプレビューを有効化

プレビューしたいサービスを開き、設定タブを展開するためにスクロールします。
**プルリクエストプレビュー** というスイッチボタンが表示されます。このスイッチボタンをクリックして、プルリクエストのプレビューを有効化してください。

![enable](/environment/preview/enable.png)

## プレビュー環境を確認する

プルリクエストプレビューを有効にしたら、Git リポジトリでプルリクエストを作成することができます。
プルリクエストを作成すると、Zeabur が自動的にプレビュー環境を作成します。

さらに、Zeabur はプレビューサービスに簡単にアクセスできるように、ユニークなドメイン名を自動的に割り当てます。
ダッシュボード内のリンクまたは Git リポジトリ内の展開リンクをクリックして、プレビューサービスにアクセスできます。

![link](/environment/preview/link.png)

そして心配いりません。プルリクエストがマージされたり閉じられたりすると、プレビュー環境は自動的に削除されます。
