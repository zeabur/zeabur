# テンプレートを作成する

Zeaburのテンプレートは誰でも作成することができ、他のユーザーが自分のプロジェクトやサービスをより簡単に作成するのに役立ちます。このガイドでは、Zeaburでテンプレートを作成する方法を紹介します。

## テンプレートの作成方法

Zeaburの[Create Template](https://dash.zeabur.com/create-template)ページにアクセスします。ここで、自分のプロジェクトやサービスのワンクリック展開を他の人に提供するためのテンプレートを作成できます。

![Create Template](/template/create/section-1.png)

### ステップ1：基本的なテンプレート情報の入力

テンプレートにサービスや設定を追加する前に、テンプレートの基本情報を入力する必要があります。

- **テンプレート名**：テンプレートの名前。
- **テンプレートの説明**：テンプレートの簡単な説明、10〜30語が推奨されています。
- **アイコンのURL**：テンプレートのアイコン画像のURL。128x128ピクセルが推奨されています。
- **カバー画像のURL**：テンプレートのカバー画像のURL。
- **プレビューデモのURL**：テンプレートのデモのURL。

### ステップ2：変数とサービス

テンプレートはサービスと変数で構成されています。テンプレートにサービスと変数を追加できます。

![セクション2](/template/create/section-2.png)

#### テンプレート変数

このセクションには、いくつかの*プロジェクトレベル*の変数を追加できます。

> プロジェクトレベルの変数は、テンプレート内のすべてのサービスで共有される変数です。

**新しいテンプレート変数**ボタンをクリックすると、入力するための簡単なフォームが表示されます。

![変数](/template/create/variables.png)

- **変数の質問**：テンプレートを展開する際にユーザーに表示される質問。
- **変数キー**：変数のキー、`${{ key }}`でサービス内で参照できます。
- **変数の説明**：変数の説明、テンプレートを展開する際にユーザーに表示されます。

#### テンプレートサービス

テンプレートにはサービスが必要です。ここで、GiHubリポジトリやプレビルドイメージからいくつかのサービスを追加できます。

そして、サービス変数、ルートディレクトリなどの設定を追加することもできます。

![サービス](/template/create/services.png)

#### テンプレートのタグ

ユーザーがテンプレートをより簡単に見つけるために、テンプレートにいくつかのタグを追加することもできます。

### ステップ3：Readme

他のユーザーにテンプレートの展開方法や使用方法を知らせるために、より詳細なガイドが必要です。ここで、Markdown形式のREADME.mdファイルを作成できます。

![Readme](/template/create/section-3.png)

### ステップ4：テンプレートの送信

3つのセクションすべてに入力が完了したら、**テンプレートを作成**ボタンをクリックしてテンプレートを送信できます。

このテンプレートを展開ボタンで共有する方法については、この[ガイド](../deploy/deploy-button)を参照してください。
