# テンプレートサービスからコードをフォーク

Zeaburではテンプレートサービスを使ってアプリケーションを素早くデプロイできます。テンプレートには複数のサービスが含まれ、それぞれPrebuiltサービス（Dockerイメージからデプロイ）またはGitサービス（GitHubからデプロイ）です。

デフォルトでは、Gitサービスを含むテンプレートをデプロイすると、Zeaburはテンプレート提供元のGitHubリポジトリから直接サービスをデプロイし、テンプレート作者が設定したリポジトリにCI/CDを自動的にバインドします。つまり、テンプレート作者がリポジトリに新しいコミットをプッシュすると、Zeaburが自動でCI/CDをトリガーしてあなたのプロジェクトに新しいバージョンをデプロイします。

しかし、テンプレート作者のコードを修正して独自の機能を追加したり設定を変更したりしたい場合があります。このような場面でZeaburは便利なワンクリックFork機能を提供しています。

## テンプレートサービスからコードをフォークする方法

[Zeabur公式Express.jsテンプレート](https://zeabur.com/templates/W96I3Z)を例に取ります。このテンプレートをデプロイ後、expressサービスの設定ページを開くと、Git Repositoryフィールドに現在 `https://github.com/zeabur/expressjs-template` と表示されており、これがテンプレートで指定されたGitHubリポジトリです。

このテンプレートを基にコードを修正したい場合、下にあるFork Repositoryボタンをクリックします：

![Express.js github fork](/template/fork-git-repo-from-template/Expressjs_github_fork.png)

クリックすると、リポジトリ名を尋ねるウィンドウが表示されます。まだ使われていないリポジトリ名を選んでください（使用済みだとForkが失敗する可能性があります）。名前の下にプライベート設定の入力ボックスもあり、コード変更を他人に見せたくない場合に便利です：

![git fork](/template/fork-git-repo-from-template/git_fork.png)

名前を決めたらForkボタンをクリック。GitHubの認証がまだの場合、GitHubのOAuth認証ページにリダイレクトされます。認証完了後、Forkの読み込み画面が表示されます：

![Configure Github](/template/fork-git-repo-from-template/Configure_Github.png)

![github connection](/template/fork-git-repo-from-template/github_connection.png)

完了すると、あなたのGitHubアカウントに元のコードと同じ新しいリポジトリが追加されます。

Zeaburは自動的にこのテンプレートのコードをあなたのGitHubアカウントにフォークし、このフォークされたリポジトリをサービスのGit Repositoryとして設定します。つまり、このフォークリポジトリをGitHubで修正・コミット・プッシュすると、Zeaburが自動でCI/CDをトリガーしてプロジェクトに新しいバージョンをデプロイします。
