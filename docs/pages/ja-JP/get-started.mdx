# **クイックスタート**

## **ステップ1: アカウントを作成する**

Zeaburでサービスをデプロイするには、アカウントを作成する必要があります。

Zeaburが初めての場合は、メールアドレスで登録してください（最初のオプション）。ここにメールアドレスを入力し、「メールで続行」をクリックします。

![SignUp.png](/start/get-started/SignUp.png)

すると「メール送信成功」というメッセージが表示されます。

![SignUp Part 2.png](/start/get-started/SignUp%20Part%202.png)

メールの受信トレイ（例: Gmail、Outlook、Apple Mail）を開くと、Zeaburからのメールが届いています。メール内の「ログイン」ボタンをクリックしてください。

![Sign Up Callback.png](/start/get-started/Sign%20Up%20Callback.png)

ログイン処理で開くブラウザが希望のデフォルトブラウザでない場合（例: ChromeではなくSafariが開く）、サインインボタンの下に表示されるリンクをコピーして、希望のブラウザに貼り付けてください。

このログインインターフェースが表示されたら、ログイン成功です！

![Dashboard.png](/start/get-started/Dashboard.png)

左パネルでインターフェースの言語を切り替えられます！現在、繁体字中国語、簡体字中国語、英語、日本語、スペイン語、インドネシア語に対応しています。

![Languages.png](/start/get-started/Languages.png)

## **ステップ2: プロジェクトを作成する**

Zeaburでは、プロジェクトは現実世界のアプリケーションを構成する関連サービスすべての単一コンテナとして機能します。例えば、フロントエンドウェブサイト、バックエンドAPI、データベースからなるアプリケーションの場合、それらすべてを同じプロジェクト内に保持する必要があります。

![Project structure](/start/project-structure.webp)

プロジェクト作成を開始するには、中央にある**プロジェクトを作成**ボタンをクリックします。

![Create New Project.png](/start/get-started/Create%20New%20Project.png)

初めてプロジェクトを作成する場合、「アカウント認証」のプロンプトが表示される可能性があります。

![Toast if not verified.png](/start/get-started/Toast%20if%20not%20verified.png)

以下のいずれかを選択できます：

1. **電話番号**で認証する。ほとんどの国・地域で対応しています。対応していない場合や複数アカウントを紐づける必要がある場合は、[お問い合わせ](https://zeabur.com/forum)に加え、2番目と3番目のオプションも参照してください。
2. いくらかの**利用クレジット**を事前入金する。Devプランに加入していない、専用サーバーを購入していない、重大な悪用（例: 高ボリュームDoS攻撃によるToS違反）に関与していない限り、残高から差し引かれることはありません。
3. アカウント認証のため**クレジットカード**が必要です。スムーズな運用を保証します。サブスクリプション関連の取引を積極的に開始しない限り、課金は発生しません。

希望する方法を選択し、対応するセクションの「認証」ボタンをクリックしてください。

認証が完了したら、プロジェクトを再度作成すると正常に作成されたプロジェクトが表示されます。

![After Verification.png](/start/get-started/After%20Verification.png)

## **ステップ3: サービスをデプロイする**

Zeaburはデータベース（PostgreSQL、MySQL、Redis）、オープンソースプロジェクト（n8n、WordPress）、または独自のカスタムアプリケーション（GitHubからインポートまたはフォルダからアップロード）など、さまざまなサービスをデプロイできます。

- データベースをデプロイしたい場合、「データベース」をクリックして一般的なデータベースを直接デプロイします。
    
    ![Databases.png](/start/get-started/Databases.png)
    
- オープンソースプロジェクトをデプロイしたい場合、Zeaburの「テンプレート」には多くの事前設定済みプロジェクトがあり、「テンプレート」を選択してワンクリックでデプロイできます。
    
    ![templates.png](/start/get-started/templates.png)
    
- 独自のカスタムアプリケーションをデプロイしたい場合、「GitHub」または「ローカルプロジェクト」をクリックしてコードをアップロードするか、「Cursor」をクリックして現在のプロジェクトを直接デプロイできます。
- デプロイしたいオープンソースプロジェクトが「テンプレート」にない場合、「Dockerイメージ」を使って自分で設定することも可能です（高度な操作）。

GitHubデプロイを例にします。まずZeaburアカウントをGitHubに連携し、約1分待ってからデプロイしたいリポジトリを検索します：

![Github.png](/start/get-started/Github.png)

リポジトリをクリックすると、Zeaburエージェントがコードを自動解析し、最適なデプロイ方法を選択します。

![Build Plan.png](/start/get-started/Build%20Plan.png)

「デプロイ」をクリックするとワンクリックでプロジェクトがデプロイされます。必要に応じて「設定」から環境変数を調整できます。

設定では、ルートディレクトリをアプリのフォルダ（例: “/” またはサブフォルダ）に設定し、キー/値または生の変数編集で必要な環境変数を追加したら、次へをクリックして確認します。

![Configure Build Plan  .png](/start/get-started/Configure%20Build%20Plan%20%20.png)

## **Zeabur AIエージェントを使って任意のサービスをデプロイ**

ZeaburのAIエージェントを使えば、自然言語で必要なサービスを素早くデプロイできます。

![Postgresql Examples.png](/start/get-started/Postgresql%20Examples.png)

右パネルの「Zeabur AI」をクリックしてAIアシスタントインターフェースにアクセスしてください。デプロイで問題が発生した場合も、そこで助けを求めることができます！

## **ステップ4: ウェブサイトを公開する**

ウェブサイトを他の人にアクセス可能にするには、「ドメイン」で独自ドメインを紐づけられます：

![Bind domain](/start/bind-domain.webp)

約30秒待つとドメインをクリックできます。ZeaburがHTTPSなどの設定を処理してくれます。すべて確認できたら、ウェブサイトを共有してください。

独自ドメインの紐づけが必要な場合は、[ドメイン紐づけ](https://zeabur.com/docs/deploy/domain-binding)のドキュメントを参照してください。

## **次は何をしますか？**

GoogleまたはGitHubアカウントでログインしたい場合、「設定」>「インテグレーション」からGitHubとGoogleアカウントを紐づけられます。

![Account integration](/start/account-integration.webp)

次に、Zeaburの豊富なテンプレートライブラリやさまざまな機能（例: ファイル管理、ログ）を引き続き探索してください。技術的な問題は[Zeabur公式フォーラム](https://zeabur.com/forum)で開発者チケットを送信してください。コミュニティでの議論やピアサポートは[Zeabur Discordサーバー](https://zeabur.com/dc)に参加してください。
