# DNS レコード

各登録済みドメインの詳細ページに DNS エディタが内蔵されています。管理リストから任意のドメインをクリックしてアクセスします。

## 対応レコードタイプ

| タイプ | 説明 | プロキシ対応 |
|--------|------|-------------|
| **A** | ドメインを IPv4 アドレスにマッピング | はい |
| **AAAA** | ドメインを IPv6 アドレスにマッピング | はい |
| **CNAME** | 別のドメイン名へのエイリアス | はい |
| **MX** | メール交換サーバー | いいえ |
| **TXT** | 任意のテキスト（SPF、DKIM、認証など） | いいえ |
| **SRV** | サービスロケーター | いいえ |
| **CAA** | 認証局の承認 | いいえ |
| **NS** | ネームサーバーの委任 | いいえ |

## レコードの追加

<Steps>

### DNS エディタを開く

ドメイン詳細ページに移動し、DNS レコードテーブルの上にある **レコードを追加** をクリックします。

### レコードを入力

- **タイプ**：ドロップダウンからレコードタイプを選択します。
- **名前**：サブドメインまたは `@`（ルートドメイン）。
- **内容**：レコードの値（IP アドレス、ホスト名、テキストなど）。
- **TTL**：プリセット値から選択（auto、60s、300s、3600s、86400s など）。プロキシが有効な場合、TTL は自動的に auto に設定されます。
- **優先度**：MX および SRV レコードで必須。
- **プロキシ**：A、AAAA、CNAME レコードのみ利用可能。[Cloudflare プロキシ](#cloudflare-プロキシ)を参照。

### 保存

**保存** をクリックしてレコードを作成します。変更は通常数分以内に反映されます。

</Steps>

## レコードの編集と削除

- レコード行の**編集アイコン**をクリックして、内容、TTL、優先度、プロキシ設定を変更できます。レコードタイプと名前は作成後に変更できません。変更が必要な場合は削除して再作成してください。
- **削除アイコン**をクリックしてレコードを削除します。

## Cloudflare プロキシ

A、AAAA、CNAME レコードにはプロキシトグルがあります。有効にすると：

- トラフィックが Cloudflare を経由してルーティングされ、DDoS 保護、キャッシュ、SSL 終端が提供されます。
- TTL は自動的に **Auto** に設定され、手動で変更できません。
- DNS ルックアップで実際のオリジン IP アドレスが隠されます。

<Callout type="info">
  プロキシはデフォルトで無効です。Cloudflare の保護とパフォーマンス機能が必要な場合に、該当するレコードで有効にしてください。
</Callout>
