# ボリューム

Zeabur では、ボリュームをマウントしてサービス内のデータを永続的に保存できます。通常、Zeabur のサービスは再起動時にデータがデフォルト値にリセットされます（これは<abbr title="ステートレス">ステートレス</abbr>と呼ばれます）。しかし、データを永続化する必要がある場合（サービスを<abbr title="ステートフル">ステートフル</abbr>にする場合）、永続ストレージが必要なディレクトリをボリュームにマウントできます。これにより、サービスが再起動してもデータは失われません。

ボリュームの料金情報については、[料金ページ](/pricing/pricing-plans)をご参照ください。

<Callout type="info">
    設定ファイルのアップロードのみが必要な場合は、[設定ファイルエディタ](/data-management/config-edit)を使用する方が便利で、追加費用もかかりません。
</Callout>

## 注意事項

- デフォルトでは、Zeabur のサービスはゼロダウンタイム再起動をサポートしています。つまり、サービスを再起動するとトラフィックは新しいインスタンスにシームレスに切り替わり、ユーザーに中断は発生しません。
- ただし、ボリュームを有効にすると、サービスは**ゼロダウンタイム再起動をサポートできなくなります**。再起動のたびにサービスが完全にシャットダウンされてから再起動されるため、短時間のダウンタイムが発生します。

## ボリュームのマウント

ボリュームをマウントしたいサービスのページを開き、「Volumes」タブをクリックすると、「Mount Volumes」ボタンが表示されます。クリックすると、2つのフィールドが表示されます：

![mount volumes](/data-management/volumes/mount-volumes.webp)

- **Volume ID** はこのボリュームの一意の識別子です。このボリュームを簡単に表すために、シンプルな名前（`data`、`cache`、`photos` など）を使用できます。この識別子は実際のマウント結果には関係ありません。
- **Mount Directory** はボリュームがマウントされるディレクトリで、本質的に「永続ストレージが必要なフォルダ」です。サービス内の `/data` ディレクトリを永続化したい場合は、ここに `/data` と入力します。

**マウント後、このディレクトリ内のデータは完全にクリアされます**。すでに重要なデータがある場合は、マウントしてサービスを再起動する前にダウンロードしてから、マウント後にデータを再インポートしてください。

## ボリュームのアンマウント

特定のボリュームの課金を停止したい場合は、ボリュームの横にある「Delete」ボタンをクリックします。削除すると、このスペースの課金が停止します。削除後、このディレクトリ内のデータは**完全にクリアされ**、**復元できません**。

![unmount volumes](/data-management/volumes/unmount-volumes.webp)

## テンプレート YAML でのボリュームの使用

「サービス仕様」で `volumes` を使用してボリュームをマウントできます。詳細については、[YAML からテンプレートを作成する](/template/template-format)の記事をご参照ください。
