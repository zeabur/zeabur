# 専用サーバーの登録

この記事では、Zeabur に [専用サーバー](../dedicated-server) を登録する方法を説明します。

## 前提条件

- すでにサーバーを所有しており、そのサーバーには接続可能な少なくとも1つのパブリックIPアドレスが必要です。
- サーバーには少なくとも1つのCPUコアと少なくとも2GBのメモリが必要です。
- サーバーは `22`, `80`, `443`, `4222`, `6443`, `30000-32767` のポートを開放している必要があります。
- サーバーはSSHログインを許可し、ユーザー名は `root` である必要があります。
- サーバーは `root` ユーザーがパスワードまたはSSHキーを使用してログインすることを許可する必要があります。

## 登録手順

1. Zeabur にログインした後、トップナビゲーションバーまたはプロジェクトリストページの上部にある「マイサーバー」をクリックするか、直接 [https://dash.zeabur.com/servers](https://dash.zeabur.com/servers) にアクセスします。
2. ページの右上隅にある「作成」ボタンをクリックします。
3. サーバーのIPアドレスとSSHポート（デフォルトは `22`）を入力します。
4. `root` ユーザーがSSHでログインする方法を選択します：パスワードまたはSSHキー。
5. パスワードを選択した場合は、下にパスワードを入力し、SSHキーを選択した場合は、下にSSHキーを入力します。
6. 「サーバーを追加」ボタンをクリックし、サーバーの登録が完了するのを待ちます（このプロセスには約30秒かかります）。
7. 登録が成功すると、サーバーリストに新しく登録されたサーバーが表示されます。
8. ステータスが「オンライン」と表示され、CPUとメモリの使用状況が正常に表示されるまで約1分待ちます。

登録に失敗した場合は、[Discord サーバー](https://zeabur.com/dc) で報告してください。

## アンインストール

まず、Zeabur からサーバーおよびその上でデプロイされたサービスを削除する必要があります。

![サーバーを削除する](/dedicated-server/remove-server.webp)

「サーバーを削除する」手順では **Zeabur がサーバー上に設定したインフラストラクチャは削除されません**。Zeabur のインフラストラクチャは主に [K3s](https://k3s.io/) 上に構築されていますので、アンインストール手順については [K3s のアンインストール手順に従ってください](https://docs.k3s.io/installation/uninstall)。

```bash
$ /usr/local/bin/k3s-uninstall.sh
```

## スペースのクリーンアップ

もしホストのディスクスペースが不足している場合は、K3s がホスト上でプルした Docker イメージを削除することを試してください（通常は自動的に実行されます）。

```bash
# 現在プルされているイメージを確認（この行は入力不要）
$ sudo k3s crictl images

# 使用されていないすべてのイメージを削除
$ sudo k3s crictl rmi --prune
```

場合によっては、K3s が各コンテナに記録したログが大量のスペースを占有することがあります。その場合は [K3s のログディレクトリ](https://docs.k3s.io/faq#where-are-the-k3s-logs) に移動してログをクリーンアップすることを検討してください。
