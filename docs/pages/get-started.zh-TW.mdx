# 快速開始

## 步驟一：建立帳號

你需要先建立一個帳號來開始在 Zeabur 部署你的服務。
Zeabur 目前僅支援透過 GitHub 登入，如果你還沒有 GitHub 帳號，可以在[這裡](https://github.com/join)建立一個。

前往 Zeabur 的[登入頁面](https://dash.zeabur.com/login)，你會看到一個透過 GitHub 登入的按鈕。
點擊該按鈕，你會被重新導向到 GitHub 的授權頁面。點擊「Authorize」按鈕授權 Zeabur 存取你的 GitHub 帳號。

![sign in](/start/sign-in.png)

## 步驟二：建立專案

在 Zeabur 中，一個專案代表的是一個你實際的工程專案。你可以在一個專案中部署多個不同類型的服務以滿足實際的需求。

授權成功後，將自動重新導向至 Zeabur 的控制台頁面。
點擊 **建立專案** 按鈕建立一個新的專案。你也可以使用快捷鍵 <kbd>CMD + k</kbd> 打開快速操作選單，然後選擇 `建立專案`。

接著會有一個彈出視窗供你選擇專案所在的區域。Zeabur 目前支援兩個區域，包括 GCP 位於 `asia-east` 和 AWS 位於 `us-west` 的叢集。
選擇你想要部署你的服務的區域，我們會以一個隨機的名稱建立專案，稍後你可以在專案設定中修改它。

![new project](/start/create-project.png)

<Callout>
  如果這是你第一次建立專案，你可能需要先遵循指引，閱讀並同意我們的服務條款以繼續
</Callout>

## 步驟三：部署服務

點擊你剛剛建立的專案，便可以查看其詳細資訊。

![project](/start/project.png)

想要部署一個服務，只需要點擊 **加入服務** 按鈕。你會看到一個彈出視窗供你選擇服務類型。

![select service type](/start/select-service-type.png)

Zeabur 支援兩種類型的服務，包括 **Git Service** 和 **Marketplace Service**。

- **Git Service** 允許你部署你的原始碼。Zeabur 會自動為你建置和部署你的原始碼。
  如果是初次使用，你需要為 Zeabur 配置 GitHub 存取權限，這樣 Zeabur 才能存取你的儲存庫。

- **Marketplace Services** 允許你從一個常用服務的市場中部署一個服務（例如資料庫或開源工具），你可以在[這裡](https://github.com/zeabur/zeabur/tree/main/prebuilts)看到我們的開源的服務市場列表。

選擇你想要部署的服務的類型，然後你就可以自由探索你想要部署的服務。

## 下一步

部署成功以後，你可以學習[網域綁定](/deploy/domain-binding)來為你的服務綁定一個網域並自動配置 TLS 憑證，或者學習[環境變數](/environment/variables)來了解 Zeabur 如何自動注入變數到你的程式中。
