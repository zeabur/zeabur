# 域名管理

要使用 Z-Send 发送邮件，您需要先验证发件域名。域名验证确保您拥有该域名，并提高邮件送达率。

## 为什么需要验证域名

import { Callout, Steps } from 'nextra/components'

<Callout type="info">
  未经验证的域名无法用于发送邮件。域名验证通过 DKIM、SPF 和 DMARC 记录提高邮件的可信度。
</Callout>

- **防止伪造**：确保只有域名所有者才能使用该域名发送邮件
- **提高送达率**：已验证的域名更容易通过垃圾邮件过滤器
- **品牌保护**：DKIM 签名保护您的品牌免受钓鱼攻击

## 添加域名

<Steps>

### 登录控制台

访问 Zeabur 控制台的 Z-Send 管理页面。

### 添加新域名

1. 进入「域名管理」
2. 点击「添加域名」
3. 输入您的域名（例如：`yourdomain.com`）
4. 选择发送区域（推荐选择靠近您用户的区域）

### 配置 DNS 记录

系统会自动生成需要添加的 DNS 记录，并在控制台页面显示。请按照页面提示，在您的 DNS 提供商（如 Cloudflare、阿里云、腾讯云等）处添加这些记录。

<Callout type="info">
  **您需要添加的记录类型**：
  - **DKIM 记录**（通常 3 条）：用于邮件数字签名，证明邮件来源真实性
  - **SPF 记录**：指定哪些服务器被授权发送您域名的邮件
  - **DMARC 记录**：定义邮件认证失败时的处理策略
  - **MX 记录**（可选）：用于接收退信通知
</Callout>

### 等待验证完成

DNS 记录添加后，点击控制台的「验证」按钮。Z-Send 会自动检查您的 DNS 配置。

**验证通常需要 5-30 分钟，DNS 传播时间取决于您的 DNS 提供商。**

<Callout type="success">
  当控制台显示域名状态为 **`verified`** 时，表示验证成功，您可以开始使用该域名发送邮件了！
</Callout>

</Steps>

## DNS 记录说明

为了确保邮件能够顺利送达，Z-Send 使用以下标准邮件认证技术：

### DKIM（邮件数字签名）

DKIM 为每封邮件添加数字签名，证明邮件确实来自您的域名，防止邮件被篡改或伪造。通常需要添加 3 条 DKIM CNAME 记录。

**作用**：
- 防止邮件伪造
- 提高邮件送达率
- 保护您的品牌声誉

### SPF（发件服务器授权）

SPF 记录指定哪些服务器被授权代表您的域名发送邮件，防止恶意第三方冒用您的域名。

**作用**：
- 防止域名被冒用
- 降低垃圾邮件误判率

<Callout type="warning">
  如果您的域名已经在其他地方（如 Google Workspace、Microsoft 365）配置了 SPF 记录，请联系支持团队获取帮助，避免冲突。
</Callout>

### DMARC（邮件认证策略）

DMARC 定义当邮件认证（SPF 或 DKIM）失败时的处理策略，帮助您监控和保护域名免受滥用。

**作用**：
- 接收认证失败报告
- 防止钓鱼攻击
- 提高域名可信度

### MX 记录（可选）

MX 记录用于接收退信通知（bounce notifications），帮助您了解邮件发送失败的原因。

**作用**：
- 接收退信通知
- 及时发现发送问题

## 验证状态

域名可能处于以下状态：

| 状态 | 说明 |
|------|------|
| `pending` | 等待验证，DNS 记录尚未配置 |
| `verifying` | 正在验证 DNS 记录 |
| `verified` | 已验证，可以使用 |
| `failed` | 验证失败，请检查 DNS 配置 |

## 常见问题

### DNS 记录需要多长时间生效？

通常 5 分钟到 48 小时不等，取决于您的 DNS 提供商。建议等待 15-30 分钟后再尝试验证。

### 验证失败怎么办？

1. **等待更长时间**：DNS 传播通常需要 5-30 分钟，某些 DNS 提供商可能需要更长时间（最多 48 小时）
2. **检查 DNS 配置**：确认您已按照控制台显示的记录准确添加，注意不要有多余的空格或引号
3. **查看错误信息**：Z-Send 控制台会显示具体的验证错误，请根据提示进行修改
4. **联系支持**：如果持续验证失败，请联系 Zeabur 支持团队获取帮助

### 可以使用子域名吗？

可以！您可以验证子域名（如 `mail.yourdomain.com`），这样可以：
- 分离不同类型的邮件（营销 vs 交易邮件）
- 更好地控制域名声誉

### 验证成功的标志是什么？

当 Z-Send 控制台显示域名状态为 **`verified`** 时，表示所有 DNS 记录都已正确配置并通过验证。此时您可以立即使用该域名发送邮件。

## 域名删除

如果不再需要某个域名：

1. 进入「域名管理」
2. 找到要删除的域名
3. 点击「删除」按钮
4. 确认删除

<Callout type="warning">
  删除域名后，使用该域名发送邮件的 API 调用将失败。请确保没有正在使用该域名的应用。
</Callout>

## 最佳实践

### 使用专用子域名

推荐为邮件发送使用专用子域名（如 `mail.yourdomain.com`），这样可以：
- 保护主域名声誉
- 更容易追踪和管理邮件
- 分离不同类型的邮件（如营销邮件、交易邮件）

### 避免频繁更改 DNS

域名验证成功后，请勿随意修改或删除 DNS 记录，否则可能导致邮件发送失败或被标记为垃圾邮件。
