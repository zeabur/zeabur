# 创建模板

Zeabur 上的模板可以由任何人创建，它们可以帮助其他用户更轻松地创建自己的项目或服务。本指南将向您展示如何在 Zeabur 上创建模板。

## 如何创建模板

前往 [创建模板](https://dash.zeabur.com/create-template) 页面。您可以在这里创建一个模板，以便为其他用户提供一个单击部署您的项目或服务的方式。

![Create Template](/template/create/section-1.png)

### 步骤一：填写基本模板信息

在你开始添加服务和配置到模板之前，你需要填写你的模板的基本信息。

- **模板名称**：您的模板的名称。
- **模板描述**：您的模板的简要描述，推荐 10-30 个字。
- **图标 URL**：您的模板的图标图片的 URL。推荐 128x128 像素。
- **封面 URL**：您的模板的封面图片的 URL。
- **预览演示 URL**：您的模板的演示的 URL。

### 步骤二：变量和服务

你的模板由服务和变量组成。您可以向您的模板添加服务和变量。

![Section 2](/template/create/section-2.png)

#### 模板变量

你可以在这个部分添加一些 *项目级别* 的变量。

> 项目级别变量是所有服务共享的变量。

点击 **新建模板变量** 按钮，你将看到一个简单的表单填写。

![Variables](/template/create/variables.png)

- **变量问题**：当用户部署模板时，问题将显示给用户，引导他们填写值。
- **变量键**：变量的键，可以在服务中使用 `${{ key }}` 引用。
- **变量描述**：变量的描述，当用户部署模板时将显示给用户。

#### 模板服务

一个模板需要它的服务才能工作。你可以在这里添加一些来自 GiHub 仓库或预构建镜像的服务。

你也可以添加一些配置，比如服务变量，根目录等。

![Services](/template/create/services.png)

#### 模板标签

你也可以添加一些标签到你的模板，以帮助用户更容易地找到你的模板。

### 步骤三：README

为了让其他用户知道如何部署和使用模板，您可能需要一个更详细的指南。您可以在这里以 markdown 格式编写 README.md 文件。

![Readme](/template/create/section-3.png)

### 步骤四：提交您的模板

当所有 3 部分都填写完毕后，您可以点击 **创建模板** 按钮提交您的模板。

关于如何使用部署按钮分享此模板，您可以查看这个[指南](../deploy/deploy-button)。
