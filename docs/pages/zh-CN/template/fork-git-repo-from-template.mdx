# 从模板服务 Fork 代码

在 Zeabur 上，你可以使用模板服务快速部署应用。一个模板通常包含多个服务，每个服务要么是 Prebuilt 服务（从 Docker 镜像部署），要么是 Git 服务（从 GitHub 部署）。

默认情况下，当你部署包含 Git 服务的模板时，Zeabur 会直接从模板作者的 GitHub 仓库部署服务，并自动将 CI/CD 绑定到模板作者设置的仓库。这意味着当模板作者向其仓库推送新提交时，Zeabur 会自动触发 CI/CD 并将新版本部署到你的项目中。

但有时你希望基于模板作者的代码进行修改，添加自己的功能或更改配置等。为此，Zeabur 提供了一键 Fork 的便捷功能。

## 如何从模板服务 Fork 代码

以 [Zeabur 官方 Express.js 模板](https://zeabur.com/templates/W96I3Z) 为例，部署该模板后，进入 express 服务的设置页面。你会看到 Git Repository 字段当前值为 `https://github.com/zeabur/expressjs-template`，这是该模板指定的 GitHub 仓库。

若想基于此模板修改代码，可点击下方的 Fork Repository 按钮：

![Express.js github fork](/template/fork-git-repo-from-template/Expressjs_github_fork.png)

点击后会弹出窗口询问仓库名称。请使用尚未使用的仓库名称，否则 Fork 可能失败。名称下方还有一个输入框，可选择是否将仓库设为私有，以防止他人看到你的代码修改：

![git fork](/template/fork-git-repo-from-template/git_fork.png)

选好名称后点击 Fork。如果尚未授权 GitHub，可能会跳转到 GitHub 的 OAuth 授权页面。授权完成后会显示 Fork 加载界面：

![Configure Github](/template/fork-git-repo-from-template/Configure_Github.png)

![github connection](/template/fork-git-repo-from-template/github_connection.png)

完成后，你的 GitHub 账户中会新增一个与原代码相同的仓库。

Zeabur 会自动将该模板的代码 Fork 到你的 GitHub 账户，并将这个 Fork 后的仓库设置为你服务的 Git Repository。这意味着你在 GitHub 上修改这个 Fork 仓库并推送提交后，Zeabur 会自动触发 CI/CD 并将新版本部署到你的项目中。
