# 共享集群（已停止服务）

<Callout type="error">
共享集群已于 **2024 年停止服务**，所有新项目请使用[专用服务器](/dedicated-server)。
</Callout>

## 什么是共享集群

共享集群是 Zeabur 早期提供的服务器方案。在共享集群模式下，多位用户的服务会部署在同一组 Kubernetes 集群中，共享底层的 CPU、内存和网络资源。用户无需管理硬件，只需按实际资源使用量付费。

## 与专用服务器的差异

| 项目 | 共享集群（已停止） | 专用服务器 |
| --- | --- | --- |
| 资源隔离 | 多租户共用硬件 | 完整独享硬件 |
| 计费方式 | 按用量计费（vCPU / 内存 / 流量） | 按月固定费用 |
| 服务器位置 | 固定区域（AWS） | 自携主机（BYOH），区域不限 |
| 性能稳定性 | 可能受邻居影响 | 资源稳定可预测 |
| 适用场景 | 小型项目、开发测试 | 生产环境、高稳定性需求 |

## 停止服务的原因

随着专用服务器方案的成熟，Zeabur 决定将共享集群整合至专用服务器架构，提供更好的资源隔离与性能保障。共享集群的按量计费模式仍保留在[定价方案](/pricing/pricing-plans)中作为费率参考。

## 从共享集群迁移到专用服务器

如果你的服务仍在共享集群上运作，请依照以下步骤迁移至专用服务器：

1. **准备专用服务器** — 在 [Zeabur 服务器页面](https://zeabur.com/servers) 购买或[注册自己的服务器](/dedicated-server/register)。
2. **备份数据** — 使用[备份与还原](/operations/backup-restore)功能，为所有有状态服务（数据库、持久存储）创建备份。
3. **导出项目** — 通过[导出项目](/deploy/export-project)功能导出项目的 YAML 配置。
4. **在专用服务器上重新部署** — 在专用服务器的区域创建新项目，导入配置并重新部署服务。
5. **还原数据** — 将步骤 2 的备份还原到新服务中。
6. **切换 DNS** — 将自定义域名的 DNS 记录指向新服务的域名。
7. **验证与清理** — 确认新环境运作正常后，移除共享集群上的旧服务。

<Callout type="info">
迁移过程中如遇问题，请到 [社群论坛](https://zeabur.com/forum) 联系我们。
</Callout>
