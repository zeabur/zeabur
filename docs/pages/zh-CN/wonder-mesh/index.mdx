# Wonder Mesh

Wonder Mesh 让你可以将自己的计算机或服务器变成 Zeabur 计算节点。无论你的设备在家中、办公室还是任何网络环境中，都可以安全地连接到 Zeabur 平台来部署和运行服务——无需公网 IP 或端口转发。

<Callout type="info">
  Wonder Mesh 目前处于 **Beta** 阶段。你可以从 Zeabur 控制台的[服务器页面](https://zeabur.com/servers)申请访问权限。
  已经获得访问权限？请参阅[快速开始指南](/wonder-mesh/get-started)来设置你的第一台 Wonder Mesh 服务器。
</Callout>

## 工作原理

Wonder Mesh 使用 [WireGuard](https://www.wireguard.com/) 在你的设备和 Zeabur 平台之间创建一个私有的加密隧道。这个隧道可以穿越 NAT、防火墙和动态 IP——你的设备会获得一个稳定的内部 Mesh IP 地址，Zeabur 通过 SSH 使用该地址来管理部署。你无需自行配置任何网络设置。

## 为什么选择 Wonder Mesh？

通常，Zeabur 服务运行在由 Zeabur 管理的云服务器上。Wonder Mesh 改变了这一点，让你可以使用**自己的硬件**——笔记本电脑、台式机、家庭实验室服务器，甚至是放在家里的迷你电脑——作为部署目标，同时享有完整的 Zeabur 体验。

- **家庭实验室爱好者**：为你的家庭服务器获得云级别的部署和管理体验。
- **开发者**：使用真正的编排系统在本地测试服务，准备好后再部署到云端。
- **自托管倡导者**：将数据保留在自己的硬件上，同时不牺牲一键部署的便利性。

## 主要特性

- **一键设置**：在控制台中创建服务器，在设备上运行一条命令，即可开始部署。
- **穿越 NAT**：无需公网 IP、静态 IP 或端口转发。Wonder Mesh 建立了一个安全的网状网络，可在任何网络环境中工作。
- **完整的 Zeabur 集成**：从模板市场部署模板、推送代码、绑定域名、查看日志和监控资源——与任何其他 Zeabur 服务器完全一样。
- **服务器管理**：从控制台监控 CPU、内存和磁盘使用情况，访问 SSH，并查看实时服务器状态。

## 支持的平台

| 操作系统 | 架构 | 备注 |
|---------|------|------|
| Linux | amd64 / arm64 | 直接安装 |
| macOS | amd64 / arm64 | 通过 Lima 虚拟机运行 |

从 Mac mini 到树莓派，任何 Linux 或 macOS 设备都可以成为 Zeabur 服务器。

## 常见问题

### 我的设备需要一直开机吗？

是的。设备必须保持开机并连接到互联网，才能让部署在其上的服务保持可访问。

### 如果我的设备离线了会怎样？

部署在该设备上的服务将暂时不可用。一旦设备重新上线，连接将自动恢复。

### 有什么网络要求吗？

只需要互联网连接即可。不需要公网 IP、端口转发或防火墙更改。

### 如何访问部署在我设备上的服务？

你有两种选择：

- **从互联网访问**：像其他 Zeabur 服务器一样为你的服务绑定域名。有关域名绑定和端口转发的详情，请参阅[公网存取](/networking/public)。
- **从局域网访问**：在服务页面打开 **Networking** 标签页，找到局域网访问地址。有关查找设备局域网 IP 的详情，请参阅[快速开始指南](/wonder-mesh/get-started#accessing-services)。

同一项目中的服务也可以使用内部主机名相互通信。详情请参阅[服务内网](/networking/private)。
