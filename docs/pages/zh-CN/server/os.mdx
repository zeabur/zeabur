# 操作系统

当你[从 Zeabur 购买服务器](/zh-CN/server/purchase)或重装现有服务器时，可以选择它运行的操作系统。这个选择决定了机器从一开始的工作方式：

- **ZeaburOS**（推荐）— 机器成为 Zeabur 部署节点，开箱即可部署项目。
- **标准发行版** — 目前提供 Ubuntu 24.04 LTS，未来会开放更多操作系统与版本——一台干净、完全由你自行管理的机器。

## 什么是 ZeaburOS [#zeaburos]

ZeaburOS 是一套将服务器变成 **Zeabur 部署节点**的操作系统。运行 ZeaburOS 的机器会连接到 Zeabur Dashboard，你可以在其中部署服务、查看日志与性能指标、管理存储与网络设置——无需通过 SSH 操作主机。

在底层，ZeaburOS 以 [K3s](https://k3s.io/)（轻量级 Kubernetes）为核心运行，并搭配 Zeabur 系统组件（部署代理、监控等），让机器与平台保持连接。

相比运行标准发行版，ZeaburOS 为你带来：

- **开箱即部署** — 无需设置运行环境、反向代理或证书；推送项目即可运行。
- **内置可观测性** — 在控制台中查看实时日志、性能指标与服务状态。
- **托管维护** — 系统组件可一键更新或修复，并由 Zeabur 提供技术支持。

<Callout type="info">
  ZeaburOS 过去在控制台中显示为「K3s」。引擎不变，名字更清楚——详见下方 [K3s 现在是 ZeaburOS](#k3s-is-now-zeaburos)。
</Callout>

## 标准发行版

选择标准发行版，你会得到一台干净的机器，与发行版原始发布的状态完全一致——就像任何供应商提供的 VPS。你完全通过 SSH 操作它：机器上运行什么、如何设置、如何维护都由你决定。Zeabur 提供机器本身，机器上的软件完全由你自行管理。

Ubuntu 24.04 LTS 现已推出，未来会开放更多操作系统与版本选择。

## 如何选择

|                                          | ZeaburOS | 标准发行版 |
| :--------------------------------------- | :------- | :-------------------- |
| 通过 Zeabur Dashboard 部署与管理服务 | ✅       | ❌                    |
| 内置日志、性能指标与监控   | ✅       | ❌                    |
| SSH 访问                               | ✅       | ✅                    |
| 对操作系统的完整控制                 | 由 Zeabur 管理 | 完全由你掌控  |
| Zeabur 技术支持                 | ✅       | ❌（自行管理）     |

**经验法则：**如果你想用 Zeabur 部署项目，选 ZeaburOS——这正是这台机器的用途；如果你想要一台完全由自己管理的干净机器，选标准发行版。

无论怎么选都不会被锁定：运行标准发行版的机器之后可以[转换为 ZeaburOS](#convert-to-zeaburos) 而无需重装，运行 ZeaburOS 的机器也可以通过重装操作系统回到干净状态。

## 之后切换

### 将现有机器转换为 ZeaburOS [#convert-to-zeaburos]

已在运行标准发行版的机器可以转换为 ZeaburOS，使其成为部署节点。转换是**在现有系统上就地升级：免重装、数据保留**。

要进行转换，打开服务器的**设置**页面，使用**转换为 ZeaburOS**。设置完成后，机器会显示为部署节点，你就可以开始向它部署服务。

<Callout type="default">
  没有 ZeaburOS 时，你仍然可以通过 SSH 连接机器——只有需要通过 Zeabur 部署和管理服务时才需要转换。
</Callout>

### 更新 ZeaburOS

**更新 ZeaburOS** 会将服务器上的所有 ZeaburOS 系统组件（部署代理、监控等）重新套用至最新状态。已部署的服务不受影响，服务异常时也可用来作为修复服务器的第一步。

你可以从服务器的菜单或其**设置**页面触发更新。此操作通常安全，但可能造成短暂停机。

### 卸载 ZeaburOS

若要从机器上卸载 ZeaburOS，在服务器设置页面的**危险区域**中重装操作系统即可，机器会回到出厂状态。重装时你可以重新选择操作系统：选标准发行版得到一台干净机器，选 ZeaburOS 则重新成为部署节点。

<Callout type="warning">
  重装操作系统会清除机器上的所有内容，包括已部署的服务及其数据。请先备份所有需要的资料。
</Callout>

## 支持范围

Zeabur 仅对 **ZeaburOS** 提供技术支持；自行安装的软件（包含 1Panel、宝塔面板等管理面板）不在支持范围内。

选择标准发行版时，机器完全由你自行管理：Zeabur 提供机器本身，机器上运行的一切都由你负责。

## K3s 现在是 ZeaburOS [#k3s-is-now-zeaburos]

Zeabur 控制台中原本显示为 **「K3s」** 的一切现在都显示为 **ZeaburOS**。引擎不变，名字更清楚：ZeaburOS 以 K3s（轻量级 Kubernetes）为核心运行，你现有的机器没有任何变化——原本显示为「K3s」的状态就是同一个东西，只是换上了产品名称。

伴随更名，有两处新变化：

- 购买机器或重装操作系统时，ZeaburOS 不再自动预装——你需要**选择**它（或标准发行版）作为操作系统。
- 未安装 ZeaburOS 的机器可以随时从服务器的设置页面进行转换。

对于主机层面的维护，底层技术名称保持不变：systemd 服务仍然是 `k3s`，持久存储空间仍位于 `/var/lib/rancher/k3s/storage`，[管理服务器](/zh-CN/server/operate)中的命令也照常可用。
