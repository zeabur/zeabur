# 共享集群（已停止服务）

<Callout type="error">
共享集群已于 **2026 年停止服务**，所有新项目请使用[服务器](/server)。
</Callout>

## 什么是共享集群

共享集群是 Zeabur 早期提供的服务器方案。在共享集群模式下，多位用户的服务会部署在同一组 Kubernetes 集群中，共享底层的 CPU、内存和网络资源。用户无需管理硬件，只需按实际资源使用量付费。

## 与服务器的差异

| 项目 | 共享集群（已停止） | 服务器 |
| --- | --- | --- |
| 资源隔离 | 多租户共用硬件 | 完整独享硬件 |
| 计费方式 | 按用量计费（vCPU / 内存 / 流量） | 按月固定费用 |
| 服务器位置 | 固定区域（AWS） | 自携主机（BYOH），区域不限 |
| 性能稳定性 | 可能受邻居影响 | 资源稳定可预测 |
| 适用场景 | 小型项目、开发测试 | 生产环境、高稳定性需求 |

## 停止服务的原因

随着服务器方案的成熟，Zeabur 决定将共享集群整合至服务器架构，提供更好的资源隔离与性能保障。

## 一定要迁移吗？

**共享集群上的既有服务会继续运行。** 目前「停止服务」的含义是不能再添加*新的*服务——已部署的服务不会被关闭，也没有公告既有服务的统一关闭期限。

Zeabur 只会在特定情况下主动联系项目所有者要求迁移：

- 项目资源用量长期偏高，影响共享集群上的其他用户
- 触发了滥用检测（例如爬虫、挖矿）
- 工作负载需要共享集群无法提供的能力（对外固定 IP、高带宽、定制网络）

如果没有收到我们的主动联系，迁移是**可选的**——不过迁到[专属服务器](/server)仍有实际好处：没有平台层速率限制、性能完全独立、不受共享租户的限制。

## 使用 Clone Project 迁移（推荐）

最简单的迁移方式是内置的 **Clone Project** 功能，它会把整个项目——服务、环境变量与配置——复制到你选择的区域：

1. **准备服务器** — 在 [Zeabur 服务器页面](https://zeabur.com/servers) 购买或[注册自己的服务器](/server/register)。
2. 在旧项目中前往 **Settings → General → Clone Project**。
3. 在 **Target Region** 选择你服务器所在的区域。
4. 设置克隆选项——可以让旧项目在克隆期间继续运行，并保留服务分组与排序。
5. 确认后等待克隆完成，逐一检查新项目中的每个服务都正常启动与运行。
6. **切换 DNS / 域名** — 把自定义域名重新绑定到新服务，并更新 DNS 记录。
7. 流量完全由新项目承接后，**删除旧项目**。

![Clone Project 对话框](/server/clone-project-dialog.png)

<Callout type="warning">
克隆过程可能产生数分钟到数十分钟不等的停机时间（取决于数据量），除非启用「旧项目继续运行」选项。Volume 数据与其他有状态存储在克隆后务必验证——数据库请先做好[备份](/operations/data/backup-restore)。
</Callout>

## 手动迁移（替代方案）

如果你想重建而不是克隆：

1. **备份数据** — 使用[备份与还原](/operations/data/backup-restore)功能，为所有有状态服务（数据库、持久存储）创建备份。
2. **导出项目** — 通过[导出项目](/deploy/manage/export-project)功能导出项目的 YAML 配置。
3. **在服务器上重新部署** — 在服务器的区域创建新项目，导入配置并重新部署服务。
4. **还原数据** — 将步骤 1 的备份还原到新服务中。
5. **切换 DNS** — 将自定义域名的 DNS 记录指向新服务的域名。
6. **验证与清理** — 确认新环境运行正常后，移除共享集群上的旧服务。

## 常见问题

**我必须在某个期限前迁移吗？**
既有服务目前没有公告统一期限。只有在 Zeabur 针对你的项目主动联系时才必须处理。不过自 2026 年 4 月 1 日起，共享集群已无法创建新服务。

**我现有的项目会停止运行吗？**
不会。既有服务照常运行。「停止服务」只是不能添加新服务，不会停掉正在运行的服务。

**Clone 会迁移我的 Volume 数据吗？**
数据库内容与 Volume 数据在克隆后需要验证——迁移前请务必先创建[备份](/operations/data/backup-restore)，如有缺漏再手动还原。

<Callout type="info">
迁移过程中如遇问题，请到 [社群论坛](https://zeabur.com/forum) 联系我们。
</Callout>
