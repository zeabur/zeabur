# 快速开始

本指南将带你在 **5 分钟内完成第一次部署**。如果你不确定从哪里开始，请查看[按需求开始](../start-by-need)找到最适合你的路径。

## 前提条件

| 项目 | 说明 |
|------|------|
| **Zeabur 账户** | 在 [zeabur.com](https://zeabur.com) 使用 Email、GitHub 或 Google 注册。创建第一个项目时需要验证账户（手机号、预存额度或信用卡）。 |
| **专用服务器**（可选） | 如果需要特定的数据中心位置，请先[注册专用服务器](/dedicated-server)。使用共享集群则可跳过。 |

## 路径 1：从模板部署（最快 30 秒）

Zeabur 提供数百个预配置模板，涵盖热门开源应用和数据库。

<Callout>
我们建议新用户从模板开始，体验完整的部署上线流程。
</Callout>

**热门模板**

| 用途 | 推荐 | 部署 |
|------|------|------|
| 博客 | WordPress, Ghost | [模板市场](https://zeabur.com/templates) |
| 自动化 | n8n, Activepieces | [模板市场](https://zeabur.com/templates) |
| 数据库 | PostgreSQL, MySQL, Redis | 在项目中直接添加 |
| AI 应用 | Dify, LobeChat | [模板市场](https://zeabur.com/templates) |

1. 在项目中，点击 **Add Service** → **Template**。
2. 搜索或浏览你想要的模板，然后点击部署。
3. 等待约 30 秒——你的服务就上线了。

![模板列表](/start/template-services.webp)

## 路径 2：从 GitHub 部署代码

最常见的方式是通过 GitHub 部署你自己的项目。

1. 在项目中，点击 **Add Service** → **GitHub**。
2. 首次使用时授权 GitHub 账户，然后等待约 1 分钟。
3. 搜索并选择要部署的仓库。

![搜索仓库](/start/search-repository.webp)

Zeabur 自动检测语言和框架，选择最佳构建策略。

![构建方案](/start/build-plan.webp)

点击 **Deploy** 开始。如需调整环境变量，前往"Configuration"标签。

## 路径 3：使用 AI 助手部署

不确定该做什么？直接用自然语言告诉 Zeabur 的 AI 助手你的需求。

![Zeabur Agent](/start/zeabur-agent.webp)

点击导航栏中的 **Agent** 开始对话。例如：

- "帮我部署一个 PostgreSQL 数据库"
- "我想搭建一个 WordPress 博客"
- "从 GitHub 部署我的 next-app 仓库"

## 绑定域名上线

服务部署完成后，前往 **Domains** 标签绑定域名。

![绑定域名](/start/bind-domain.webp)

- Zeabur 提供免费的 `.zeabur.app` 子域名用于快速测试。
- 你也可以绑定自己的自定义域名——Zeabur 自动处理 HTTPS 证书。
- 详细设置请参见[域名绑定指南](/deploy/domain-binding)。
