# 暂停服务

Zeabur 允许你直接从仪表板暂停（停止）个别服务。被暂停的服务会停止运行、停止消耗资源、停止产生费用——同时保留所有设置，方便你随时恢复。

## 暂停服务

暂停服务的步骤：

1. 在 Zeabur 仪表板中打开你的服务。
2. 前往 **Settings**。
3. 点击 **Suspend Service**。

服务会立即停止。暂停期间：

- 服务容器会被关闭，不再运行。
- 该服务**不会产生运算费用**。
- 所有设置都会保留，包括环境变量、域名绑定和存储空间挂载。
- [存储空间](/operations/volumes)中的数据会保留。

## 恢复服务

恢复已暂停服务的步骤：

1. 在仪表板中打开已暂停的服务。
2. 前往 **Overview**。
3. 点击 **Restart Service**。

Zeabur 会使用最新的设置启动一个新的部署。服务会经过正常的[健康检查](/operations/health-checks)流程，并在就绪后开始接收流量。

## 使用场景

### 暂时性维护

当你需要对相关基础设施进行维护（例如数据库迁移），并希望确保在维护期间没有流量到达应用程序时，可以暂停服务。

### 节省费用

如果你的服务只在特定时段或特定任务中需要使用（例如预备环境或批处理服务），在不使用时暂停它们可以降低费用。

### 调试

暂停出现异常的服务，使其停止处理请求，同时你进行调查。找到并修复问题后，重新部署并恢复服务。

## 暂停与删除的比较

| | 暂停 | 删除 |
| --- | --- | --- |
| 服务停止运行 | 是 | 是 |
| 设置保留 | 是 | 否（宽限期过后） |
| 存储空间保留 | 是 | 否（宽限期过后） |
| 可以恢复 | 是，立即可用 | 有限时间内可联系客服 |
| 计费 | 停止 | 停止 |

<Callout type="warning">
  删除服务是一个破坏性操作。删除后有短暂的宽限期，在此期间你可以联系客服尝试恢复服务。宽限期过后，所有数据——包括存储空间——都会被永久移除。如果你只需要暂时停止服务，请使用**暂停**。
</Callout>

## 已暂停的服务与域名

当服务被暂停时，绑定到该服务的任何域名都会显示错误页面。如果你需要显示自定义的维护页面，请考虑使用另一个轻量级服务（例如静态网站）来提供维护通知，并在维护期间将域名的 DNS 指向该服务。
