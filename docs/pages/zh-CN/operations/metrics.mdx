# 监控指标

Zeabur 为每个服务提供实时的资源使用指标，帮助你监控性能并做出明智的扩展决策。

## 查看指标

查看服务指标的步骤：

1. 在 Zeabur 仪表板中打开你的项目。
2. 选取你要监控的服务。
3. 点击 **Metrics** 分页。

指标仪表板会实时显示以下图表：

- **CPU 使用率** — 当前及历史 CPU 消耗量。
- **内存使用量** — 当前及历史 RAM 消耗量，包括已设定的上限。
- **网络流量** — 输入和输出的数据传输量。
- **磁盘使用量** — 即将推出（Coming soon）。

## 了解资源使用情况

### CPU

CPU 使用率以已分配 CPU 资源的百分比显示。CPU 使用率飙升可能表示：

- 请求量过高
- 运算密集的操作
- 可以优化的低效代码路径

### 内存

内存使用量显示你的服务相对于设定上限所消耗的 RAM。如果你的服务持续接近内存上限，请考虑在[服务设置](/operations/scaling)中增加分配量，以避免内存不足（OOM）导致的重新启动。

### 网络

网络指标显示服务传入和传出的数据量。这有助于了解流量模式及估算带宽成本。

## 服务用量与计费

如需详细的资源消耗及相关费用明细，请前往项目设置中的 **Service Usage** 页面。此页面会显示：

- 计费周期内的累计 CPU 和内存使用量
- 网络出口流量费用
- 各服务的费用明细

详情请参阅 [Service Usage](/operations/service-usage)，了解用量如何对应计费。

<Callout>
  Zeabur 采用以用量计费的定价模式。定期监控你的指标有助于掌握资源消耗情况，避免意外费用。详细定价信息请参阅[方案价格](/pricing)。
</Callout>

## 利用指标引导扩展

指标是决定何时以及如何扩展的关键依据：

- **CPU 持续超过 80%** — 考虑增加 CPU 分配或优化你的应用程序。
- **内存接近上限** — 增加内存分配以防止 OOM 崩溃。
- **流量稳定增长** — 提前规划，在使用量飙升导致性能下降之前调整资源。

如需调整服务资源的相关信息，请参阅[扩展](/operations/scaling)。
