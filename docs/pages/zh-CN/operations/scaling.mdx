# 扩展

Zeabur 允许你调整分配给服务的资源，使其能够应对不同的工作负载。你可以通过增加 CPU 和内存进行垂直扩展，并选择适合需求的基础设施层级。

## 垂直扩展

垂直扩展是指增加（或减少）分配给单一服务实例的 CPU 和内存。

调整资源的步骤：

1. 在 Zeabur 仪表板中打开你的服务。
2. 前往 **Settings**。
3. 在 **Resources** 区段中，设定所需的 CPU 和内存上限。
4. 保存变更。新的资源分配将在下次部署时生效。

### 资源方案

Zeabur 提供多种资源层级。默认分配适用于大多数轻量级服务，但你可以为更高需求的工作负载（如数据库、构建服务器或 AI 推理）增加资源。

<Callout>
  资源分配会直接影响计费。较高的 CPU 和内存上限会产生较高的费用。详细的单位定价请参阅[方案价格](/pricing)。
</Callout>

## 服务器

对于需要保证资源、可预测性能或符合数据驻留要求的工作负载，Zeabur 提供**服务器**。服务器为你提供不与其他用户共享的隔离运算环境。

服务器的优势包括：

- **保证资源** — 不受邻居干扰（noisy neighbor）问题影响。
- **自定区域** — 部署在特定的地理位置。
- **更高的资源上限** — 运行超出共享集群限制的服务。

设置说明和可用配置请参阅[服务器](/dedicated-server)。

## 扩展策略

| 场景 | 建议 |
| --- | --- |
| 偶发的流量高峰 | 增加 CPU / 内存以应对尖峰负载 |
| 持续性高流量 | 使用服务器以获得可预测的性能 |
| 成本优化 | 根据[监控指标](/operations/metrics)合理调整资源 |
| 暂时闲置 | [暂停](/operations/deployment/suspend-service)服务以停止计费 |

## 监控资源使用量

使用[监控指标](/operations/metrics)仪表板实时监控 CPU、内存和网络使用量。这些数据有助于你判断何时需要扩展或缩减。如需费用分析，请查看 [Service Usage](/operations/service-usage) 页面。
