# 部署 Vapor 项目

[Vapor](https://vapor.codes) 是 Swift 生态中较为知名的 Web 框架，藉由 Vapor 的帮助，你可以使用 Swift 来开发一个基于 Http 的 Web 应用程序。

本文将介绍如何将你的 Vapor 项目部署到 Zeabur。

## 事前准备

请确保你的电脑已经准备好 Swift 和 Vapor 的开发环境：

```bash
brew install vapor
```

## 建立 Vapor 项目

有两个方法可以建立 Vapor 项目：

### 使用 Vapor 命令工具

```bash
vapor new <project-name>
cd <project-name>
vapor xcode
```

使用 Vapor 命令新建的项目，预设带有 `Dockerfile` 及 `docker-compose.yml`。好消息是，你不需要 Dockerfile 也能直接在 Zeabur 部署，所以你可以直接删除这两个文件：

```bash
rm Dockerfile docker-compose.yml
```

### 直接使用 Zeabur 为 Vapor 准备的模板

打开 https://github.com/zeabur/vapor-template ，点击 `Use this template` 按钮，输入项目名称后，点击 `Create repository from template` 按钮。

```bash
git clone <your-repository-url>
cd <project-name>
vapor xcode
```

## 开发

在开发时，你可以直接在本机执行项目：

```bash
vapor run
```

接下来，根据你的需求修改 Swift 程序代码。

## 部署

首先，确保你的所有改动都已经同步到 GitHub 上：

```bash
git add .
git commit -m "your commit message"
git push
```

接下来，前往 [Zeabur 控制台](https://zeabur.com) 并建立一个新项目。

然后，点击 `Deploy New Service` 按钮，选择从 GitHub 部署。

<Callout>
    如果你是第一次在 Zeabur 上从 GitHub 部署项目，你需要先配置并安装 GitHub App。
    你可以在 [这里](/deploy/github) 查看更详细的指南。
</Callout>

当你选择好储存库和分支后，Zeabur 将会自动开始建置你的服务。

Zeabur 会自动检测到你的服务是由 Vapor 建置的，所以你不需要做任何额外的配置。你的部署将会在几分钟内完成。

## 下一步

部署完成后，你可能需要为你的 Vapor 网站配置域名。

打开服务页面的 "Domain" 分页，然后点击 "Generate Domain" 或 "Custom Domain"。

关于如何为你的服务绑定域名的更多信息，请参考 [绑定域名](/deploy/domain-binding)。
