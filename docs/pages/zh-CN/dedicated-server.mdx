# 独立服务器

你可以使用 Zeabur 方便地将各种服务部署到两种不同类型的服务器上：共享集群（Shared Cluster）和独立服务器（Dedicated Server），两种类型的计算资源适合不同的使用情境。

<Callout type="warning">
  Zeabur 正在逐步淘汰共享集群。共享集群已不再接受新项目，现有共享集群项目将于 2026 年 4 月 1 日起停止接受新服务。现有服务将继续运行。详情请参阅 [changelog](https://zeabur.com/changelogs/phasing-out-shared-cluster)。
</Callout>

## 共享集群

共享集群是最简单使用 Zeabur 部署服务的方式，因为你无需理解任何服务器相关的知识和设置，甚至感知不到服务器的概念，仅需要选择适合的机房位置即可。

每个共享集群都是由 Zeabur 管理的多台服务器所组合而成的计算集群，一个集群中包含多台服务器，你部署的服务会和其他选择同个集群的用户所部署的服务共享这些服务器。

共享集群适合用于部署硬件需求波动较大的服务，因为它采用按量计费的方式，在硬件需求低的时候可以节省成本，而在需求高的时候会自动扩容，因此不会发生硬件资源不足的情况。

目前 Zeabur 已有以下几个共享集群：

- 美国加州（由 Amazon Web Services 提供）
- 德国法兰克福（由 Amazon Web Services 提供）
- 日本东京（由 Amazon Web Services 提供）
- 香港（由 Amazon Web Services 提供）
- 台北（由 Google Cloud 提供）—— 仅提供给企业客户使用

## 独立服务器

独立服务器是只属于你一个人的服务器，因此服务不会受到其他用户服务的影响，硬件资源也完全由你独享，不会有被其他用户占用的情况。

独立服务器适合以下几种使用情境：

- 已在其他云端厂商租用或购买服务器的用户，可以一键将服务器注册到 Zeabur 上，如此一来就不需要支付硬件资源的使用费用。
- 对服务稳定性及安全性有较高要求的用户，例如大型系统的生产环境。
- 服务主要访问地区尚未有共享集群区域的用户，可以租用或自建符合自己地理位置的独立服务器。

你可以在一台独立服务器上建立多个项目，但你需要自行注意这些服务的硬件资源使用情况，避免因为独立服务器的硬件资源不足导致服务受到影响。

## 注册独立服务器

你可以选择在 Zeabur 上购买一台独立服务器，或者是注册一台你已经在其他云服务商租用或购买的服务器。

如果你想了解如何注册自己的服务器，可以参考[注册独立服务器](/dedicated-server/register)。

如果你想要从 Zeabur 购买独立服务器的话，可以参考[购买服务器](/dedicated-server/purchase)了解完整购买流程。

## 通过 SSH 连接

对于从 Zeabur 购买的独立服务器，你可以按照以下步骤通过 SSH 连接：

### 步骤一：显示 SSH 密码

前往服务器信息页面，点击 SSH 密码字段以显示密码。

![显示 SSH 密码](/dedicated-server/ssh-step-1.png)

### 步骤二：使用 SSH 连接

显示密码后，使用服务器信息页面上显示的 IP 地址、SSH 用户名和 SSH 密码通过 SSH 连接。

![SSH 连接详情](/dedicated-server/ssh-step-2.png)

你可以使用以下命令在终端中连接：

```bash
ssh <用户名>@<IP地址>
```

然后在提示时输入密码。

<Callout type="warning">
  一旦你更改了 SSH 密码，Zeabur 将无法再为此服务器提供技术支持。请谨慎操作。
</Callout>

## 常见问题

### 使用专用服务器需要订阅方案吗？

不需要，你不需要订阅方案即可使用专用服务器。不过，我们建议订阅开发者方案以发挥 Zeabur 的完整潜力——备份等功能仅提供给开发者方案的订阅者。开发者方案用户还可获得来自工作人员的优先支持，而免费用户可以通过 [Zeabur 论坛](https://zeabur.com/forum)获得帮助，但没有保证的回复时间。详情请参阅[订阅方案](/billing/plans)。

### 我只想架设小型网站，需要同时购买方案和服务器吗？

你可以在没有订阅方案的情况下，在专用服务器上部署服务。不过，备份等特定功能仅限开发者方案的用户使用。如果你想完整使用 Zeabur 的平台功能和优先支持，我们建议在服务器之外订阅开发者方案。更多信息请参阅[社区与支持](/community/forum)。
