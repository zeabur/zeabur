# 复制项目

Zeabur 提供**项目复制功能（Project Copy）**，让您可以轻松地将项目复制到任何地区。

您只需要前往项目设置页面，就能找到**一键项目复制功能**。这项功能会自动将您的项目完整复制到您选择的任何地区，包含项目架构、环境变量，甚至是您的硬盘数据。整个过程只需短短几分钟，大幅节省您的时间与心力。

- **无硬盘项目：** 如果您的项目没有挂载硬盘，复制过程甚至能做到**零停机**，体验就像重新启动服务一样流畅。
- **有硬盘项目：** 如果您的项目有挂载硬盘，复制时间会根据数据量大小而定。我们后台会自动将您的硬盘数据备份至 S3 存储空间，再还原到新的项目中，确保您的数据完整无虞。

## 复制内容说明

<Callout>
    项目复制功能会完整复制您的项目，**包括所有数据**。复制后需要重新绑定域名，不会自动继承原项目的域名设置。
</Callout>

复制项目时，以下内容会被完整复制：
- **项目架构**：所有服务的配置和关系
- **服务类型**：Docker 镜像、Git 仓库等
- **环境变量**：所有配置的环境变量
- **硬盘数据**：硬盘中的所有数据
- **网络设置**：端口配置和网络规则

## 注意事项

1. **零停机复制**：无硬盘的项目可以实现零停机复制，有硬盘的项目复制时间取决于数据量大小。

2. **数据完整性**：复制过程中所有数据都会被完整备份和还原，确保数据安全。

3. **域名重新绑定**：复制后的项目需要重新绑定域名，不会自动继承原项目的域名设置。

4. **资源消耗**：复制过程中会临时消耗额外的存储空间用于数据备份。
