# 管理成员

只有当团队中有合适权限的同事时，团队才真正发挥作用。本页是关于各角色能做什么、不能做什么、如何邀请与移除成员，以及团队成员关系与项目级或服务器级协作有何不同的权威参考。

如果你还在考虑是否需要团队，请先阅读[团队](/team)和[创建团队](/team/create-team)。

## 角色一览

| 角色 | 管理成员与团队资料 | 管理账单 | 创建 / 修改资源 | 删除资源 |
| --- | :---: | :---: | :---: | :---: |
| **Administrator** | 是 | 是 | 是 | 是 |
| **Editor** | 否 | 否 | 是 | 否 |
| **Viewer** | 否 | 否 | 否 | 否 |

读取权限是分级的：任意角色都可以查看团队的项目、服务、部署历史、服务器状态，并查看运行日志。**搜索或下载日志仅限 Editor 和 Administrator。** 发票、付款方式与账单详情仅对 **administrator** 可见。

创建团队的用户会被记录为**团队创建者**（在控制台中显示为 "Creator"）。创建者始终是管理员，且不能被任何其他人移除或降级——只有创建者本人可以通过[转移所有权](/team/transfer-ownership)主动卸任。每个团队必须始终至少有一位管理员。

### 详细权限矩阵

完整的按功能权限拆分如下：

| 功能 | Viewer | Editor | Administrator |
| --- | :---: | :---: | :---: |
| **Team 管理** | | | |
| 查看团队成员 | 是 | 是 | 是 |

### Administrator 可以做什么

- 更新团队资料：名称、Logo、用于接收系统通知的联系邮箱。
- 邀请、移除并修改其他成员的角色。
- 激活、取消或恢复 Team 方案。
- 管理付款方式、账单信息（法人名称、地址、税号）以及预付额度。
- 创建、修改**并**删除团队所有的项目、服务、环境、数据库与专用服务器。
- 将个人项目或服务器转入团队（参阅[转移所有权](/team/transfer-ownership)）。
- 在团队没有任何剩余项目或服务器时删除团队本身。

### Editor 可以做什么

Editor 是日常的"开发与发布"角色。

- 创建并修改团队所有的项目、服务、环境、变量与域名。
- 触发部署、回滚到先前的部署、重启服务。
- 搜索并下载日志。
- 添加或修改归属于团队的专用服务器与数据库。

Editor **不能**做的事：

- 邀请、移除或修改任何成员的角色。
- 编辑团队名称、Logo 或联系邮箱。
- 查看发票、修改付款方式、激活或取消 Team 方案，或充值余额。
- 删除项目、服务、数据库或服务器。Editor 只能创建与修改；只有管理员才能删除。

<Callout type="info">
  Editor 的写操作以 Team 方案处于激活状态为前提。当团队处于[团队总览](/team#激活-team-方案)中描述的未激活状态时，Editor 无法创建或修改资源——只有管理员才能完成激活步骤。
</Callout>

### Viewer 可以做什么

Viewer 对团队工作区中的大部分内容拥有只读访问权限。适用于利益相关者、审计人员、长期签约的外部合作方，或任何需要查看部署但不应修改任何内容的人。

Viewer **可以**：

- 查看项目、服务、部署历史，以及服务器 / 数据库状态。
- 打开运行日志和构建日志（只读）。
- 切换到团队工作区，并像其他成员一样浏览。

Viewer **不能**：

- 部署、重启、回滚或修改服务设置。
- 创建或修改项目、服务、环境变量、数据库或服务器。
- 搜索或下载日志（只允许在控制台中查看）。
- 邀请、移除或修改成员角色。
- 查看发票、付款方式或账单详情，也不能修改账单中的任何内容。

## 邀请成员

只有管理员可以邀请。被邀请人必须已经在 Zeabur 上拥有一个使用相同邮箱的账号——这里没有"待处理邀请"的收件箱；一旦你将其添加，他们立即成为正式成员，并可在下次登录时通过工作区切换器进入团队工作区。

<Steps>
### 切换到团队工作区

在控制台侧边栏的工作区切换器中，选择你要邀请人加入的团队。你必须处于团队工作区才能看到**成员**页面；个人工作区下不会暴露成员管理。

### 打开成员页面

进入**账号 → 成员**。管理员会在成员列表的右上角看到 **Invite a member** 按钮。Editor 和 Viewer 看不到此按钮。

### 输入邮箱并选择角色

点击 **Invite a member**，输入同事的邮箱地址，并选择 **Administrator**、**Editor** 或 **Viewer**。角色之后可以再调整，所以可以先把成员加为 Viewer，等你信任其工作流后再升权。

### 确认席位费用预览

如果你的团队订阅了 Team 方案，并且新增此成员会超出已含席位数，控制台会在发送邀请前显示额外席位费用预览。剩余账单周期内的按比例费用会加到下一张发票上。席位计费的工作方式请参阅[团队账单](/team/billing)。

### 发送

点击 **Send invitation**。新成员会立刻以所选角色出现在列表中。他们下次刷新控制台时，就能在工作区切换器中看到该团队。
</Steps>

如果你输入的邮箱与 Zeabur 上已有的账号不匹配，邀请会被拒绝——请让同事先到 [zeabur.com](https://zeabur.com) 注册账号，再重试邀请。

## 修改成员角色

管理员可以随时修改任何成员的角色，但有一个例外：**没有人**（即便是另一位管理员）可以修改**创建者**的角色。如果你希望更换实际管理团队的人，请改用[转移所有权](/team/transfer-ownership)。

要修改角色，进入**账号 → 成员**，点击该成员旁的角色下拉菜单，选择新角色。修改会在该成员下次加载页面时立即生效——无需"保存"步骤。

你不能在此页面修改自己的角色。要把自己从团队中移除，请使用 **Leave team**（见下文）。

## 移除成员

管理员可以移除创建者以外的任何成员。移除一名成员后：

- 立即生效。被移除的用户失去对所有团队所有的项目、服务、服务器及账单数据的访问权限。
- **不会**删除其个人项目或服务器——这些仍保留在他们的个人工作区中。
- 释放一个 Team 方案席位，并在下一张发票上减少相应的席位费用。

要移除某人，进入**账号 → 成员**，在其行上点击 **Remove**，然后确认。成员被移除时不会再发出额外通知。

### 自己离开团队

任何成员都可以离开自己所属的团队，但创建者不能。在团队工作区下进入**账号 → 团队设置**，并使用 danger zone 中的 **Leave team**。离开后，你将失去对该团队资源的访问权限，控制台会回到你的个人工作区。

如果你是创建者并希望离开，必须先[转移所有权](/team/transfer-ownership)给另一位管理员。

## 团队成员 vs 项目 / 服务器协作者

Zeabur 有三套相互独立的邀请系统。选对系统很重要，因为它们授予的访问范围不同，管理入口也不同。

| | 在哪里邀请 | 对方获得什么 | 谁能移除 |
| --- | --- | --- | --- |
| **Team member** | 在团队工作区下的**账号 → 成员** | 当前及未来所有团队所有的项目、服务和服务器，按你分配的角色受限。Administrator 还包含账单访问权限。 | 由管理员移除；成员可主动离开。 |
| **Project collaborator** | 在单一项目内的项目设置面板中 | 单一项目的服务、部署与变量。**不会**授予对底层专用服务器的访问权限，即便项目运行在团队所有或个人所有的服务器上。 | 由项目所有者移除；协作者可主动离开项目。 |
| **Server collaborator** | 在单一服务器内的协作者部分 | 一台专用服务器的管理权限——重启、重装、账单、服务器级设置。**不会**自动授予对其上部署的项目的访问权限。 | 由服务器所有者移除。 |

### 何时使用哪一种

- **使用 Team 成员**：当多人需要共享账单、看到所有当前与未来的项目和服务器，并以一个组织的形式协作。这适用于公司、付费的代理客户或长期合作者。
- **使用项目协作者**：当你想让某人对单一项目有编辑权限，仅此而已——比如做单个应用的自由职业者，或与你结对编程做副业的朋友。这种方式不需要团队。
- **使用服务器协作者**：当某人需要协助管理一台专用服务器（重启、重装），但不一定需要看到其上运行的项目。

三套系统可以共存：一名用户可能同时是你团队中的 Editor、是别人个人项目里的项目协作者，并且是第三方专用服务器上的服务器协作者。每一种授权都仅限于其颁发来源的资源。
