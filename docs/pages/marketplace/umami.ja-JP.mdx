# Deploy Umami

[Umami](https://umami.is/) は、プライバシーを重視する人々のためのシンプルで高速なウェブ解析ツールです。これは、Google Analytics のオープンソースの代替品です。

この記事では、カスタムドメインを使用して Zeabur に Umami をデプロイする方法を紹介します。

## テンプレートから Umami をデプロイする

Umami はデータベースとして PostgreSQL を必要とするため、イメージから直接デプロイすることはできません。

PostgreSQL と一緒に Umami をワンクリックでデプロイできるテンプレートを作成しました。

[![Deploy on Zeabur](https://zeabur.com/button.svg)](https://zeabur.com/templates/01NQCC)

### ステップ 1: Umami テンプレートページを開く

まず、Umami の [テンプレートページ](https://zeabur.com/templates/01NQCC) にアクセスします。

テンプレートにはすでに PostgreSQL サービスが設定されているのがわかりますので、何もする必要はありません。

![Template](/marketplace/umami/template.png)

### ステップ 2: テンプレートをデプロイする

デプロイボタンをクリックし、サービスをデプロイする地域を選択します。

![Select Region](/marketplace/umami/region.png)

選択後、自動的にプロジェクトページにリダイレクトされ、デプロイの進行状況を確認できます。

![Deployment](/marketplace/umami/deployment.png)

数秒待つと、Umami サービスが正常にデプロイされ、稼働を開始します。

### ステップ 3: ドメインをバインドする

デプロイ後、Linkwarden サービスにアクセスするためにドメインをバインドする必要があります。

詳細については、[ドメインをバインドするガイド](/deploy/domain-binding)を参照してください。

### ステップ 4: Umami にログインして設定する

ドメインをバインドした後、ドメインをクリックして Umami のログインページにアクセスし、デフォルトのアカウント `admin` とパスワード `umami` を使用してログインします。

![Umami login page](/marketplace/umami/screenshot.png)

Umami についての詳細は、[Umami 公式ドキュメント](https://umami.is/docs)を参照してください。
