# n8n をデプロイする

[n8n](https://github.com/n8n-io/n8n) は拡張可能なワークフロー自動化ツールです。フェアコード配布モデルを採用しており、n8nは常にオープンソースで、自分でホストすることができ、独自のカスタム関数、ロジック、およびアプリケーションを追加することができます。n8nのノードベースのアプローチにより、非常に柔軟で、何でも接続することができます。

この記事では、Zeabur で PostgreSQL を使用して n8n をデプロイする方法を紹介します。

## テンプレートから n8n をデプロイする

n8n はデータベースとして PostgreSQL を必要とするため、イメージから直接デプロイすることはできません。

私たちは PostgreSQL を含む n8n をワンクリックでデプロイできるテンプレートを作成しました。

[![Zeaburでデプロイする](https://zeabur.com/button.svg)](https://zeabur.com/templates/W2H4RW)

### ステップ1：n8n テンプレートページを開く

まず、[n8n のテンプレートページ](https://zeabur.com/templates/W2H4RW)にアクセスします。

テンプレートにはすでに PostgreSQL サービスが設定されているため、追加の設定は不要です。

![テンプレートページ](/marketplace/n8n/template.png)

### ステップ2：テンプレートをデプロイする

デプロイボタンをクリックし、サービスをデプロイする地域を選択します。

![地域を選択](/marketplace/n8n/region.png)

選択後、プロジェクトページに自動的にリダイレクトされ、デプロイの進行状況が表示されます。

![デプロイ進行状況](/marketplace/n8n/deployment.png)

数秒待つと、n8n サービスが正常にデプロイされ、実行を開始します。

### ステップ3：ドメインをバインドする

デプロイが完了したら、n8n サービスにアクセスするためにドメインをバインドする必要があります。

詳細については、[ドメインバインディングガイド](../deploy/domain-binding)を参照してください。

### ステップ4：アカウントを設定する

デプロイが完了したら、バインドしたドメインを通じてn8nサービスにアクセスできます。次のようなページが表示されます：

![n8nセットアップ](/marketplace/n8n/setup.png)

画面の指示に従って n8n アカウントを設定します。設定が完了すると、n8n を通常通り使用できるようになります。

![完了](/marketplace/n8n/finish.png)
