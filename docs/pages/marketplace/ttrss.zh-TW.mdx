# 在 Zeabur 部署 Tiny Tiny RSS

[Tiny Tiny RSS](https://tt-rss.org/) 是一個免費且開源的基於網路的新聞摘要 (RSS/Atom) 閱讀和聚集工具

這篇文章將教你如何透過 Marketplace 在 Zeabur 上部署自己的 Tiny Tiny RSS 服務，並綁定自己的網域。

## 步驟一：建立專案

<CreateProject />

## 步驟二：部署 PostgreSQL 服務

雖然 Tiny Tiny RSS 相容 MySQL，但是強烈建議使用 PostgreSQL 作為資料庫。

首先在 Marketplace 中一鍵部署 PostgreSQL 服務。

按下 Command+K 或者點選「加入服務」按鈕，然後選擇「Marketplace」。

![Select Marketplace](/deploy/common/select-service.en-US.png)

選擇 PostgreSQL：

![Deploy PostgreSQL](/marketplace/postgresql/deploy.png)

## 步驟三：部署 Tiny Tiny RSS 服務

你可以在 Marketplace 中一鍵部署 Tiny Tiny RSS 服務。

按下 Command+K 或者點選「加入服務」按鈕，然後選擇「Marketplace」。

![Select Marketplace](/deploy/common/select-service.en-US.png)

在列表中找到 Tiny Tiny RSS 並點選它以開始部署。

## 步驟四：綁定網域

在 Zeabur 控制台中 Tiny Tiny RSS 服務的「網域」區塊中，你可以綁定一個網域。

這個網域可以是你自己的網域，也可以是 Zeabur 提供的免費網域。

關於如何為你的服務綁定網域，請參閱 [綁定網域](../deploy/domain-binding)。

## 步驟四：部署 Mercury Parser API 服務（可選）

如果你想要使用 Mercury 擴充套件，你需要額外部署一個 Mercury Parser API 服務。

首先從 GitHub Fork 一份 [Mercury Parser API](https://github.com/HenryQW/mercury-parser-api) 儲存庫

然後建立一個服務，選擇從 Git 建置：

![Deploy](/deploy/common/select-service.en-US.png)

還需要在 Zeabur 控制台中 Mercury Parser API 服務的「網域」區塊中，綁定一個網域，或者是使用 Zeabur 提供的免費網域。

最後在 Tiny Tiny RSS 中啟用 Mercury 擴充套件，並輸入你的網域。

## 步驟五：開始使用

當部署完成後，你可以透過綁定的網域造訪你的 Tiny Tiny RSS 服務。
