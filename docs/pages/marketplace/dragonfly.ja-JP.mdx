# Dragonサービスをデプロイする

[Dragonfly](https://www.dragonflydb.io/)は、シンプルで、高性能で、低コストのメモリデータストレージです。DragonflyはRedis APIと完全に互換性がありますが、Redisの管理の複雑さはありません。

この記事は、サービスマーケットを通じてZeaburで自分のDragonflyサービスをデプロイする方法を教えます。

## ステップ1：プロジェクトを作成する

第一歩として、プロジェクトを作成する必要があります。 [プロジェクト](https://dash.zeabur.com/projects)ページに行くと、プロジェクトのリストが表示されます。

リストの末尾にある「新規プロジェクトを作成」ボタンをクリックして新しいプロジェクトを作成します。

![プロジェクト作成](/deploy/common/create-project.en-US.png)

## ステップ2：Dragonfly サービスをデプロイする

サービスマーケットでDragonflyサービスをワンクリックでデプロイすることができます。

"cmd + k" を押すか、"Deploy New Service" ボタンをクリックして、"サービスマーケット" を選択します。

![事前に構築したものを選択](/deploy/common/select-prebuilt-service.en-US.png)

リストから Dragonfly を見つけてクリックしてデプロイを始めます。

![Dragonflyを選択](/databases/dragonfly/deploy-dragonfly.en-US.png)

## ステップ3：接続情報を取得する

Dragonfly サービスをデプロイした後、Zeaburは関連接続用の環境変数を他のサービスに自動的に注入します。

- `DRAGONFLY_HOST`
- `DRAGONFLY_PORT`
- `DRAGONFLY_PASSWORD`

時折、上記の環境変数の代わりに `DRAGONFLY_URI` を使用することもあります、例えば：

```bash copy
redis://:${DRAGONFLY_PASSWORD}@${DRAGONFLY_HOST}:${DRAGONFLY_PORT}
```

私たちはコンソールの "Instructions" ボードでも接続情報を見つけることができます

![接続情報を取得](/databases/dragonfly/connection-dragonfly.en-US.png)

## ステップ4：使用を開始する

あなたは、上述の接続情報を使用して、あなたのアプリケーションやクライアントからDragonflyサービスに接続することができます。

DragonflyはRedis APIと完全に互換性があるため、Redisをサポートする任意のクライアントを使用してDragonflyサービスに接続することができます。

![Dragonflyに接続](/databases/dragonfly/client-dragonfly.en-US.png)
