# Deploy Dragon Service

[Dragonfly](https://www.dragonflydb.io/) 是一種簡單、高效能、低成本的記憶體內資料儲存空間。Dragonfly 與 Redis API 完全相容，但沒有 Redis 管理如此複雜。

這篇文章將教你如何透過服務市場，在 Zeabur 上部署自己的 Dragonfly 服務。

## 步驟一：建立專案

<CreateProject />

## 步驟二：部署 Dragonfly 服務

你可以在服務市場中一鍵部署 Dragonfly 服務。

按下 "cmd + k" 或者點選 "Deploy New Service" 按鈕，然後選擇「服務市場」。

![Select Prebuilt](/deploy/common/select-prebuilt-service.en-US.png)

在列表中找到 Dragonfly 並點選它以開始部署。

![Select Dragonfly](/databases/dragonfly/deploy-dragonfly.en-US.png)

## 步驟三：取得連線資訊

部署完 Dragonfly 服務後，Zeabur 會自動幫你注入和連線相關使用的環境變數到其他的服務中。

- `DRAGONFLY_HOST`
- `DRAGONFLY_PORT`
- `DRAGONFLY_PASSWORD`

可以選擇用 `DRAGONFLY_URI` 來取代上面的環境變數，例如：

```bash
redis://:${DRAGONFLY_PASSWORD}@${DRAGONFLY_HOST}:${DRAGONFLY_PORT}
```

我們也可以在控制台的「Instructions」區塊找到連線資訊。

![Get Connection Info](/databases/dragonfly/connection-dragonfly.en-US.png)

## 步驟四：開始使用

你可以在你的應用程式或者用戶端中，使用上面的連線資訊來連線到 Dragonfly 服務。

因為 Dragonfly 與 Redis API 完全相容，所以你可以使用任何支援 Redis 的用戶端來連接 Dragonfly 服務。

![Connect Dragonfly](/databases/dragonfly/client-dragonfly.en-US.png)
