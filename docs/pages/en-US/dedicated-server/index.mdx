# Dedicated Server

Beyond shared clusters, you can **register your own server with Zeabur** and let Zeabur handle service deployment, firewall rules, and resource allocation on your hardware — no manual SSH required.

## Why Use a Dedicated Server

- **Exclusive resources**: Your services don't share hardware with other users, so performance is stable and predictable.
- **Bring Your Own Hardware (BYOH)**: Already running servers on AWS, GCP, Hetzner, or your own datacenter? Mount them to Zeabur in one click and get full deployment management capabilities.
- **Geographic flexibility**: Cover regions not yet served by shared clusters using your own infrastructure.
- **High-reliability production**: For large systems with strict SLA and security requirements.

## What Zeabur Manages for You

Once registered, you can handle all of the following from the Zeabur Dashboard — without touching the host directly:

- Deploy and update services
- View real-time logs and performance metrics
- Set environment variables
- Manage persistent storage and network settings

<Callout type="info">
  Dedicated servers are billed monthly at a fixed rate, not by resource usage. Best suited for production workloads with stable hardware requirements.
</Callout>
