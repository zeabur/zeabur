# 共享叢集（已淘汰）

<Callout type="error">
共享叢集正在逐步退場。新專案已無法在共享叢集上建立，現有共享叢集專案已自 **2026 年 4 月 1 日**起停止接受新服務，所有新專案請使用[伺服器](/server)。
</Callout>

## 什麼是共享叢集

共享叢集是 Zeabur 早期提供的伺服器方案。在共享叢集模式下，多位使用者的服務會部署在同一組 Kubernetes 叢集中，共享底層的 CPU、記憶體和網路資源。使用者無需管理硬體，只需按實際資源使用量付費。

## 與伺服器的差異

| 項目 | 共享叢集（已淘汰） | 伺服器 |
| --- | --- | --- |
| 資源隔離 | 多租戶共用硬體 | 完整獨享硬體 |
| 計費方式 | 按用量計費（vCPU / 記憶體 / 流量） | 按月固定費用 |
| 伺服器位置 | 固定區域（AWS、GCP） | 自攜主機（BYOH），區域不限 |
| 效能穩定性 | 可能受鄰居影響 | 資源穩定可預測 |
| 適用場景 | 小型專案、開發測試 | 生產環境、高穩定性需求 |

## 停止服務的原因

隨著伺服器方案的成熟，Zeabur 決定將共享叢集整合至伺服器架構，提供更好的資源隔離與效能保障。

## 一定要搬遷嗎？

**共享叢集上的既有服務會繼續運作。** 目前「停止服務」的意思是不能再新增*新的*服務——已部署的服務不會被關閉，也沒有公告既有服務的統一關閉期限。

Zeabur 只會在特定情況下主動聯絡專案擁有者要求搬遷：

- 專案資源用量長期偏高，影響共享叢集上的其他使用者
- 觸發了濫用偵測（例如爬蟲、挖礦）
- 工作負載需要共享叢集無法提供的能力（對外固定 IP、高頻寬、客製網路）

如果沒有收到我們的主動聯絡，搬遷是**可選的**——不過搬到[專屬伺服器](/server)仍有實際好處：沒有平台層速率限制、效能完全獨立、不受共享租戶的限制。

## 使用 Clone Project 搬遷（建議）

最簡單的搬遷方式是內建的 **Clone Project** 功能，它會把整個專案——服務、環境變數與設定——複製到你選擇的區域：

1. **準備伺服器** — 在 [Zeabur 伺服器頁面](https://zeabur.com/servers) 購買或[註冊自己的伺服器](/server/register)。
2. 在舊專案中前往 **Settings → General → Clone Project**。
3. 在 **Target Region** 選擇你伺服器所在的區域。
4. 設定複製選項——可以讓舊專案在複製期間繼續運作，並保留服務群組與排序。
5. 確認後等待複製完成，逐一檢查新專案中的每個服務都正常啟動與運作。
6. **切換 DNS / 網域** — 把自訂網域重新綁定到新服務，並更新 DNS 記錄。
7. 流量完全由新專案承接後，**刪除舊專案**。

![Clone Project 對話框](/server/clone-project-dialog.webp)

<Callout type="warning">
複製過程可能產生數分鐘到數十分鐘不等的停機時間（取決於資料量），除非啟用「舊專案繼續運作」選項。Volume 資料與其他有狀態儲存在複製後務必驗證——資料庫請先做好[備份](/operations/data/backup-restore)。
</Callout>

## 手動搬遷（替代方案）

如果你想重建而不是複製：

1. **備份資料** — 使用[備份與還原](/operations/data/backup-restore)功能，為所有有狀態服務（資料庫、持久儲存）建立備份。
2. **匯出專案** — 透過[匯出專案](/deploy/manage/export-project)功能匯出專案的 YAML 配置。
3. **在伺服器上重新部署** — 在伺服器的區域建立新專案，匯入配置並重新部署服務。
4. **還原資料** — 將步驟 1 的備份還原到新服務中。
5. **切換 DNS** — 將自訂網域的 DNS 記錄指向新服務的網域。
6. **驗證與清理** — 確認新環境運作正常後，移除共享叢集上的舊服務。

## 常見問題

**我必須在某個期限前搬遷嗎？**
既有服務目前沒有公告統一期限。只有在 Zeabur 針對你的專案主動聯絡時才必須處理。不過自 2026 年 4 月 1 日起，共享叢集已無法建立新服務。

**我現有的專案會停止運作嗎？**
不會。既有服務照常運作。「停止服務」只是不能新增服務，不會停掉正在運行的服務。

**Clone 會搬移我的 Volume 資料嗎？**
資料庫內容與 Volume 資料在複製後需要驗證——搬遷前請務必先建立[備份](/operations/data/backup-restore)，如有缺漏再手動還原。
