# 作業系統

當你[從 Zeabur 購買主機](/zh-TW/server/purchase)或重灌既有機器時，可以選擇機器要執行的作業系統。這個選擇決定了機器從第一天起的運作方式：

- **ZeaburOS**——機器由 Zeabur 管理：開箱即可部署專案、套用一鍵模板，並享有 Zeabur 技術支援。
- **標準發行版**——目前提供 Ubuntu 24.04 LTS，未來會開放更多作業系統與版本——機器完全由你自行管理；Zeabur 僅對 ZeaburOS 提供技術支援。

## 什麼是 ZeaburOS [#zeaburos]

ZeaburOS 是把伺服器變成 **Zeabur 部署節點**的作業系統。執行 ZeaburOS 的機器會連上 Zeabur Dashboard，讓你部署服務、查看日誌與效能指標、管理儲存與網路——日常操作無需 SSH 進主機，但你仍保有完整的 SSH 存取權。

在底層，ZeaburOS 以 [K3s](https://k3s.io/)（輕量級 Kubernetes）為核心，搭配 Zeabur 系統元件（部署代理、監控等），讓機器保持與平台連線。

相較於執行標準發行版，ZeaburOS 為你帶來：

- **開箱即部署**——不用設定執行環境、反向代理或憑證；推送專案就能執行。
- **內建可觀測性**——在 Dashboard 中即時查看日誌、效能指標與服務狀態。
- **託管維護**——系統元件可以一鍵更新或修復，並享有 Zeabur 技術支援。

<Callout type="info">
  ZeaburOS 過去在 Dashboard 中顯示為「K3s」。引擎不變，名字更清楚——詳見下方 [K3s 現在是 ZeaburOS](#k3s-is-now-zeaburos)。
</Callout>

## 標準發行版

選擇標準發行版，你會得到一台標準 VPS，與發行版原始出貨的狀態完全相同——就跟任何供應商提供的一樣。你完全透過 SSH 操作它：機器上執行什麼、如何設定、如何維護，全都由你決定。Zeabur 提供機器，上面的軟體完全由你自行管理。

Ubuntu 24.04 LTS 現已推出，未來會開放更多作業系統與版本選擇。

目前只想要一台標準 VPS、暫時不打算使用 Zeabur 平台？直接以標準發行版購買機器即可——不會預裝任何東西，也就沒有事後需要解除安裝的負擔。

## 如何選擇

|                                          | ZeaburOS | 標準發行版 |
| :--------------------------------------- | :------- | :-------------------- |
| 從 Zeabur Dashboard 部署與管理服務 | ✅       | ❌                    |
| 一鍵部署模板            | ✅       | ❌                    |
| 內建日誌、效能指標與監控   | ✅       | ❌                    |
| SSH 存取與完整 OS 控制權           | ✅       | ✅                    |
| 系統元件由 Zeabur 維護   | ✅       | ❌（完全自行管理） |
| Zeabur 技術支援                 | ✅       | ❌                    |

**簡單原則**：想用 Zeabur 部署專案，就選 ZeaburOS——這正是這台機器的用途；想要一台完全自行管理的標準 VPS，就選標準發行版。

## 之後想換？

### 將現有機器轉換為 ZeaburOS [#convert-to-zeaburos]

已經在執行標準發行版的機器可以轉換為 ZeaburOS，使其成為部署節點。轉換的方式是**重灌機器**：機器會被重新安裝成 ZeaburOS，得到一個乾淨、可預期的環境，不受既有系統上裝過什麼影響。

要進行轉換，打開伺服器的**設定**頁面，在**危險區域**使用「重灌作業系統」並選擇 **ZeaburOS**。重灌與供裝需要數分鐘，完成後機器會顯示為部署節點，你就可以開始在上面部署服務。

<Callout type="warning">
  **重灌會清除機器上的所有資料**，包括你自己安裝的軟體與資料。切換作業系統在**兩個方向**都是重灌——換成 ZeaburOS 與換回標準發行版一樣會清空磁碟。請先備份需要保留的內容。
</Callout>

<Callout type="default">
  沒有 ZeaburOS 你仍然可以透過 SSH 連線到機器——只有要透過 Zeabur 部署與管理服務時才需要轉換。
</Callout>

### 更新 ZeaburOS

**更新 ZeaburOS** 會將伺服器上的所有 ZeaburOS 系統元件（部署代理、監控等）重新套用至最新狀態。已部署的服務不受影響；伺服器異常時，這也是修復問題的第一步。

你可以從伺服器的選單或**設定**頁面觸發更新。操作通常安全，但可能造成短暫停機。

### 重灌為標準 VPS

如果你試用了 ZeaburOS，卻發現 Zeabur 平台不適合你，機器也不會被綁死。在伺服器設定頁面的**危險區域**使用「重灌作業系統」並選擇標準發行版——機器會變回一台標準 VPS，完全由你自行管理。

<Callout type="warning">
  重灌作業系統會清除機器上的所有資料，包括已部署的服務及其資料。請先備份需要的內容。
</Callout>

## 支援範圍

Zeabur 僅對 **ZeaburOS** 提供技術支援；自行安裝的軟體——包含 1Panel、寶塔面板等管理面板——不在支援範圍內。

選擇標準發行版時，機器完全由你自行管理：Zeabur 提供機器，上面執行的一切由你負責。

## K3s 現在是 ZeaburOS [#k3s-is-now-zeaburos]

過去在 Zeabur Dashboard 中顯示為 **「K3s」** 的一切，現在都顯示為 **ZeaburOS**。引擎不變，名字更清楚：ZeaburOS 以 K3s（輕量級 Kubernetes）為核心，你現有的機器沒有任何改變——原本顯示「K3s」的狀態，就是同一個東西換上了產品名稱。

伴隨更名，有兩件事是新的：

- 購買機器或重灌作業系統時，ZeaburOS 不再自動預裝——你可以**選擇** ZeaburOS 或標準發行版作為作業系統。
- 未安裝 ZeaburOS 的機器之後仍可從伺服器的設定頁面進行轉換。

主機層級的維護上，底層技術名稱維持不變：systemd 服務仍是 `k3s`，持久儲存空間仍位於 `/var/lib/rancher/k3s/storage`，[管理伺服器](/zh-TW/server/operate)中的指令也照常使用。
