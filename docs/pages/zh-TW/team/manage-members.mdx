# 管理成員

團隊只有在加入了具備正確權限等級的隊友後才有意義。本頁是各角色能做與不能做什麼、如何邀請與移除成員，以及團隊成員身分與專案／伺服器層級協作之間有何差異的權威參考。

如果你還在評估自己是否需要團隊，請從 [團隊](/team) 與 [建立團隊](/team/create-team) 開始。

## 角色一覽

| 角色 | 管理成員與團隊資料 | 管理帳單 | 建立 / 修改資源 | 刪除資源 |
| --- | :---: | :---: | :---: | :---: |
| **Administrator** | 是 | 是 | 是 | 是 |
| **Editor** | 否 | 否 | 是 | 否 |
| **Viewer** | 否 | 否 | 否 | 否 |

讀取權限是分層的：任何角色都能看到團隊的專案、服務、部署歷史、伺服器狀態，並檢視執行階段日誌。**搜尋與下載日誌僅限 Editor 與 Administrator。** 發票、付款方式與帳單明細**僅限 Administrator 可見**。

建立團隊的使用者會被記錄為 **團隊建立者（Creator）**（控制台中顯示為「Creator」）。建立者一定是管理員，且任何人都不能將其移除或降級 — 只有建立者本人能透過 [轉移擁有權](/team/transfer-ownership) 退位。每個團隊都必須隨時保留至少一位管理員。

### Administrator（管理員）能做什麼

- 更新團隊資料：名稱、Logo、系統通知用的聯絡電子郵件。
- 邀請、移除、變更其他成員的角色。
- 啟用、取消、恢復 Team 方案。
- 管理付款方式、帳單資訊（法定名稱、地址、稅號）與預付點數。
- 建立、修改 **以及** 刪除團隊擁有的專案、服務、環境、資料庫與專用伺服器。
- 將個人專案或伺服器轉移到團隊（請參閱 [轉移擁有權](/team/transfer-ownership)）。
- 在團隊不再擁有任何專案或伺服器時刪除團隊本身。

### Editor（編輯者）能做什麼

Editor 是日常「開發與上線」角色。

- 建立與修改團隊擁有的專案、服務、環境、環境變數與網域。
- 觸發部署、回滾至先前的部署、重啟服務。
- 搜尋與下載日誌。
- 新增或修改掛在團隊下的專用伺服器與資料庫。

Editor **不能** 做的事：

- 邀請、移除或變更任何成員的角色。
- 編輯團隊名稱、Logo 或聯絡電子郵件。
- 檢視發票、變更付款方式、啟用或取消 Team 方案，或加值餘額。
- 刪除專案、服務、資料庫或伺服器。Editor 只能建立與修改；只有管理員能刪除。

<Callout type="info">
  Editor 的寫入操作必須在 Team 方案處於啟用狀態下才可使用。當團隊處於 [團隊概觀](/team#啟用-team-方案) 中描述的未啟用狀態時，Editor 無法建立或修改資源 — 必須由管理員完成啟用步驟。
</Callout>

### Viewer（檢視者）能做什麼

Viewer 對團隊工作區的大部分內容只有唯讀存取權限。適合給利害關係人、稽核人員、長期顧問，或任何需要看到部署狀況但不應變更任何內容的人。

Viewer **可以**：

- 檢視專案、服務、部署歷史、伺服器／資料庫狀態。
- 開啟執行階段日誌與建構日誌（唯讀）。
- 切換到團隊工作區，並像其他成員一樣瀏覽。

Viewer **不能**：

- 部署、重啟、回滾，或變更服務設定。
- 建立或修改專案、服務、環境變數、資料庫或伺服器。
- 搜尋或下載日誌（僅允許在控制台中檢視）。
- 邀請、移除或變更成員角色。
- 檢視發票、付款方式或帳單明細，亦不能變更任何帳單相關內容。

## 邀請成員

只有管理員可以邀請。被邀請者必須已經以同一個電子郵件註冊 Zeabur 帳號 — 系統沒有「待接受邀請」的收件箱；一旦你新增了該成員，他立刻就是有效成員，下次登入時即可從工作區切換器切換到團隊工作區。

<Steps>
### 切換到團隊工作區

在控制台側邊欄的工作區切換器中選擇你要邀請對象加入的團隊。你必須在團隊工作區中才能看到 **成員（Members）** 頁面；個人工作區並不會顯示成員管理。

### 打開「成員」

前往 **帳號 → 成員**。管理員會在成員清單右上角看到 **邀請成員（Invite a member）** 按鈕。Editor 與 Viewer 看不到此按鈕。

### 輸入電子郵件並選擇角色

點擊 **邀請成員（Invite a member）**，輸入隊友的電子郵件，並選擇 **Administrator**、**Editor** 或 **Viewer**。日後可以變更角色，因此一開始把成員設為 Viewer，等熟悉其工作模式後再升級也是合理的做法。

### 確認席次費用預覽（如有）

如果你的團隊已啟用 Team 方案，且新增此成員會超過內含 seat 數，控制台會在邀請送出前顯示超額 seat 的費用預覽。本期剩餘時間以按比例計算的方式收取，並加到下一張發票上。席次的計價方式請參閱 [團隊帳單](/team/billing)。

### 送出

點擊 **送出邀請（Send invitation）**。新成員會立即出現在清單中，並擁有你選擇的角色。下次他重新載入控制台時，工作區切換器中就會看到此團隊。
</Steps>

如果你輸入的電子郵件並未對應到既有的 Zeabur 帳號，邀請會被拒絕 — 請對方先到 [zeabur.com](https://zeabur.com) 註冊，再重試一次。

## 變更成員的角色

管理員可以隨時變更任何成員的角色，但有一個例外：任何人（即便是其他管理員）都不能變更 **建立者** 的角色。如果你需要變更實際掌管團隊的人，請改用 [轉移擁有權](/team/transfer-ownership)。

要變更角色，打開 **帳號 → 成員**，點擊成員旁的角色下拉選單，選擇新的角色即可。對該成員而言，變更會在下次重新整理頁面時立即生效 — 沒有額外的「儲存」步驟。

你不能在這頁變更自己的角色。要將自己從團隊中移除，請使用 **離開團隊（Leave team）**（見下文）。

## 移除成員

管理員可以移除除建立者以外的任何成員。移除成員後：

- 立即生效。被移除的使用者會立刻失去對所有團隊擁有的專案、服務、伺服器與帳單資料的存取權。
- **不會** 刪除他的任何個人專案或伺服器 — 這些仍保留在他的個人工作區。
- 釋放 Team 方案的一個 seat，下一張發票的 seat 費用會相應減少。

要移除某人，打開 **帳號 → 成員**，在該列點擊 **移除（Remove）**，並確認。移除後不會有額外的通知步驟。

### 自行離開團隊

任何成員都可以離開自己所屬的團隊，但建立者除外。在團隊工作區中前往 **帳號 → 團隊設定**，並使用危險區的 **離開團隊（Leave team）**。離開後，你會失去對團隊資源的存取權，控制台會把你帶回個人工作區。

如果你是建立者並希望離開，必須先 [轉移擁有權](/team/transfer-ownership) 給另一位管理員。

## 團隊成員 vs 專案／伺服器協作者

Zeabur 有三套互相獨立的邀請系統。選對系統很重要，因為它們授予的存取範圍不同，管理位置也不同。

| | 在哪裡邀請 | 對方獲得什麼 | 由誰移除 |
| --- | --- | --- | --- |
| **團隊成員** | 團隊工作區的 **帳號 → 成員** | 所有現有與未來的團隊擁有專案、服務與伺服器，依你指派的角色限定範圍。管理員另外擁有帳單存取權。 | 由管理員移除；成員可自行離開。 |
| **專案協作者** | 在單一專案內的設定面板中 | 單一專案的服務、部署與環境變數。**不會** 授予對底層專用伺服器的存取權，即使該專案運行在團隊擁有或個人擁有的伺服器上。 | 由專案擁有者移除；協作者可自行離開該專案。 |
| **伺服器協作者** | 在單一伺服器內的協作者區段中 | 一台專用伺服器的管理權 — 重啟、重灌、帳單、伺服器層級設定。**不會** 自動授予對部署在該伺服器上專案的存取權。 | 由伺服器擁有者移除。 |

### 何時該使用哪一種

- **使用團隊成員身分** — 當有多人需要共享帳單、看到所有現有與未來的專案與伺服器，並以同一個組織運作時。這是公司、付費的代理客戶或長期合作夥伴的正確選擇。
- **使用專案協作者** — 當你想讓某人對單一專案具有編輯權限，其他則完全不開放時 — 例如為單一應用程式工作的自由接案者，或一起 pair programming 副專案的朋友。不需要建立團隊。
- **使用伺服器協作者** — 當有人需要協助管理專用伺服器（重啟、重灌），但不一定要看到上面運行的專案時。

這三套系統可以並存：一位使用者可能同時是你團隊的 Editor、別人個人專案的協作者，以及第三方專用伺服器的協作者。每一份權限都僅限於它所授權的資源範圍。
