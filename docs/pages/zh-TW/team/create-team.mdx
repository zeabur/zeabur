# 建立團隊

團隊（Team）是一個獨立的工作區，擁有自己的成員、專案、伺服器與帳單。本頁將帶你逐步建立一個團隊，並說明建議的後續步驟，讓團隊真正可以開始使用。

如果你還想了解團隊的概念以及它與個人工作區的差異，請先閱讀 [團隊](/team)。

## 在哪裡建立團隊

在控制台中打開 **工作區切換器** — 它位於左側邊欄頂端，旁邊有頭像、名稱以及一個下拉箭頭。點擊它打開選單，然後在工作區清單最下方點擊 **建立團隊（Create team）**。

任何已登入的使用者都可以建立團隊。沒有數量限制 — 你可以加入任意多個團隊，也可以建立多個團隊（例如針對不同客戶或不同事業單位，需要分開的帳單情境時）。

## 建立表單

**建立團隊（Create Team）** 對話框只需要填一個欄位：

- **團隊名稱（Team name）** — 顯示給所有成員、也會出現在發票上的名稱。之後可以在團隊設定中修改。

僅此而已。點擊 **建立（Create）** 即完成。

<Callout type="info">
  建立過程中不會設定團隊頭像（Logo） — 每個新建的團隊都會以名稱自動產生一個預設頭像。建立完成後，你可以在團隊工作區中前往 **帳號 → 團隊設定** 上傳自訂 Logo。
</Callout>

## 建立完成後立即發生的事

<Steps>

### 你成為第一位管理員

建立團隊的使用者會自動成為該團隊的管理員。團隊中必須至少保留一位管理員 — 此規則會在後續移除或降級管理員時被強制執行。

### 控制台會切換到新團隊

工作區切換器會更新為新團隊的名稱與頭像，控制台其他區塊（專案、伺服器、帳號設定）也會重新整理為團隊的情境。要回到個人工作區，再次打開工作區切換器，從清單中選擇你的個人工作區即可。

### 團隊處於未啟用狀態

剛建立的團隊 **尚未訂閱 Team 方案**。在 Team 方案啟用前，成員都無法在團隊下建立專案、部署模板、管理伺服器、使用 AI Hub 或透過 Z‑Send 寄送郵件。

未啟用的團隊仍可進行的操作 — 讓你能完成設定：

- 更新團隊資料（名稱、頭像、聯絡電子郵件）
- 邀請成員並指派角色
- 綁定付款方式
- 啟用 Team 方案
- 切換回個人工作區
- 刪除團隊（僅在沒有任何專案或伺服器時）

</Steps>

## 建議的後續步驟

要讓團隊真正能用起來，建議依照下列順序進行：

<Steps>

### 啟用 Team 方案

進入團隊工作區後，前往 **帳號 → 訂閱**，並點擊 **啟用 Team 方案**。控制台會在付款前先預覽依席次計算的費用（基本方案內含 3 個 seat，超過部分依每個 seat 計價）。

完整的價格、付款方式、發票與預付點數說明，請參閱 [團隊帳單](/team/billing)。

### 邀請成員

進入團隊工作區後前往 **帳號 → 成員**，輸入隊友的電子郵件，並為每位成員選擇角色（**Administrator**、**Editor** 或 **Viewer**）。當新增的成員人數超過內含 seat 數時，下一張發票會以按比例計算的方式收取超額 seat 費用。

完整的角色權限矩陣與邀請流程，請參閱 [管理成員與角色](/team/manage-members)。

### 將既有的個人專案或伺服器帶入（選用）

如果你個人工作區中已有專案或專用伺服器希望歸屬於團隊，管理員可以將擁有權轉移過來，往後的帳單與配額就會掛在團隊上。原本的擁有者仍可作為團隊成員繼續操作該資源。

請參閱 [將擁有權轉移到團隊](/team/transfer-ownership)。

</Steps>

## 常見問題

### 我建立了團隊，但無法建立專案或重啟任何東西

Team 方案還沒啟用。剛建立的團隊處於未啟用狀態，在管理員完成訂閱前大多數操作都會被擋下。請在團隊工作區中前往 **帳號 → 訂閱** 啟用 Team 方案 — 詳見 [團隊帳單](/team/billing)。

### 可以建立多個團隊嗎？

可以。一位使用者可以建立並加入任意多個團隊，每個團隊都有獨立的成員、資源與帳單。透過工作區切換器即可在不同團隊之間切換。

### 之後可以重新命名團隊或更換頭像嗎？

可以。在團隊工作區中前往 **帳號 → 團隊設定**。管理員可以更新團隊名稱、上傳新的 Logo、編輯系統通知用的聯絡電子郵件。

### 可以刪除團隊嗎？

可以 — 但前提是團隊下沒有任何專案與伺服器。在 **帳號 → 團隊設定 → 危險區（Danger zone）** 中輸入團隊名稱以確認，然後點擊 **刪除團隊（Delete team）**。只有管理員能刪除團隊。如果團隊仍擁有資源，請先轉移或刪除這些資源。

如果你只是想離開一個你不再參與的團隊（而不是刪除它），請在同一個團隊設定頁面使用 **離開團隊（Leave team）** — 此選項適用於非管理員成員，以及團隊中還有其他管理員時的管理員。
