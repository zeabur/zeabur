# 快速開始

## 步驟一：建立帳戶

你需要先建立一個帳戶來開始在 Zeabur 部署你的服務。

如果您是第一次使用 Zeabur，需要先使用 E-mail 註冊（也就是第一個選項）。在這裡填寫您的 E-mail 然後，點選「透過電子郵件繼續」。

![login](/start/login.webp)

接著您會收到「登入連結已經寄送」的提示。

![verification mail sent](/start/verification-mail-sent.webp)

打開您的電子信箱頁面（比如 Gmail、Outlook、Apple Mail 等等），您會看到一封來自 Zeabur 的信件。點選信件內的「登入」按鈕。

![verification mail](/start/verification-mail.webp)

<Callout>
  如果您登入的瀏覽器不是您系統設定的預設瀏覽器（也就是說打開連結時出現的是 Safari 而不是 Chrome），可以先右鍵「登入」按鈕複製連結，
  然後再貼上到您登入的瀏覽器打開。
</Callout>

出現這個登入介面，就代表您完成登入了！

![onboarding ui](/start/onboarding-ui.webp)

記得右上角可以切換介面語言喔！我們目前有繁體中文、簡體中文、英文、日文、西班牙文和印尼文可以選擇。

![switch language](/start/switch-language.webp)

## 步驟二：建立專案

在 Zeabur 中，一個「專案 (Project)」用於容納構成真實世界應用程式的所有相關服務。舉例來說，如果您的應用程式由一個前端網站、一個後端 API 和一個資料庫所組成，那麼這些服務都應該被放在同一個專案中。

![project structure](/start/project-structure.webp)

要開始建立專案，請點選正中央（或右上角）的 **建立專案** 按鈕。

![create new project](/start/create-new-project.webp)

如果您是第一次建立專案，很可能會看到「帳號驗證」的提示。

![account verification](/start/account-verification.webp)

您可以選擇：

1. 使用電話號碼進行驗證。大部分的國家或地區都支援這種驗證方式，如果您的國家或地區不支援這種方式，或者是需要綁定多個 Zeabur 帳號，除了 [聯絡我們](https://zeabur.com/dc) 外也可以參考第二和第三種方案。
2. 預存一些使用額度。只要您沒有訂閱 Dev Plan 或伺服器，且沒有涉及嚴重濫用行為（比如違反 ToS 發起大流量 DoS），我們都不會扣除您的餘額。
3. 綁定信用卡。和第二點相似，除非您有操作訂閱項目，否則我們都不會發起扣款，純粹作為驗證用途。

選擇您想要的方式，然後點選對應區塊的「驗證」按鈕。

驗證完成後，重新建立專案就可以看到建立完成的專案了。

![project page with add service](/start/project-page-with-add-service.webp)

## 步驟三：部署服務

Zeabur 可以部署各式各樣的服務，包括資料庫（PostgreSQL、MySQL、Redis）、開源專案（n8n、WordPress），或者是你自己寫的程式（從 GitHub 匯入或者是用資料夾上傳）。

![project page with add service](/start/project-page-with-add-service.webp)

- 如果你想要部署資料庫的話，點選「資料庫」就可以直接部署各種常見的資料庫。
  ![database services](/start/database-services.webp)
- 如果您想要部署某個開源專案，Zeabur 的「模板」有預先設定好很多常見的開源專案，點選「模板」就可以一鍵部署。
  ![template services](/start/template-services.webp)
- 如果您想要部署您自己寫的程式，點選「GitHub」、「本機專案」就可以上傳您的程式碼，也可以按「Cursor」直接部署您目前的專案。
- 如果您想部署的開源專案不在「模板」裡面，也可以用「Docker Image」自己架設，屬於進階操作。

就以 GitHub 部署為例，您需要先將帳號綁定到 GitHub 上後等待 1 分鐘，然後搜尋您想要部署的儲存庫：

![search repository](/start/search-repository.webp)

點下去儲存庫後，Zeabur 會自動分析您的程式碼，然後為您選擇最適合的部署方式。

![build plan](/start/build-plan.webp)

直接點選「部署」就能一鍵部署專案。如果有需要，可以到「配置」調整環境變數。

![build configuration](/start/build-configuration.webp)

## 使用 AI 部署服務

Zeabur 的 AI 助理可以讓您用自然語言，快速部署出您需要的服務。

![zeabur agent](/start/zeabur-agent.webp)

點選導覽列的「Agent」即可進入 AI 助理的介面。如果部署遇到障礙，也可以請它幫忙！

## 步驟四：上線網站

要讓別人可以瀏覽你的網站，可以到「網域」中綁定自己的網域：

![bind domain](/start/bind-domain.webp)

等待 30 秒左右就能點開網域，Zeabur 會幫您處理好 HTTPS 等等的設定，確認沒問題就能分享自己的網站了。

如果需要綁定自己的網域，可以到 [網域綁定](/deploy/domain-binding) 文件中了解更多資訊。

## 下一步可以做什麼？

如果想要用 Google、GitHub 帳號登入的話，可以到「設定」>「整合」中綁定 GitHub 和 Google 帳號。

![account integration](/start/account-integration.webp)

接下來您可以繼續探索 Zeabur 豐富的模板庫，以及各種面向不同用途的功能（如檔案管理、Logs）。如果您需要技術支援的話，[Zeabur 的 Discord 伺服器](https://zeabur.com/dc) 有專門的頻道可以幫助您解決各種疑難雜症！
