# 網域管理

要使用 Z-Send 發送郵件，您需要先驗證發件網域。網域驗證確保您擁有該網域，並提高郵件送達率。

## 為什麼需要驗證網域

import { Callout, Steps } from 'nextra/components'

<Callout type="info">
  未經驗證的網域無法用於發送郵件。網域驗證透過 DKIM、SPF 和 DMARC 記錄提高郵件的可信度。
</Callout>

- **防止偽造**：確保只有網域擁有者才能使用該網域發送郵件
- **提高送達率**：已驗證的網域更容易通過垃圾郵件過濾器
- **品牌保護**：DKIM 簽章保護您的品牌免受釣魚攻擊

## 新增網域

<Steps>

### 登入控制台

造訪 Zeabur 控制台的 Z-Send 管理頁面。

### 新增新網域

1. 進入「網域管理」
2. 點擊「新增網域」
3. 輸入您的網域（例如：`yourdomain.com`）
4. 選擇發送區域（推薦選擇靠近您使用者的區域）

### 設定 DNS 記錄

系統會自動產生需要新增的 DNS 記錄，並在控制台頁面顯示。請按照頁面提示，在您的 DNS 提供商（如 Cloudflare、阿里雲、騰訊雲等）處新增這些記錄。

<Callout type="info">
  **您需要新增的記錄類型**：
  - **DKIM 記錄**（通常 3 條）：用於郵件數位簽章，證明郵件來源真實性
  - **SPF 記錄**：指定哪些伺服器被授權發送您網域的郵件
  - **DMARC 記錄**：定義郵件認證失敗時的處理策略
  - **MX 記錄**（可選）：用於接收退信通知
</Callout>

### 等待驗證完成

DNS 記錄新增後，點擊控制台的「驗證」按鈕。Z-Send 會自動檢查您的 DNS 設定。

**驗證通常需要 5-30 分鐘，DNS 傳播時間取決於您的 DNS 提供商。**

<Callout type="success">
  當控制台顯示網域狀態為 **`verified`** 時，表示驗證成功，您可以開始使用該網域發送郵件了！
</Callout>

</Steps>

## DNS 記錄說明

為了確保郵件能夠順利送達，Z-Send 使用以下標準郵件認證技術：

### DKIM（郵件數位簽章）

DKIM 為每封郵件新增數位簽章，證明郵件確實來自您的網域，防止郵件被竄改或偽造。通常需要新增 3 條 DKIM CNAME 記錄。

**作用**：
- 防止郵件偽造
- 提高郵件送達率
- 保護您的品牌聲譽

### SPF（發件伺服器授權）

SPF 記錄指定哪些伺服器被授權代表您的網域發送郵件，防止惡意第三方冒用您的網域。

**作用**：
- 防止網域被冒用
- 降低垃圾郵件誤判率

<Callout type="warning">
  如果您的網域已經在其他地方（如 Google Workspace、Microsoft 365）設定了 SPF 記錄，請聯絡支援團隊獲取協助，避免衝突。
</Callout>

### DMARC（郵件認證策略）

DMARC 定義當郵件認證（SPF 或 DKIM）失敗時的處理策略，幫助您監控和保護網域免受濫用。

**作用**：
- 接收認證失敗報告
- 防止釣魚攻擊
- 提高網域可信度

### MX 記錄（可選）

MX 記錄用於接收退信通知（bounce notifications），幫助您了解郵件發送失敗的原因。

**作用**：
- 接收退信通知
- 及時發現發送問題

## 驗證狀態

網域可能處於以下狀態：

| 狀態 | 說明 |
|------|------|
| `pending` | 等待驗證，DNS 記錄尚未設定 |
| `verifying` | 正在驗證 DNS 記錄 |
| `verified` | 已驗證，可以使用 |
| `failed` | 驗證失敗，請檢查 DNS 設定 |

## 常見問題

### DNS 記錄需要多長時間生效？

通常 5 分鐘到 48 小時不等，取決於您的 DNS 提供商。建議等待 15-30 分鐘後再嘗試驗證。

### 驗證失敗怎麼辦？

1. **等待更長時間**：DNS 傳播通常需要 5-30 分鐘，某些 DNS 提供商可能需要更長時間（最多 48 小時）
2. **檢查 DNS 設定**：確認您已按照控制台顯示的記錄準確新增，注意不要有多餘的空格或引號
3. **檢視錯誤訊息**：Z-Send 控制台會顯示具體的驗證錯誤，請根據提示進行修改
4. **聯絡支援**：如果持續驗證失敗，請聯絡 Zeabur 支援團隊獲取協助

### 可以使用子網域嗎？

可以！您可以驗證子網域（如 `mail.yourdomain.com`），這樣可以：
- 分離不同類型的郵件（行銷 vs 交易郵件）
- 更好地控制網域聲譽

### 驗證成功的標誌是什麼？

當 Z-Send 控制台顯示網域狀態為 **`verified`** 時，表示所有 DNS 記錄都已正確設定並通過驗證。此時您可以立即使用該網域發送郵件。

## 網域刪除

如果不再需要某個網域：

1. 進入「網域管理」
2. 找到要刪除的網域
3. 點擊「刪除」按鈕
4. 確認刪除

<Callout type="warning">
  刪除網域後，使用該網域發送郵件的 API 呼叫將失敗。請確保沒有正在使用該網域的應用。
</Callout>

## 最佳實踐

### 使用專用子網域

推薦為郵件發送使用專用子網域（如 `mail.yourdomain.com`），這樣可以：
- 保護主網域聲譽
- 更容易追蹤和管理郵件
- 分離不同類型的郵件（如行銷郵件、交易郵件）

### 避免頻繁更改 DNS

網域驗證成功後，請勿隨意修改或刪除 DNS 記錄，否則可能導致郵件發送失敗或被標記為垃圾郵件。
