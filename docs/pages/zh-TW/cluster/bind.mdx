# 綁定既有 Kubernetes 叢集

如果你已經在其他雲端供應商購買或建立 Kubernetes 叢集，也可以將該叢集綁定到 Zeabur，繼續使用 Zeabur 的部署與管理體驗。

常見情境包括：

- 你已經在 AWS、GCP、DigitalOcean、Hetzner 或其他供應商上有既有叢集。
- 你希望保留現有網路拓撲、資料位置或合規設定。
- 你想把 Kubernetes 的日常操作入口整合回 Zeabur Dashboard。

## 綁定前準備

在開始之前，請先確認：

- 你的 Kubernetes API Server 可從 Zeabur 存取。
- 你擁有足夠權限可提供 Zeabur 綁定叢集所需的存取資訊。
- 你的叢集運作正常，且節點、網路與儲存元件都已完成基本配置。

## 綁定步驟

1. 登入 Zeabur，前往[「伺服器」選單](https://zeabur.com/servers)。
2. 選擇「叢集」後，點選右上角的「建立」。
   ![cluster page](/cluster/cluster-page.webp)
3. 在建立流程中選擇 **Connect your own Cluster**。
   ![create cluster](/cluster/create-cluster.webp)
4. 上傳你的 Kubeconfig。Zeabur 會自動解析 kubeconfig 中的叢集資訊，並嘗試連線驗證。
   ![upload kubeconfig](/cluster/upload-kubeconfig.webp)
5. 設定 Provider（如 `LINODE`）、國家（如 `US`）、城市、大洲等資訊，主要用於在前端顯示，可以隨意填寫。
   ![cluster provider](/cluster/update-details.webp)
6. 點選 **Connect Cluster** 後等待 Zeabur 完成叢集建立與初始化，建立時間大約會耗時 3-5 分鐘。
7. 建立完成後，你會在伺服器列表中看到新的叢集，可直接在 Zeabur 上用它來建立專案與部署服務。
   ![cluster information](/cluster/cluster-information.webp)

<Callout type="warning">
  綁定既有叢集不會自動重建你的 Kubernetes 環境。若你的叢集使用自訂網路、Ingress、StorageClass 或安全政策，請先確認這些設定與 Zeabur 的部署需求相容。
</Callout>
