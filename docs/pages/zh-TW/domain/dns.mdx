# DNS 記錄


每個已註冊網域的詳情頁都有一個 DNS 編輯器。在管理列表中點擊任意網域即可存取。

## 支援的記錄類型

| 類型 | 說明 | 代理支援 |
|------|------|---------|
| **A** | 將網域對應到 IPv4 位址 | 是 |
| **AAAA** | 將網域對應到 IPv6 位址 | 是 |
| **CNAME** | 另一個網域名稱的別名 | 是 |
| **MX** | 郵件交換伺服器 | 否 |
| **TXT** | 任意文字（SPF、DKIM、驗證等） | 否 |
| **SRV** | 服務定位器 | 否 |
| **CAA** | 憑證授權機構授權 | 否 |
| **NS** | 名稱伺服器委派 | 否 |

## 新增記錄

<Steps>

### 開啟 DNS 編輯器

進入網域詳情頁，點擊 DNS 記錄表格上方的 **新增記錄**。

### 填寫記錄資訊

- **類型**：從下拉選單中選擇記錄類型。
- **名稱**：子網域或 `@` 表示根網域。
- **內容**：記錄值（IP 位址、主機名稱、文字等）。
- **TTL**：存活時間。從預設值中選擇（auto、60s、300s、3600s、86400s 等）。啟用代理時，TTL 自動設為 auto。
- **優先順序**：MX 和 SRV 記錄必填。
- **代理**：僅 A、AAAA 和 CNAME 記錄可用。參見下方 [Cloudflare 代理](#cloudflare-代理)。

### 儲存

點擊 **儲存** 建立記錄。變更通常在幾分鐘內生效。

</Steps>

## 編輯和刪除記錄

- 點擊記錄列的 **編輯圖示** 修改內容、TTL、優先順序或代理設定。記錄類型和名稱在建立後無法變更——如需變更，請刪除後重新建立。
- 點擊 **刪除圖示** 移除記錄。

## Cloudflare 代理

A、AAAA 和 CNAME 記錄有代理開關。啟用後：

- 流量透過 Cloudflare 路由，提供 DDoS 防護、快取和 SSL 終端。
- TTL 自動設為 **Auto**，無法手動變更。
- 實際來源 IP 位址在 DNS 查詢中被隱藏。

<Callout type="info">
  代理預設為關閉。當你需要 Cloudflare 的防護和效能功能時，為該記錄啟用代理。
</Callout>
