# 專用伺服器

你可以使用 Zeabur 方便地將各種服務部署到兩種不同類型的伺服器上：共享叢集（Shared Cluster）和專用伺服器（Dedicated Server），兩種類型的計算資源適合不同的使用情境。

## 共享叢集

共享叢集是最簡單使用 Zeabur 部署服務的方式，因為你無需理解任何伺服器相關的知識和設定，甚至感知不到伺服器的概念，僅需要選擇適合的機房位置即可。

每個共享叢集都是由 Zeabur 管理的多台伺服器所組合而成的計算叢集，一個叢集中包含多台伺服器，你部署的服務會和其他選擇同個叢集的使用者所部署的服務共享這些伺服器。

共享叢集適合用於部署硬體需求波動較大的服務，因為他採用按量計費的方式，在硬體需求低的時候可以節省成本，而在需求高的時候會自動擴容，因此不會發生硬體資源不足的情況。

目前 Zeabur 已有以下幾個共享叢集：

- 美國加州（由 Amazon Web Services 提供）
- 德國法蘭克福（由 Amazon Web Services 提供）
- 日本東京（由 Amazon Web Services 提供）
- 香港（由 Amazon Web Services 提供）
- 台北（由 Google Cloud 提供）—— 僅提供給企業客戶使用
- 新加坡（由 DigitalOcean 提供）—— 僅提供給企業客戶使用
- 美國紐澤西（由 Vultr 提供）

## 專用伺服器

專用伺服器是只屬於你一個人的伺服器，因此服務不會受到其他使用者服務的影響，硬體資源也完全由你獨享，不會有被其他使用者佔用的情況。

專用伺服器適合以下幾種使用情境：

- 已在其他雲端廠商租用或購買伺服器的使用者，可以一鍵將伺服器註冊到 Zeabur 上，如此一來就不需要支付硬體資源的使用費用。 
- 對服務穩定性及安全性有較高要求的使用者，例如大型系統的生產環境。
- 服務主要訪問地區尚未有共享叢集區域的使用者，可以租用或自架符合自己地理位置的專用伺服器。

你可以在一台專用伺服器上建立多個專案，但你需要自行注意這些服務的硬體資源使用情況，避免因為專用伺服器的硬體資源不足導致服務受到影響。

## 註冊專用伺服器

你可以選擇在 Zeabur 上購買一台專用伺服器（正在開發中），或者是註冊一台你已經在其他雲端廠商租用或購買的伺服器。

如果你想瞭解如何註冊自己的伺服器，可以參考[註冊專用伺服器](./dedicated-server/register)。

## 已知限制

- 目前專用伺服器不支援 Serverless 模式部署。 
- 目前專用伺服器不支援 `.zeabur.app` 免費子網域，你需要有自己的網域才能綁定到部署在專用伺服器的服務上。
