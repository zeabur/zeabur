# 從模板服務 Fork 程式碼

在 Zeabur 上，你可以使用模板服務快速部署應用。一個模板通常包含多個服務，每個服務要麼是 Prebuilt 服務（從 Docker 映像部署），要麼是 Git 服務（從 GitHub 部署）。

預設情況下，當你部署包含 Git 服務的模板時，Zeabur 會直接從模板作者的 GitHub 儲存庫部署服務，並自動將 CI/CD 綁定到模板作者設定的儲存庫。這意味著當模板作者向其儲存庫推送新提交時，Zeabur 會自動觸發 CI/CD 並將新版本部署到你的專案中。

但有時你希望基於模板作者的程式碼進行修改，加入自己的功能或更改設定等。為此，Zeabur 提供了一鍵 Fork 的便利功能。

## 如何從模板服務 Fork 程式碼

以 [Zeabur 官方 Express.js 模板](https://zeabur.com/templates/W96I3Z) 為例，部署該模板後，進入 express 服務的設定頁面。你會看到 Git Repository 欄位目前值為 `https://github.com/zeabur/expressjs-template`，這是該模板指定的 GitHub 儲存庫。

若想基於此模板修改程式碼，可點擊下方的 Fork Repository 按鈕：

![Express.js github fork](/template/fork-git-repo-from-template/Expressjs_github_fork.png)

點擊後會彈出視窗詢問儲存庫名稱。請使用尚未使用的儲存庫名稱，否則 Fork 可能失敗。名稱下方還有輸入框，可選擇是否將儲存庫設為私人，以防止他人看到你的程式碼修改：

![git fork](/template/fork-git-repo-from-template/git_fork.png)

選好名稱後點擊 Fork。若尚未授權 GitHub，可能會跳轉至 GitHub 的 OAuth 授權頁面。授權完成後會顯示 Fork 載入畫面：

![Configure Github](/template/fork-git-repo-from-template/Configure_Github.png)

![github connection](/template/fork-git-repo-from-template/github_connection.png)

完成後，你的 GitHub 帳戶中會新增一個與原程式碼相同的儲存庫。

Zeabur 會自動將該模板的程式碼 Fork 到你的 GitHub 帳戶，並將這個 Fork 後的儲存庫設為你服務的 Git Repository。這意味著你在 GitHub 上修改這個 Fork 儲存庫並推送提交後，Zeabur 會自動觸發 CI/CD 並將新版本部署到你的專案中。
