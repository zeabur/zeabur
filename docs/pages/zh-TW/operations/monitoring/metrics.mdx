# 監控指標

Zeabur 為每個服務提供即時的資源使用指標，幫助你監控效能並做出明智的擴展決策。

## 查看指標

查看服務指標的步驟：

1. 在 Zeabur 儀表板中開啟你的專案。
2. 選取你要監控的服務。
3. 點擊 **Metrics** 分頁。

指標儀表板會即時顯示以下圖表：

- **CPU 使用率** — 目前及歷史 CPU 消耗量。
- **記憶體使用量** — 目前及歷史 RAM 消耗量，包括已設定的上限。
- **網路流量** — 輸入和輸出的資料傳輸量。
- **磁碟使用量** — 即將推出（Coming soon）。

## 了解資源使用情況

### CPU

CPU 使用率以已分配 CPU 資源的百分比顯示。CPU 使用率飆升可能表示：

- 請求量過高
- 運算密集的操作
- 可以最佳化的低效程式碼路徑

### 記憶體

記憶體使用量顯示你的服務相對於設定上限所消耗的 RAM。如果你的服務持續接近記憶體上限，請考慮在[服務設定](/operations/scaling)中增加分配量，以避免記憶體不足（OOM）導致的重新啟動。

### 網路

網路指標顯示服務傳入和傳出的資料量。這有助於了解流量模式及估算頻寬成本。

## 服務用量與計費

如需詳細的資源消耗及相關費用明細，請前往專案設定中的 **Service Usage** 頁面。此頁面會顯示：

- 計費週期內的累計 CPU 和記憶體使用量
- 網路出口流量費用
- 各服務的費用明細

詳情請參閱 [Service Usage](/operations/service-usage)，了解用量如何對應計費。

<Callout>
  Zeabur 採用以用量計費的定價模式。定期監控你的指標有助於掌握資源消耗情況，避免意外費用。詳細定價資訊請參閱[方案價格](/pricing)。
</Callout>

## 利用指標引導擴展

指標是決定何時以及如何擴展的關鍵依據：

- **CPU 持續超過 80%** — 考慮增加 CPU 分配或最佳化你的應用程式。
- **記憶體接近上限** — 增加記憶體分配以防止 OOM 崩潰。
- **流量穩定增長** — 提前規劃，在使用量飆升導致效能下降之前調整資源。

如需調整服務資源的相關資訊，請參閱[擴展](/operations/scaling)。
