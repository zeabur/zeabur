# 擴展

Zeabur 允許你調整分配給服務的資源，使其能夠應對不同的工作負載。你可以透過增加 CPU 和記憶體進行垂直擴展，並選擇適合需求的基礎設施層級。

## 垂直擴展

垂直擴展是指增加（或減少）分配給單一服務實例的 CPU 和記憶體。

調整資源的步驟：

1. 在 Zeabur 儀表板中開啟你的服務。
2. 前往 **Settings**。
3. 在 **Resources** 區段中，設定所需的 CPU 和記憶體上限。
4. 儲存變更。新的資源分配將在下次部署時生效。

### 資源方案

Zeabur 提供多種資源層級。預設分配適用於大多數輕量級服務，但你可以為更高需求的工作負載（如資料庫、建置伺服器或 AI 推論）增加資源。

<Callout>
  資源分配會直接影響計費。較高的 CPU 和記憶體上限會產生較高的費用。詳細的單位定價請參閱[方案價格](/pricing)。
</Callout>

## 專用伺服器

對於需要保證資源、可預測效能或符合資料駐留要求的工作負載，Zeabur 提供**專用伺服器**。專用伺服器為你提供不與其他使用者共享的隔離運算環境。

專用伺服器的優勢包括：

- **保證資源** — 不受鄰居干擾（noisy neighbor）問題影響。
- **自訂區域** — 部署在特定的地理位置。
- **更高的資源上限** — 執行超出共享叢集限制的服務。

設定說明和可用配置請參閱[專用伺服器](/dedicated-server)。

## 擴展策略

| 情境 | 建議 |
| --- | --- |
| 偶發的流量高峰 | 增加 CPU / 記憶體以應對尖峰負載 |
| 持續性高流量 | 使用專用伺服器以獲得可預測的效能 |
| 成本最佳化 | 根據[監控指標](/operations/metrics)合理調整資源 |
| 暫時閒置 | [暫停](/operations/deployment/suspend-service)服務以停止計費 |

## 監控資源使用量

使用[監控指標](/operations/metrics)儀表板即時監控 CPU、記憶體和網路使用量。這些資料有助於你判斷何時需要擴展或縮減。如需費用分析，請查看 [Service Usage](/operations/service-usage) 頁面。
