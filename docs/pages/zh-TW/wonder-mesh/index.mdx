# Wonder Mesh

Wonder Mesh 讓你將自己的電腦或伺服器變成 Zeabur 運算節點。無論你的裝置在家裡、辦公室，或是在任何網路環境之後，都能安全地連接到 Zeabur 平台來部署和執行你的服務——不需要公開 IP 或通訊埠轉發。

<Callout type="info">
  Wonder Mesh 目前處於 **Beta** 階段。你可以在 Zeabur 控制台的[伺服器頁面](https://zeabur.com/servers)申請存取權限。
  已經有存取權限了嗎？請參閱[快速開始指南](/wonder-mesh/get-started)來設定你的第一台 Wonder Mesh 伺服器。
</Callout>

## 運作原理

Wonder Mesh 使用 [WireGuard](https://www.wireguard.com/) 在你的裝置與 Zeabur 平台之間建立一條私密的加密通道。這條通道可以穿越 NAT、防火牆和動態 IP——你的裝置會獲得一個穩定的內部 Mesh IP 位址，Zeabur 透過 SSH 使用這個位址來管理部署。你不需要自行設定任何網路。

## 為什麼選擇 Wonder Mesh？

通常，Zeabur 服務運行在 Zeabur 管理的雲端伺服器上。Wonder Mesh 改變了這一點，讓你使用**自己的硬體**——筆記型電腦、桌上型電腦、家用實驗室伺服器，甚至是家裡的迷你電腦——作為部署目標，同時享有完整的 Zeabur 體驗。

- **家用實驗室愛好者**：為你的家用伺服器獲得雲端等級的部署和管理體驗。
- **開發者**：使用真實的編排系統在本機測試服務，準備好後再部署到雲端。
- **自架倡導者**：將資料保留在自己的硬體上，同時不犧牲一鍵部署的便利性。

## 主要功能

- **一鍵設定**：在控制台建立伺服器，在你的裝置上執行一條指令，即可開始部署。
- **穿越 NAT 運作**：不需要公開 IP、固定 IP 或通訊埠轉發。Wonder Mesh 建立安全的網狀網路，可在任何網路環境下運作。
- **完整的 Zeabur 整合**：從市集部署模板、推送程式碼、綁定網域、檢視日誌和監控資源——就像任何其他 Zeabur 伺服器一樣。
- **伺服器管理**：從控制台監控 CPU、記憶體和磁碟使用量，存取 SSH，並查看即時伺服器狀態。

## 支援的平台

| 作業系統 | 架構 | 備註 |
|---------|------|------|
| Linux | amd64 / arm64 | 直接安裝 |
| macOS | amd64 / arm64 | 透過 Lima 虛擬機執行 |

從 Mac mini 到 Raspberry Pi，任何 Linux 或 macOS 裝置都可以成為 Zeabur 伺服器。

## 常見問題

### 我的裝置需要保持開機嗎？

是的。裝置必須保持開機並連接到網際網路，部署在上面的服務才能持續存取。

### 如果我的裝置離線會怎樣？

部署在該裝置上的服務將暫時無法使用。一旦裝置重新上線，連線會自動恢復。

### 有任何網路需求嗎？

只需要網際網路連線。不需要公開 IP、不需要通訊埠轉發，也不需要防火牆變更。

### 如何存取部署在我裝置上的服務？

你有兩個選擇：

- **從網際網路**：像其他 Zeabur 伺服器一樣為你的服務綁定網域。詳細的網域綁定和通訊埠轉發說明請參閱[公網存取](/networking/public)。
- **從區域網路**：在服務頁面打開 **Networking** 分頁，找到區域網路存取位址。詳細的裝置區域網路 IP 查找方式請參閱[快速開始指南](/wonder-mesh/get-started#accessing-services)。

同一專案中的服務也可以使用內部主機名稱互相通訊。詳情請參閱[服務內網](/networking/private)。
