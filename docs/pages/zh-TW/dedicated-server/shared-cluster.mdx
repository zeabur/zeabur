# 共享叢集（已停止服務）

<Callout type="error">
共享叢集已於 **2024 年停止服務**，所有新專案請使用[專用伺服器](/dedicated-server)。
</Callout>

## 什麼是共享叢集

共享叢集是 Zeabur 早期提供的伺服器方案。在共享叢集模式下，多位使用者的服務會部署在同一組 Kubernetes 叢集中，共享底層的 CPU、記憶體和網路資源。使用者無需管理硬體，只需按實際資源使用量付費。

## 與專用伺服器的差異

| 項目 | 共享叢集（已停止） | 專用伺服器 |
| --- | --- | --- |
| 資源隔離 | 多租戶共用硬體 | 完整獨享硬體 |
| 計費方式 | 按用量計費（vCPU / 記憶體 / 流量） | 按月固定費用 |
| 伺服器位置 | 固定區域（AWS） | 自攜主機（BYOH），區域不限 |
| 效能穩定性 | 可能受鄰居影響 | 資源穩定可預測 |
| 適用場景 | 小型專案、開發測試 | 生產環境、高穩定性需求 |

## 停止服務的原因

隨著專用伺服器方案的成熟，Zeabur 決定將共享叢集整合至專用伺服器架構，提供更好的資源隔離與效能保障。共享叢集的按量計費模式仍保留在[定價方案](/pricing/pricing-plans)中作為費率參考。

## 從共享叢集遷移到專用伺服器

如果你的服務仍在共享叢集上運作，請依照以下步驟遷移至專用伺服器：

1. **準備專用伺服器** — 在 [Zeabur 伺服器頁面](https://zeabur.com/servers) 購買或[註冊自己的伺服器](/dedicated-server/register)。
2. **備份資料** — 使用[備份與還原](/operations/backup-restore)功能，為所有有狀態服務（資料庫、持久儲存）建立備份。
3. **匯出專案** — 透過[匯出專案](/deploy/export-project)功能匯出專案的 YAML 配置。
4. **在專用伺服器上重新部署** — 在專用伺服器的區域建立新專案，匯入配置並重新部署服務。
5. **還原資料** — 將步驟 2 的備份還原到新服務中。
6. **切換 DNS** — 將自訂網域的 DNS 記錄指向新服務的網域。
7. **驗證與清理** — 確認新環境運作正常後，移除共享叢集上的舊服務。

<Callout type="info">
遷移過程中如遇問題，請到 [社群論壇](https://zeabur.com/forum) 聯絡我們。
</Callout>
