# 預覽環境

有些時候，你可能想要為一個 pull request 部署一個預覽環境。這對於在將更改合併到主分支之前測試更改非常有用。
我們為所有部署在 Zeabur 上的服務提供了一種簡單的方法來實現這一點。

這篇文件將會向你展示如何在 Zeabur 上一鍵啟用 pull requests 預覽環境。

<Callout>本功能僅對 Developer Plan 和 Team Plan 使用者開放</Callout>

## 選擇一個 Git 服務

正如在 [快速開始](/get-started) 中提到的，Zeabur 上有兩種類型的服務，包括 **Git 服務** 和 **Marketplace 服務**。
Pull requests 預覽環境僅適用於 Git 服務。

如果你還沒有建立 Git 服務，你可以按照 [快速開始](/get-started) 中的指導建立一個。

## 啟用 pull requests 預覽環境

打開你想要啟用 pull requests 預覽環境的服務，滾動到底部展開設定頁籤。
你會看到一個名為 **Pull Requests Preview** 的開關按鈕。點選開關按鈕來啟用 pull requests 預覽環境。

![enable](/environment/preview/enable.png)

## 查看預覽

在你成功啟用 pull requests 預覽環境後，你可以在你的 Git 儲存庫上建立一個 pull request。
當你建立一個 pull request 時，Zeabur 將會自動為你建立一個預覽環境。

除此之外，Zeabur 還會自動為你綁定一個唯一的網域，以便你輕鬆地造訪預覽服務。
你可以在我們的控制台中找到這個網域，也可以直接在你的 Git 儲存庫中點選部署連結。

![link](/environment/preview/link.png)

不要擔心，當 pull request 被合併或關閉後，預覽環境將會被自動刪除。
