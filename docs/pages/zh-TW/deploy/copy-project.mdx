# 複製專案

Zeabur 提供**專案複製功能（Project Copy）**，讓您可以輕鬆地將專案複製到任何地區。

您只需要前往專案設定頁面，就能找到**一鍵專案複製功能**。這項功能會自動將您的專案完整複製到您選擇的任何地區，包含專案架構、環境變數，甚至是您的硬碟資料。整個過程只需短短幾分鐘，大幅節省您的時間與心力。

- **無硬碟專案：** 如果您的專案沒有掛載硬碟，複製過程甚至能做到**零停機**，體驗就像重新啟動服務一樣流暢。
- **有硬碟專案：** 如果您的專案有掛載硬碟，複製時間會根據資料量大小而定。我們後台會自動將您的硬碟資料備份至 S3 儲存空間，再還原到新的專案中，確保您的資料完整無虞。

## 複製內容說明

<Callout>
    專案複製功能會完整複製您的專案，**包括所有資料**。複製後需要重新綁定網域，不會自動繼承原專案的網域設定。
</Callout>

複製專案時，以下內容會被完整複製：
- **專案架構**：所有服務的配置和關係
- **服務類型**：Docker 映像、Git 儲存庫等
- **環境變數**：所有配置的環境變數
- **硬碟資料**：硬碟中的所有資料
- **網路設定**：連接埠配置和網路規則

## 注意事項

1. **零停機複製**：無硬碟的專案可以實現零停機複製，有硬碟的專案複製時間取決於資料量大小。

2. **資料完整性**：複製過程中所有資料都會被完整備份和還原，確保資料安全。

3. **網域重新綁定**：複製後的專案需要重新綁定網域，不會自動繼承原專案的網域設定。

4. **資源消耗**：複製過程中會臨時消耗額外的儲存空間用於資料備份。
