# 更新 Docker 服務的映像引用

Zeabur 能讓你在建立服務後更新映像的引用，適合用來更新 Docker 服務的映像版本，或者是切換服務到其他映像。

<Callout type="info">
    只有 **Docker 服務** 可以更新映像引用。Git 和 Once 服務考慮到沒有映像引用，故無法更改。
</Callout>

## 更新 tag

「Tag」（標籤）是映像引用中指定映像版本的部分。例如，在 `nginx:latest` 映像引用中，`latest` 是 tag。你可以藉由更新 tag 來切換到映像的不同版本。

首先展開服務的「設定」區域，你會看到「服務映像」區塊。這裡可以讓你更新服務的映像引用。

![服務映像](/deploy/update-image-reference/service-image.webp)

點選並修改映像引用中的 tag 部分（即第二個輸入框），你會看到允許你套用新 tag 的 **儲存** 按鈕。

![更新 tag](/deploy/update-image-reference/update-tag.webp)

儲存更動後，Zeabur 會自動重新啟動你的服務並套上新的映像引用。如果無法拉取映像，Zeabur 會提示你再次修改映像引用。

![拉取失敗](/deploy/update-image-reference/pull-failed.webp)

## 更新 repository

「Repository」（映像庫）是映像引用中指定映像名稱的部分。例如，在映像引用 `nginx:latest` 中，`nginx` 是 repository 名稱。你可以透過更新 repository 來切換到不同的映像。

開啟服務的「設定」區域，你會看到「服務映像」區塊。在這裡，你可以更新服務的映像 repository。

![服務映像](/deploy/update-image-reference/service-image.webp)

點選並修改映像引用中的 repository 部分（即第一個輸入框），你會看到允許你套用新 repository 的 **儲存** 按鈕。

![更新 repository](/deploy/update-image-reference/update-repository.webp)

儲存更動後，Zeabur 會自動重新啟動你的服務並套上新的映像引用。如果無法拉取映像，Zeabur 會提示你再次修改映像引用。

![拉取失敗](/deploy/update-image-reference/pull-failed.webp)
