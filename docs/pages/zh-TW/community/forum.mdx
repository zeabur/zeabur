# 社群論壇

[Zeabur 社群論壇](https://zeabur.com/forum) 是使用者討論、提問和獲得 Zeabur 相關幫助的官方平台。無論你遇到服務問題、需要手機驗證，還是有帳單相關的疑問，論壇都是你的首選。

## 支援優先順序

我們致力於為每位使用者提供幫助。為了確保最佳體驗，我們會根據你的方案優先處理支援請求：

| 方案 | 預期回應時間 |
| --- | --- |
| Team 方案 | 12 小時內 |
| Dev 方案 | 3 個工作天內 |
| Free 方案 | 5 個工作天內 |

為了讓我們能盡快解決你的問題，請在建立貼文時盡可能提供詳細的資訊。你提供的細節越多，我們需要追問的問題就越少。

<Callout type="info">
  如果你是 **Team 方案**使用者，我們會在你最常使用的通訊軟體（例如 Discord、WhatsApp）中建立一個私人群組聊天。你可以直接在該私人頻道中提出所有問題，而不需要在論壇上發文——這為你提供更高的隱私保護、更直覺的支援體驗，並幫助我們的工作人員更快速地了解你的情況。
</Callout>

## 建立新貼文

前往 [zeabur.com/forum/new](https://zeabur.com/forum/new) 建立新貼文。你需要填寫以下欄位。

![new post](/community/forum/new-post.png)

### 問題類型

選擇你遇到的問題類型。表單欄位會根據你選擇的問題類型而有所不同。

- **Service Issue（服務問題）** — 部署服務相關的問題（例如：部署失敗、執行時錯誤、網路問題）。
- **Phone Verification（手機驗證）** — 請求帳戶手機號碼驗證。
- **Refund Request（退款請求）** — 請求伺服器、訂閱或餘額的退款。
- **Others（其他）** — 以上類別未涵蓋的任何其他問題。

### 標題

簡短描述你的問題（最多 150 個字元）。請清楚且具體，以便我們的團隊能快速了解你的問題。

### 基本資訊

<Callout>
  根據問題類型的不同，你可能需要提供額外的資訊。這些資訊不會被公開，僅用於加速問題排查——它能讓我們的工作人員在調查你的問題時更加清楚明瞭。
</Callout>

#### Service Issue（服務問題）

選擇 **Project** 或 **Server** 分頁：

- **Project 分頁**：從下拉選單中選擇你的 **Project** 和 **Service**。這是必填的，以便我們能定位你的部署。
- **Server 分頁**：如果問題與特定伺服器相關，請從下拉選單中選擇你的 **Server**。

你也可以貼上 **Error Message**（選填）。如果你在 Zeabur 控制台右下角看到錯誤提示，請點擊提示上的複製按鈕，並將錯誤訊息貼到此欄位。

#### Phone Verification（手機驗證）

- **Country / Region**：選擇你的國家或地區。
- **Phone Number**：輸入你要驗證的電話號碼。

<Callout>
  手機驗證貼文為私密貼文，僅你和 Zeabur 工作人員可見。這確保你的個人資訊受到保護，同時讓我們的團隊獲得所需的資訊來幫助你。
</Callout>

#### Refund Request（退款請求）

根據你要退款的項目選擇 **Server**、**Subscription** 或 **Balance** 分頁：

- **Server 分頁**：選擇你的 **Server** 和退款 **Reason**。
- **Subscription 分頁**：選擇退款 **Reason**。
- **Balance 分頁**：選擇退款 **Reason**。

<Callout>
  退款請求貼文為私密貼文，僅你和 Zeabur 工作人員可見。這確保你的帳單資訊安全，同時讓我們的團隊能有效率地處理你的請求。
</Callout>

#### Others（其他）

不需要額外的基本資訊。請在內容欄位中盡可能提供詳細的描述。

### 內容

使用富文字編輯器詳細描述你的問題。你可以使用標題、列表、程式碼區塊、連結等格式。

**撰寫優質貼文的技巧：**

- 描述你嘗試做什麼以及實際發生了什麼。
- 包含重現問題的步驟。
- 提到問題開始前你做了哪些更改。
- 貼上相關的日誌或錯誤訊息。
- 透過拖放或貼上的方式附加截圖或檔案。

你的描述越詳細，我們就能越快幫助你，而不需要反覆詢問。
