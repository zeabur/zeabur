# 部署 Vapor 專案

[Vapor](https://vapor.codes) 是 Swift 生態中較為知名的 Web 框架，藉由 Vapor 的幫助，你可以使用 Swift 來開發一個基於 Http 的 Web 應用程式。

本文將介紹如何將你的 Vapor 專案部署到 Zeabur。

## 事前準備

請確保你的電腦已經準備好 Swift 和 Vapor 的開發環境：

```bash
brew install vapor
```

## 建立 Vapor 專案

有兩個方法可以建立 Vapor 專案：

### 使用 Vapor 指令工具

```bash
vapor new <project-name>
cd <project-name>
vapor xcode
```

使用 Vapor 指令建立的專案，預設帶有 `Dockerfile` 及 `docker-compose.yml`。好消息是，你不需要 Dockerfile 也能直接在 Zeabur 部署，所以你可以直接刪除這兩個檔案：

```bash
rm Dockerfile docker-compose.yml
```

### 直接使用 Zeabur 為 Vapor 準備的模板

打開 https://github.com/zeabur/vapor-template ，點擊 `Use this template` 按鈕，輸入專案名稱後，點擊 `Create repository from template` 按鈕。

```bash
git clone <your-repository-url>
cd <project-name>
vapor xcode
```

## 開發

在開發時，你可以直接在本機執行專案：

```bash
vapor run
```

接下來，根據你的需求修改 Swift 程式碼。

## 部署

首先，確保你的所有改動都已經同步到 GitHub 上：

```bash
git add .
git commit -m "your commit message"
git push
```

接下來，前往 [Zeabur 控制台](https://zeabur.com) 並建立一個新專案。

然後，點選 `Deploy New Service` 按鈕，選擇從 GitHub 部署。

<Callout>
    如果你是第一次在 Zeabur 上從 GitHub 部署專案，你需要先配置並安裝 GitHub App。
    你可以在 [這裡](/deploy/github) 查看更詳細的指南。
</Callout>

當你選擇好儲存庫和分支後，Zeabur 將會自動開始建置你的服務。

Zeabur 會自動檢測到你的服務是由 Vapor 建置的，所以你不需要做任何額外的配置。你的部署將會在幾分鐘內完成。

## 下一步

部署完成後，你可能需要為你的 Vapor 網站配置網域。

打開服務頁面的 "Domain" 分頁，然後點選 "Generate Domain" 或 "Custom Domain"。

關於如何為你的服務綁定網域的更多資訊，請參考 [綁定網域](/deploy/domain-binding)。
