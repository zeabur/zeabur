# 團隊

**團隊（Team）** 是一個獨立的工作區，讓多人能在同一個實體下共享專案、伺服器與帳單。每個團隊擁有自己的訂閱、付款方式與發票，與任何個人帳號完全獨立。

本頁說明什麼是團隊、團隊與個人帳號的差異，以及啟用 **Team 方案** 後可以做什麼。

## 個人帳號 vs 團隊

每位 Zeabur 使用者都擁有一個 **個人工作區**（你的個人帳號）。同時你也可以加入一個或多個團隊。每個團隊都是獨立的工作區，擁有各自的資源、成員與帳單。

你可以透過控制台側邊欄中的 **工作區切換器** 在個人工作區與你所屬的任一團隊之間切換。

| | 個人工作區 | 團隊工作區 |
| --- | --- | --- |
| 擁有者 | 你本人 | 團隊實體 |
| 訂閱 | 你的個人方案（Free / Dev / Pro） | Team 方案（單一級距） |
| 付款方式 | 你的卡片 | 團隊的卡片 |
| 發票 | 開立給你 | 開立給團隊 |
| 可見的專案 / 伺服器 | 你的 | 團隊的 |
| 成員 | 不適用 | 多名使用者搭配角色 |

一位使用者可以同時擁有有效的個人訂閱 **以及** 加入一個有自己有效訂閱的團隊。兩者的計費與管理彼此獨立 — 為團隊新增成員不會影響你的個人帳單，降級你的個人方案也不會影響你所加入的任何團隊。

<Callout type="info">
  在你的帳單歷史紀錄中，使用舊版「Team」級距的個人訂閱現在會顯示為 **Team Plan（舊版）**。這與本頁所述的團隊工作區無關。目前的團隊產品即為本頁所描述的工作區。
</Callout>

## 啟用 Team 方案

建立團隊後你會得到一個空的工作區，但團隊必須 **啟用 Team 方案** 後，成員才能建立專案、部署服務或使用任何由團隊擁有的資源。

<Callout type="warning">
  剛建立的團隊處於 **未啟用** 狀態。在團隊管理員啟用 Team 方案之前，所有團隊成員都無法執行下列操作：

  - 在團隊工作區中建立專案
  - 從模板部署服務
  - 管理由團隊擁有的伺服器與叢集
  - 在團隊下使用 AI Hub
  - 透過 Z‑Send 在團隊下寄送郵件
</Callout>

未啟用的團隊仍可進行的操作 — 讓管理員能完成設定：

- 建立團隊並命名
- 邀請成員並指派角色
- 更新團隊資料（名稱、頭像、聯絡電子郵件）
- 綁定付款方式
- **啟用 Team 方案**
- 隨時切換回個人工作區
- 刪除團隊（僅在沒有任何專案或伺服器時）

要啟用 Team 方案，團隊管理員需在團隊工作區中前往 **帳號 → 訂閱**，並點擊 **啟用 Team 方案**。控制台會在付款前顯示依席次計算的價格預覽。

## 角色

團隊有三種角色，由管理員在邀請成員時指定：

- **Administrator（管理員）** — 完整控制權：成員、帳單、所有團隊擁有的資源，可刪除團隊。
- **Editor（編輯者）** — 可建立與修改團隊擁有的專案、服務與資源，但不能管理成員、帳單或刪除團隊。
- **Viewer（檢視者）** — 對團隊資源僅有唯讀權限。不能建立、修改或刪除任何內容。

建立團隊的使用者會自動成為團隊的第一位管理員。團隊中必須至少保留一位管理員。

各角色完整的權限矩陣，請參閱 [管理成員](/team/manage-members)。

## 獨立帳單

每個團隊都有自己的 Stripe 客戶紀錄。這代表：

- 團隊發票的抬頭使用團隊的帳單資訊（法定名稱、地址、稅號），而非任何個別成員。
- 團隊擁有自己的付款方式，與任何管理員的個人卡片分開。
- 團隊擁有自己的信用額度 — 加值到團隊的點數只會用於團隊擁有的資源。
- 在控制台中切換工作區，會同時切換你正在檢視的帳單情境。

關於付款方式、發票、預付點數，以及 Team 方案席次費用的計算方式，請參閱 [團隊帳單](/team/billing)。

## 將個人專案與伺服器轉移到團隊

如果你在個人工作區中已有專案或專用伺服器，你所屬團隊的管理員可以將其擁有權轉移到團隊。轉移後，帳單與資源配額會歸屬到團隊，原本的個人擁有者仍可作為團隊成員繼續在其上工作。

請參閱 [將擁有權轉移到團隊](/team/transfer-ownership)。

## 何時使用團隊，何時使用專案 / 伺服器協作者

如果你只想讓某人存取單一專案中的服務，[將他邀請為專案協作者](/operations/team/invite-member) 即可。如果你想分享單一專用伺服器，請從伺服器設定頁面將他邀請為伺服器協作者。當你想跨多個專案、伺服器與成員共用帳單與單一工作區時，**團隊** 才是正確的選擇 — 此時請將他邀請為團隊成員。

關於三者差異的完整說明，請參閱 [管理成員](/team/manage-members)。
